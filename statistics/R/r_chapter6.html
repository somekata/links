生存解析と時間軸
〜ログランク検定が捉える命の軌跡〜

「いつ」イベントが起きるかを分析する——死亡・再発・退院・離脱。時間という次元を組み込んだ統計の世界を、カプランマイヤー曲線からCox回帰まで体系的に学ぶ。

📌 この章の学習目標

  • 生存解析が通常の検定と異なる理由（打ち切り・時間軸）を説明できる
  • Surv オブジェクトを作成し、カプランマイヤー曲線を描ける
  • ログランク検定で群間の生存曲線を比較できる
  • Cox比例ハザードモデルを構築し、ハザード比を解釈できる
  • 比例ハザード仮定をSchoenfeld残差で確認できる
  • 競合リスクの概念と Fine-Gray モデルを理解できる

6.1生存解析とは何か ── 時間をデータに加える

師
先生 これまでの検定は「ある時点のスナップショット」を比較するものだった。しかし医学では「いつ再発するか」「いつ死亡するか」という時間つきのイベントを扱う場面が多い。生存解析はまさにそのための道具だ。
学
学生 「死亡までの時間」なら、普通に平均や中央値を計算するのとどう違うんですか？
師
先生 それが「打ち切り（censoring）」の問題だ。研究期間中に死亡しなかった患者は、その後どうなるかわからない。試験終了時点でまだ生きているなら「少なくともその期間は生存した」という情報はある——しかしそれ以上はわからない。この「不完全な情報」を捨てずに使う仕組みが生存解析だ。

🕐 生存解析の基本概念

イベント（event）：死亡・再発・退院など、分析対象とする事象。研究によって定義が変わる。

生存時間（survival time）：観察開始からイベント発生（または打ち切り）までの時間。

打ち切り（censoring）：観察期間中にイベントが観察されなかった場合。「少なくともこの期間は生存した」という情報は利用する。

ハザード（hazard）：その瞬間にイベントが起きる「瞬間的なリスク」。t時点まで生存していた条件のもとでの、t時点でのイベント発生率。

6.2打ち切りの概念 ── 不完全な情報を捨てない

師
先生 5人の患者を追跡した例を見てみよう。観察期間は24ヶ月だ。
患者の追跡タイムライン（24ヶ月）
患者A
24ヶ月で死亡
患者B
14ヶ月で死亡
患者C
18ヶ月で転院（打ち切り）
患者D
24ヶ月で生存（打ち切り）
患者E
9ヶ月で同意撤回（打ち切り）
● 塗り潰し = イベント発生　○ 白抜き = 打ち切り（生存のまま離脱）
R# ── Surv オブジェクトの作成 ──
library(survival)
library(survminer)

# time：観察時間、event：1=イベント発生, 0=打ち切り
surv_obj <- Surv(time = patients$os_months,
                  event = patients$death)

# Surv の表示：+ は打ち切りを示す
head(surv_obj)
# [1]  24   14   18+  24+   9+

# ── survivalデータの作成（lubridate）──
library(lubridate)
patients <- patients |>
  mutate(
    os_months = interval(enrollment_date, last_contact_date) /
                dmonths(1),
    death     = as.integer(vital_status == "dead")
  )
打ち切りの種類： ①右側打ち切り（最も一般的。観察終了時点でまだ生存）②区間打ち切り（イベントがある区間内に起きたとわかるが正確な時刻不明）③左側打ち切り（観察開始前にイベントが起きた可能性）。多くの医学研究では右側打ち切りのみを扱う。

6.3カプランマイヤー曲線 ── 生存率を時間軸で描く

師
先生 カプランマイヤー（KM）曲線は「時刻tまで生存している確率」をノンパラメトリックに推定する方法だ。イベントが起きるたびに曲線が階段状に下がる。打ち切りの時点では曲線は動かず、「＋」記号でマークされる。
カプランマイヤー曲線のイメージ（治療群 vs 対照群）
R# ── KM曲線の推定と描画 ──
library(survival)
library(survminer)

# Surv オブジェクト作成
km_fit <- survfit(
  Surv(os_months, death) ~ treatment,
  data = patients
)

# 基本情報の確認
summary(km_fit, times = c(6, 12, 24))
# time n.risk n.event survival std.err lower95%CI upper95%CI
#    6    43       8    0.840   0.056      0.736      0.959
#   12    38      14    0.700   0.069      0.575      0.852
#   24    18      22    0.440   0.078      0.311      0.622

# ── ggsurvplot による論文品質の描画 ──
ggsurvplot(
  km_fit,
  data          = patients,
  pval          = TRUE,          # p値を自動表示（ログランク検定）
  conf.int      = TRUE,          # 95% CI の帯
  risk.table    = TRUE,          # リスク表（At Risk数）を下部に
  risk.table.height = 0.25,
  xlab          = "観察期間（月）",
  ylab          = "生存率",
  legend.labs   = c("治療群", "対照群"),
  palette       = c("#2a6b6e", "#8b3a2a"),
  ggtheme       = theme_classic(),
  surv.median.line = "hv"       # 中央生存時間に破線を引く
)

# ── 中央生存時間の取得 ──
surv_median(km_fit)
# strata         median  0.95LCL  0.95UCL
# treatment=A      20.5     17.1     24.0
# treatment=B      11.3      8.2     15.8
リスク表（at-risk table）の重要性： KM曲線の右端は少数の患者しか残っていないため、信頼区間が非常に広くなる。リスク表を必ず添えることで「この時点の曲線は何人の情報に基づくか」が読者に伝わる。論文投稿時には多くのジャーナルが必須としている。

6.4ログランク検定 ── 2つの生存曲線を比較する

師
先生 ログランク検定は「2群（以上）の生存曲線が同じか」を検定する。イベントが起きる各時点で「期待されるイベント数」と「観察されたイベント数」のずれを集積して検定統計量を作る。打ち切りを自然に組み込める点が優れている。 
R# ── ログランク検定 ──
lr_test <- survdiff(
  Surv(os_months, death) ~ treatment,
  data = patients
)
print(lr_test)
# Call:
# survdiff(formula = Surv(os_months, death) ~ treatment, data = patients)
#
#                  N Observed Expected (O-E)^2/E (O-E)^2/V
# treatment=A     50       22     30.1      2.18      5.12
# treatment=B     50       30     21.9      3.00      5.12
#
# Chisq= 5.1  on 1 degrees of freedom, p= 0.023

# ── p値の取り出し ──
p_logrank <- 1 - pchisq(lr_test$chisq, df = length(lr_test$n) - 1)

# ── 3群以上のログランク検定 ──
survdiff(Surv(os_months, death) ~ stage, data = patients)
# 有意な場合は pairwise.survdiff() で多重比較へ
pairwise.survdiff(
  Surv(os_months, death) ~ stage,
  data = patients,
  p.adjust.method = "BH"
)

# ── 重み付きログランク検定（Peto 法：早期差を重視）──
survdiff(Surv(os_months, death) ~ treatment,
         data = patients, rho = 1)  # rho=1: Wilcoxon型（早期差を重視）
ログランク検定の前提： 群間のハザード比が時間を通じて一定であること（比例ハザード仮定）。2つの曲線が交差するような場合（たとえば初期は対照群が優り、後期は治療群が優る）にはログランク検定は検出力が低い。この場合は重み付きログランク検定や複合エンドポイントを検討する。

6.5Cox比例ハザードモデル ── 複数の因子を同時に調整する

師
先生 ログランク検定は「治療群 vs 対照群」の1変数比較だ。しかし実際の研究では年齢・性別・病期などの交絡因子を同時に調整する必要がある。Cox比例ハザードモデルは重回帰の「生存解析版」で、複数の共変量を組み込んでハザード比（HR）を推定する。

📐 Coxモデルのハザード関数

h(t|X) = h₀(t) × exp(β₁X₁ + β₂X₂ + … + βₚXₚ)

h₀(t) はベースラインハザード（特定の分布を仮定しない）。exp(βᵢ) がハザード比（HR）で、「その共変量が1単位増えたとき、イベントリスクが何倍になるか」を示す。HR > 1はリスク増加、HR < 1はリスク減少。

R# ── 単変量Cox回帰 ──
cox_uni <- coxph(
  Surv(os_months, death) ~ treatment,
  data = patients
)
summary(cox_uni)
# coef   exp(coef) se(coef)  z       Pr(>|z|)
# -0.682   0.506    0.239   -2.85    0.0043
# HR = 0.51 → 治療群は対照群比でハザードが約半分

# ── 多変量Cox回帰（交絡因子を調整）──
cox_multi <- coxph(
  Surv(os_months, death) ~ treatment + age + sex + stage + hba1c,
  data = patients
)
summary(cox_multi)

# ── Forest plot（broom + ggplot2）──
library(broom)
cox_multi |>
  tidy(exponentiate = TRUE, conf.int = TRUE) |>
  filter(term != "(Intercept)") |>
  ggplot(aes(y = fct_rev(factor(term)),
             x = estimate, xmin = conf.low, xmax = conf.high)) +
  geom_pointrange(color = "#2a6b6e") +
  geom_vline(xintercept = 1, linetype = "dashed",
             color = "#8b3a2a") +
  scale_x_log10() +
  labs(title = "Cox比例ハザードモデル（Forest plot）",
       x = "ハザード比（95% CI）", y = NULL) +
  theme_classic()

# ── gtsummary で論文スタイルの出力 ──
library(gtsummary)
cox_multi |>
  tbl_regression(exponentiate = TRUE) |>
  bold_p(t = 0.05) |>
  bold_labels()
ハザード比の解釈： HR = 0.6 は「このグループのハザードが基準群の60%」すなわち「死亡リスクが40%低い」という意味だ。相対リスクと似ているが、時間を通じた瞬間的なリスクの比という点で異なる。HR の95%CIが1をまたがなければ統計的に有意。

6.6比例ハザード仮定の確認 ── モデルの前提を検証する

師
先生 Coxモデルの最重要な前提は「ハザード比が時間を通じて一定」という比例ハザード仮定だ。これが崩れているとモデルの解釈が危うくなる。Schoenfeld残差の検定と時間-残差プロットで確認するのが標準的だ。 
R# ── Schoenfeld残差によるPH仮定の検定 ──
ph_test <- cox.zph(cox_multi)
print(ph_test)
# chisq df      p
# treatment  0.82  1  0.365   # 有意でない → PH仮定 OK
# age        2.11  1  0.146
# sex        0.38  1  0.537
# stage      5.43  1  0.020   # 有意 → PH仮定が怪しい
# GLOBAL     8.12  4  0.087

# ── 残差プロット（PH仮定の視覚確認）──
ggcoxzph(ph_test, point.size = 0.5,
         ggtheme = theme_classic())

# ── PH仮定が成立しない場合の対処：時間交互作用の導入 ──
# stage の効果が時間によって変化する場合
cox_tv <- coxph(
  Surv(os_months, death) ~
    treatment + age + sex + stage + hba1c +
    tt(stage),                # tt()で時間変化共変量を指定
  data   = patients,
  tt     = function(x, t, ...) x * log(t)  # log(t)との交互作用
)

# ── 代替：層別Cox（strata）──
# 層別変数のHRは推定せず、各層で別のh₀(t)を持たせる
cox_strat <- coxph(
  Surv(os_months, death) ~
    treatment + age + sex + hba1c + strata(stage),
  data = patients
)

6.7競合リスクと拡張モデル ── 現実の複雑さに対応する

師
先生 がん研究で「がん特異的死亡」を主エンドポイントとするとき、他原因による死亡が起きると「がんで死亡できなくなる」——これが競合リスクだ。通常のKMやCoxでは他原因死を単純に打ち切りとして扱うが、これは生存率を過大推定してしまう。
学
学生 交通事故で亡くなった患者を「打ち切り」として扱うのは確かに不自然な感じがします。
師
先生 そう。Fine-Gray モデルは「サブ分布ハザード」という概念でこの問題を解決する。他原因死亡例も分母に残したまま推定するため、累積発生率が過大にならない。高齢患者を対象とした研究では特に重要だ。 
R# ── 競合リスク解析（cmprsk パッケージ）──
library(cmprsk)

# event: 1=目的イベント（がん死）, 2=競合イベント（他原因死）, 0=打ち切り
cif <- cuminc(
  ftime   = patients$os_months,
  fstatus = patients$event_type,    # 0/1/2
  group   = patients$treatment
)

# 累積発生率の可視化
ggcompetingrisks(
  cif,
  gnames   = c("治療群 がん死", "治療群 他原因死",
               "対照群 がん死", "対照群 他原因死"),
  palette  = c("#2a6b6e", "#a0d0d0", "#8b3a2a", "#d0a090"),
  ggtheme  = theme_classic()
)

# ── Fine-Gray モデル ──
fg_model <- crr(
  ftime   = patients$os_months,
  fstatus = patients$event_type,
  cov1    = model.matrix(~ treatment + age + sex, patients)[, -1],
  failcode = 1     # 目的イベントのコード
)
summary(fg_model)

# ── tidycmprsk（モダンな実装）──
library(tidycmprsk)
cr_fit <- crr(
  Surv(os_months, factor(event_type)) ~ treatment + age + sex,
  data = patients
)
tbl_regression(cr_fit, exponentiate = TRUE)
