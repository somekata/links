CHAPTER 3 ── 第3章

可視化という次元削減
〜ggplot2の文法と医学グラフの定石〜

多次元の数値を人間が認識できる2次元の図に変換する——それが可視化の本質。ggplot2の文法を習得し、論文品質のグラフを作る。

📌 この章の学習目標

  • ggplot2の「レイヤー文法（Grammar of Graphics）」の概念を説明できる
  • aes() でデータ変数をグラフ属性にマッピングできる
  • 目的に応じた geom_* を選択・組み合わせられる
  • scale_* と theme_* でグラフを論文品質に磨ける
  • facet_wrap / facet_grid で小多重図を作れる
  • Forest plot・Volcano plot・生存曲線など医学定石グラフを作れる

3.1グラフの文法（GoG） ── レイヤーを積む哲学

師
先生 ggplot2 は Leland Wilkinson の「グラフィックスの文法（Grammar of Graphics）」という理論に基づいている。グラフを「データ・美的マッピング・図形・統計変換・スケール・座標・ファセット・テーマ」という8つのレイヤーで記述する。

🎨 ggplot2 の8層構造

すべてのグラフは以下の順にレイヤーを重ねることで構築される。最初の2層（データとaes）は必須、残りは任意。+ 演算子でレイヤーを足していくのがggplot2の文法だ。

1data描画するデータフレーム（必須）
2aes()変数 → 視覚属性（x, y, color, size, shape など）のマッピング（必須）
3geom_*()点・線・棒・箱ひげ…実際に描く図形の種類
4stat_*()データの統計変換（平均・回帰・密度推定など）
5scale_*()軸範囲・色・サイズ・ラベルの変換ルール
6coord_*()座標系（デカルト・極座標・対数軸など）
7facet_*()グループ別に小パネルを並べる小多重図
8theme_*()背景・フォント・グリッド線などの非データ要素
Rlibrary(tidyverse)

# 最小構成：データ + aes + geom だけでグラフが描ける
ggplot(data = patients,          # 層1: data
       aes(x = bmi, y = hba1c)) + # 層2: aes
  geom_point()                    # 層3: geom

# パイプで書くことも可能
patients |>
  ggplot(aes(x = bmi, y = hba1c)) +
  geom_point()
学
学生 データ整形には |>（パイプ）を使って、グラフのレイヤーを重ねるには +（プラス）を使うんですね。混ざりそう……。
師
先生 混乱しやすい点だね。|> は「データを次の関数に渡す」もの。+ は「ggplot オブジェクトにレイヤーを追加する」もの。ggplot() の出力はオブジェクトとして存在し、+ で拡張されていく——この違いを頭に入れておこう。

3.2aes マッピング ── データを視覚属性に変換する

師
先生 aes()（aesthetics）はデータの変数を「x軸・y軸・色・形・サイズ・透明度」といった視覚的属性に対応させる関数だ。これがggplot2の核心といっても過言ではない。 
R# よく使う aes 属性
patients |>
  ggplot(aes(
    x     = bmi,          # x 軸（連続値）
    y     = hba1c,        # y 軸
    color = sex,          # 点の色（離散 → 自動でパレット）
    size  = age,          # 点の大きさ（連続値）
    shape = on_insulin    # 点の形（論理値）
  )) +
  geom_point(alpha = 0.7)  # alpha は aes の外＝全点に一律適用

# ── aes の中 vs 外 ──
# aes の中：データの値に応じて変化する（凡例が生成される）
geom_point(aes(color = sex))         # M/F で色分け

# aes の外：全要素に固定値を適用する（凡例は生成されない）
geom_point(color = "#2a6b6e")         # 全点をティール色に

# ── グローバル vs ローカル aes ──
# ggplot() 内の aes → 全 geom に継承
# geom_*() 内の aes → そのレイヤーのみに適用
patients |>
  ggplot(aes(x = bmi, y = hba1c)) +      # グローバル
  geom_point(aes(color = sex)) +           # ローカル（この geom のみ）
  geom_smooth(method = "lm", se = TRUE)   # x,y を継承、color は継承しない
よくあるミス： color = "blue" を aes() の中に書いてしまうと、"blue" という文字列を持つ変数として解釈されて意図しない動作になる。固定色は必ず aes() の外に書くこと。

3.3geom_ 図形の種類 ── 目的に合わせて選ぶ

⚫

必須: x, y
📦

必須: x, y（連続）
🎻
geom_violin()
バイオリン図。分布の形状を詳細に表示
必須: x, y
📊
geom_col() / geom_bar()
棒グラフ。カテゴリ別の集計値
必須: x, y（col）
📈
geom_line()
折れ線グラフ。時系列・変化量
必須: x, y, group
🌊

密度曲線。ヒストグラムの滑らかな代替

🟩

必須: x, y, fill
〰️

R# ── 箱ひげ図 + ジッタープロット（医学論文定番）──
patients |>
  ggplot(aes(x = sex, y = hba1c, fill = sex)) +
  geom_violin(alpha = 0.3, color = NA) +              # バイオリン
  geom_boxplot(width = 0.25, outlier.shape = NA,      # 箱ひげ（外れ値は jitter に任せる）
               fill = "white", color = "#1c1710") +
  geom_jitter(width = 0.1, size = 2, alpha = 0.7,      # 個別データ点
              aes(color = sex)) +
  scale_fill_manual(values  = c(M = "#2a6b6e", F = "#c8922a")) +
  scale_color_manual(values = c(M = "#2a6b6e", F = "#c8922a")) +
  labs(title = "性別別 HbA1c 分布", x = "性別", y = "HbA1c (%)") +
  theme_classic() +
  theme(legend.position = "none")

# ── 散布図 + 回帰直線 + 信頼区間 ──
patients |>
  ggplot(aes(x = bmi, y = hba1c)) +
  geom_point(aes(color = sex), size = 3, alpha = 0.8) +
  geom_smooth(method = "lm", se = TRUE,              # 線形回帰＋95%CI
              color = "#8b3a2a", fill = "#f0e6df") +
  geom_text(aes(label = id), nudge_y = 0.15,          # 患者IDをラベル
            size = 2.5, color = "#7a6e5f") +
  labs(title = "BMI と HbA1c の関連",
       x = "BMI (kg/m²)", y = "HbA1c (%)") +
  theme_classic()

3.4stat_ 統計変換 ── データを集計してから描く

師
先生 geom_bar() や geom_histogram() は内部で自動的にデータを集計している。この統計変換を明示的に制御するのが stat_* だ。また geom_* にも stat 引数があり、どの集計を使うかを指定できる。 
R# ── ヒストグラム（stat_bin が内部で使われる）──
patients |>
  ggplot(aes(x = hba1c)) +
  geom_histogram(binwidth = 0.5, fill = "#2a6b6e",
                 color = "white", alpha = 0.8) +
  geom_vline(xintercept = 6.5, linetype = "dashed",
             color = "#c8922a", linewidth = 0.8) +    # HbA1c目標値
  labs(title = "HbA1c 分布", x = "HbA1c (%)") +
  theme_classic()

# ── stat_summary：平均 ± SD を表示 ──
patients |>
  ggplot(aes(x = sex, y = hba1c, color = sex)) +
  geom_jitter(width = 0.15, alpha = 0.5) +
  stat_summary(fun = mean, geom = "point",
               size = 4, shape = 18) +                # 平均を大きなひし形で
  stat_summary(fun.data = mean_sd, geom = "errorbar",
               width = 0.15, linewidth = 0.8) +        # ±SD のエラーバー
  labs(y = "HbA1c (%)") + theme_classic()

# ── 密度曲線（複数グループを重ねる）──
patients |>
  ggplot(aes(x = hba1c, fill = sex, color = sex)) +
  geom_density(alpha = 0.3, linewidth = 0.8) +
  scale_fill_manual(values  = c(M = "#2a6b6e", F = "#c8922a")) +
  scale_color_manual(values = c(M = "#2a6b6e", F = "#c8922a")) +
  theme_classic()

3.6facet_ 小多重図 ── グループを並べて比較する

師
先生 小多重図（small multiples）は同じ構造のグラフをカテゴリ別に並べる強力な比較手法だ。1枚のグラフに色分けで詰め込むより、読みやすく誤解も少ない。 
R# ── facet_wrap：1変数で分割 ──
patients |>
  ggplot(aes(x = bmi, y = hba1c)) +
  geom_point() +
  geom_smooth(method = "lm", se = FALSE) +
  facet_wrap(~ sex, ncol = 2)        # M / F で2パネル

# ── facet_grid：行×列で分割 ──
patients |>
  ggplot(aes(x = hba1c)) +
  geom_histogram(binwidth = 0.5) +
  facet_grid(
    rows = vars(sex),
    cols = vars(bmi_cat)     # 行=性別、列=BMIカテゴリ
  )

# ── 軸スケールをパネルごとに自由にする ──
facet_wrap(~ disease_stage, scales = "free_y")  # y軸だけ自由
facet_wrap(~ gene_set,      scales = "free")     # 両軸自由

# ── ラベルを日本語にする ──
sex_labels <- c(M = "男性", F = "女性")
facet_wrap(~ sex, labeller = as_labeller(sex_labels))
facet vs color の使い分け： グループが2〜3の場合は色分け（color/fill）で1枚に収めてもよい。4グループ以上、またはグループ内の構造を詳しく見たい場合は facet が有効。論文の Figure では誌面幅の制約から facet_wrap(ncol = 2) が頻用される。

3.7theme_ デザインの磨き方 ── 論文品質へ

師
先生 theme_* は背景・グリッド線・フォント・凡例位置など「データを表現しない装飾」を制御する。まず組み込みテーマで大枠を決め、theme() で細部を上書きする、というのが定石の流れだ。 
R# ── 組み込みテーマ ──
theme_classic()   # 論文に最も使われる。グリッドなし・外枠あり
theme_bw()        # 白地 + 黒枠 + グリッドあり
theme_minimal()   # 最小限。グリッド薄め・枠なし
theme_void()      # 完全に空白（マップ・図解向け）

# ── theme() で細部を上書き ──
my_theme <- theme_classic(base_size = 12,
                            base_family = "sans") +
  theme(
    # タイトル
    plot.title     = element_text(size = 13, face = "bold", hjust = 0),
    plot.subtitle  = element_text(size = 10, color = "gray50", hjust = 0),

    # 軸
    axis.title     = element_text(size = 11),
    axis.text      = element_text(size = 10),
    axis.line      = element_line(linewidth = 0.5),

    # 凡例
    legend.position  = "bottom",
    legend.key.size  = unit(0.4, "cm"),
    legend.text      = element_text(size = 9),

    # ファセットラベル
    strip.background = element_rect(fill = "#f5f1e8", color = "#d4c9b0"),
    strip.text       = element_text(size = 10, face = "bold"),

    # マージン
    plot.margin = margin(12, 12, 12, 12)
  )

# 共通テーマを先に設定しておく方法
theme_set(my_theme)   # 以降の全グラフに自動適用

# ── 保存 ──
ggsave("fig1_hba1c.pdf", width = 7, height = 5,
       device = cairo_pdf)   # 日本語フォント対応
ggsave("fig1_hba1c.png", width = 7, height = 5,
       dpi = 300)           # 300 dpi で論文品質

3.8医学論文グラフの定石 ── Forest plot と Volcano plot

師
先生 医学論文には特有のグラフ形式がある。メタアナリシスの森林図（Forest plot）とRNASeqの火山図（Volcano plot）は特に重要だ。第1章で予告したVolcano plotをggplot2で丁寧に作ってみよう。
Forest plot（森林図）
R# Forest plot：各研究のOR/RR と 95%CI を表示
meta_df <- tibble(
  study  = c("Smith 2018", "Jones 2020", "Lee 2021",
             "Wang 2022", "Pooled"),
  or     = c(1.42, 0.88, 1.15, 1.31, 1.18),
  lower  = c(0.95, 0.61, 0.82, 0.97, 1.02),
  upper  = c(2.11, 1.28, 1.61, 1.76, 1.37),
  weight = c(18, 24, 21, 20, NA)   # Pooled はダイヤモンド
)

meta_df |>
  mutate(study = fct_rev(factor(study))) |>    # 上から順に表示
  ggplot(aes(y = study, x = or)) +
  geom_vline(xintercept = 1, linetype = "dashed",
             color = "gray50", linewidth = 0.6) +      # HR=1 の参照線
  geom_errorbarh(aes(xmin = lower, xmax = upper,
                      linewidth = if_else(study == "Pooled", 1.2, 0.6)),
                 height = 0.25) +
  geom_point(aes(
    size  = if_else(!is.na(weight), weight, 8),     # 重みを点サイズに
    shape = if_else(study == "Pooled", 18L, 15L),    # Pooled はひし形
    color = if_else(study == "Pooled", "#8b3a2a", "#1c1710")
  )) +
  scale_x_continuous(trans = "log",
                     breaks = c(0.5, 1, 1.5, 2, 3)) +  # 対数軸
  scale_size(range = c(2, 6)) +
  labs(title = "Forest Plot: Drug X vs Placebo",
       x = "Odds Ratio (95% CI)", y = NULL) +
  theme_classic() +
  theme(legend.position = "none")
Volcano plot（火山図）
R# DESeq2 の結果データフレーム res_df が存在する前提
volcano_df <- res_df |>
  filter(!is.na(padj)) |>
  mutate(
    sig = case_when(
      padj < 0.05 & log2FoldChange >  1 ~ "Up",
      padj < 0.05 & log2FoldChange < -1 ~ "Down",
      TRUE                               ~ "NS"
    ),
    label = if_else(padj < 0.001 & abs(log2FoldChange) > 2,
                    gene, NA_character_)   # 上位遺伝子にラベル
  )

ggplot(volcano_df, aes(x = log2FoldChange, y = -log10(pvalue))) +
  geom_point(aes(color = sig), alpha = 0.5, size = 0.9) +
  geom_vline(xintercept = c(-1, 1),
             linetype = "dashed", color = "gray60") +
  geom_hline(yintercept = -log10(0.05),
             linetype = "dashed", color = "gray60") +
  ggrepel::geom_text_repel(            # ggrepel でラベル重複を回避
    aes(label = label),
    size = 2.5, max.overlaps = 20,
    color = "#1c1710"
  ) +
  scale_color_manual(
    values = c(Up = "#c83a2a", Down = "#2a6b8a", NS = "#c0b8a8"),
    labels = c(glue::glue("Up ({sum(volcano_df$sig=='Up')})"),
               glue::glue("Down ({sum(volcano_df$sig=='Down')})"),
               "NS")
  ) +
  labs(title = "Volcano Plot: Tumor vs Normal",
       x = "log2 Fold Change", y = "-log10(p-value)",
       color = "DEG status") +
  theme_classic()
ggrepel パッケージ： geom_text_repel() / geom_label_repel() はラベルが重複しないよう自動的に位置を調整する。Volcano plot・Forest plot などラベルが密集するグラフでは必須に近い。

3.9生存曲線（survminer） ── 時間とイベントを可視化する

師
先生 Kaplan-Meier 曲線は生存解析の顔。survminer パッケージを使えば、ggplot2ベースの美しい生存曲線をほぼ自動で描ける。p値・at-risk テーブル・信頼区間も1関数で追加できる。 
Rlibrary(survival)
library(survminer)

# KM推定
fit <- survfit(Surv(time, status) ~ treatment, data = trial_df)

# 美しい生存曲線
ggsurvplot(
  fit,
  data          = trial_df,
  pval          = TRUE,               # log-rank p値を自動表示
  conf.int      = TRUE,               # 95%CI のリボン
  risk.table    = TRUE,               # At-risk テーブルを下部に追加
  risk.table.col= "strata",
  linetype      = "strata",
  palette       = c("#2a6b6e", "#c8922a"),
  ggtheme       = theme_classic(),
  xlab          = "Time (months)",
  ylab          = "Overall Survival",
  legend.labs   = c("Drug A", "Placebo"),
  surv.median.line = "hv"            # 中央生存期間の参照線
)

# 累積発症率（1-KM）で表示する場合
ggsurvplot(fit, data = trial_df,
           fun = "event",             # 1-survival = cumulative incidence
           ylab = "Cumulative Incidence")

3.10インタラクティブ化（plotly） ── 探索的解析のために

師
先生 論文投稿には静的なPDF・PNG形式が必要だが、自分の探索的解析やR Markdownレポートではインタラクティブグラフが便利だ。plotly パッケージの ggplotly() は、作成済みのggplot2オブジェクトを1行でインタラクティブ化できる。 
Rlibrary(plotly)

# 通常通り ggplot を作成
p <- patients |>
  ggplot(aes(x = bmi, y = hba1c,
             color = sex,
