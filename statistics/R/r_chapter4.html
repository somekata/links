CHAPTER 4 ── 第4章

基本統計量の本質
〜平均・分散・分布による情報の濃縮〜

グラフで「見えていた」ものを数字で語る。平均・中央値・分散・標準偏差……それぞれが何を意味し、どう使い分けるのかを徹底的に掘り下げる。

📌 この章の学習目標

  • 平均・中央値・最頻値の違いと使い分けを説明できる
  • 分散・標準偏差・IQR の計算原理を理解し、Rで算出できる
  • 分布の形（正規・右裾・左裾・二峰性）を見て判断できる
  • 外れ値が統計量に与える影響を把握し、頑健な統計量を選べる
  • group_by + summarise で層別集計ができる
  • 正規分布・標準正規分布・68-95-99.7 則を説明できる
  • 信頼区間の概念を「繰り返し実験の文脈」で説明できる

4.1統計量とは何か ── データを1つの数字に圧縮する

師
先生 第3章では「可視化は次元削減だ」と言ったね。統計量はその究極形だ。100人のHbA1cデータを1枚のグラフに圧縮するのが可視化なら、統計量はそれをさらに1つの数字に圧縮する。平均値は「100人の情報を1個の数字で代表する」という大胆な試みだ。
学
学生 でも1つの数字だと、情報がたくさん失われませんか？
師
先生 そう、失われる。だから複数の統計量を組み合わせるんだ。「中心はどこか（位置の統計量）」「どれだけ散らばっているか（散らばりの統計量）」「分布の形はどうか（歪度・尖度）」——この3つを揃えてはじめてデータの全体像が見えてくる。

📐 統計量の3つの役割

位置（Location）：データの中心はどこにあるか。平均・中央値・最頻値。

散らばり（Spread）：データはどれほど散らばっているか。分散・標準偏差・IQR・範囲。

形（Shape）：分布はどんな形か。歪度・尖度、あるいはヒストグラムで直接確認。

4.2位置の統計量 ── 中心をどう定義するか

師
先生 「中心」の定義には複数の流儀がある。それぞれが「中心」の異なる側面を捉えている。どれが正しいのではなく、データの性質と問いに応じて選ぶんだ。
mean()
平均値
Σxᵢ / n
全データの合計を個数で割る。外れ値の影響を受けやすい。
median()
中央値
順位の真ん中の値
データを並べたときの真ん中。外れ値に強い頑健な統計量。
table() / mode
最頻値
最も頻繁に現れる値
カテゴリ変数での代表値。連続変数ではあまり使わない。
R# patients データで位置の統計量を計算
patients$hba1c |> mean(na.rm = TRUE)    # 平均値
patients$hba1c |> median(na.rm = TRUE)  # 中央値

# summarise で一括計算
patients |>
  summarise(
    n      = n(),
    mean   = mean(hba1c, na.rm = TRUE),
    median = median(hba1c, na.rm = TRUE),
    min    = min(hba1c, na.rm = TRUE),
    max    = max(hba1c, na.rm = TRUE)
  )

# summary() でまとめて確認
summary(patients$hba1c)
#   Min. 1st Qu.  Median    Mean 3rd Qu.    Max.
#   4.80    6.20    7.30    7.42    8.50   12.80
学
学生 平均が7.42で中央値が7.30……少し違いますね。これは何を意味しているんですか？
師
先生 平均 > 中央値ということは、右に裾が長い分布——つまり高HbA1cの患者が少数いて、平均を引き上げているサインだ。HbA1cは血糖コントロールが悪い患者が12や13を超えることがあるが、下限は理論上4前後で打ち止めになる。こういう一方向に歪んだ分布では、中央値のほうが「典型的な患者」を代表しやすい。
医学データの傾向： 検査値（AST・クレアチニン・CRP）や入院期間・費用など、医学データの多くは右裾が長い分布を示す。このようなデータでは平均より中央値を報告するほうが適切なことが多い。論文の Table 1 で「中央値（IQR）」と書かれているのはそのためだ。

4.3散らばりの統計量 ── データのばらつきを測る

師
先生 2つの病院のHbA1c平均がどちらも7.5だったとして、一方は全員7.4〜7.6の範囲に密集し、もう一方は4.5から12.5まで散らばっているとしたら——平均だけ見ていたら気づけない。「散らばり」を測る統計量が必要だ。
分散（Variance） s² = Σ(xᵢ − x̄)² / (n − 1) 各データと平均の差を二乗して平均したもの（標本分散はn-1で割る）
標準偏差（Standard Deviation） s = √s² 分散の平方根。元のデータと同じ単位になるので解釈しやすい 
R# 散らばりの主要統計量
patients |>
  summarise(
    sd      = sd(hba1c, na.rm = TRUE),       # 標準偏差
    var     = var(hba1c, na.rm = TRUE),      # 分散
    iqr     = IQR(hba1c, na.rm = TRUE),      # 四分位範囲（Q3 - Q1）
    range   = diff(range(hba1c, na.rm = TRUE)), # 範囲（max - min）
    cv      = sd(hba1c, na.rm = TRUE) /
              mean(hba1c, na.rm = TRUE) * 100  # 変動係数（%）
  )

# ── 四分位数（Quantiles）──
quantile(patients$hba1c, probs = c(0.25, 0.5, 0.75), na.rm = TRUE)
#  25%  50%  75%
# 6.20 7.30 8.50

# 任意のパーセンタイル（例：5th / 95th）
quantile(patients$hba1c, probs = c(0.05, 0.95), na.rm = TRUE)
学
学生 分散の式で「n-1」で割るのはなぜですか？nで割れば普通の平均なのに。
師
先生 これは「自由度」の問題だ。手元のデータ（標本）は母集団の一部に過ぎない。n個のデータで平均を1つ計算すると、残り「n-1個」のデータの値が決まれば最後の1個は自動的に決まってしまう——つまり「独立に動ける個数」はn-1なんだ。n-1で割ることで、母集団の分散を偏りなく推定できる（不偏推定）。これをベッセルの補正という。
論文での書き方： 正規分布に従うデータは「平均 ± SD」、非正規分布のデータは「中央値（IQR）」で報告する。IQRは第1四分位数〜第3四分位数の範囲（Q1–Q3）として記載するのが一般的だ。

4.4分布の形を見る ── ヒストグラムと密度曲線

師
先生 位置と散らばりを計算したら、必ず分布の形も確認しろ、というのが鉄則だ。同じ平均・標準偏差でも、形がまったく異なることがある。
正規分布（symmetric）
右裾分布（right-skewed）
R# ── ヒストグラム + 密度曲線で分布の形を確認 ──
patients |>
  ggplot(aes(x = hba1c)) +

             color = "#2a6b6e", linetype = "dashed", linewidth = 0.8) +
  annotate("text", x = 9, y = 0.25,
           label = "赤：平均\n青：中央値", size = 3, color = "#4a3f30") +

4.5外れ値と頑健統計量 ── 異常値に騙されない

師
先生 患者データには外れ値がつきものだ。入力ミスかもしれないし、本当に極端な患者かもしれない。外れ値の存在を知らずに平均・標準偏差だけで議論するのは危険だ。
統計量外れ値への感受性代替の頑健統計量Rの関数
平均 高い 中央値 / トリム平均 median() / mean(x, trim=0.1)
標準偏差 高い IQR / MAD IQR() / mad()
分散 非常に高い IQR²（近似） IQR()^2 / 1.35
中央値 低い —（頑健） median()
IQR 低い —（頑健） IQR() 
R# ── 外れ値の検出 ──
# 方法1：Tukey フェンス（IQRの1.5倍）
q1 <- quantile(patients$hba1c, 0.25, na.rm = TRUE)
q3 <- quantile(patients$hba1c, 0.75, na.rm = TRUE)
iqr_val <- q3 - q1

lower_fence <- q1 - 1.5 * iqr_val
upper_fence <- q3 + 1.5 * iqr_val

patients |>
  filter(hba1c < lower_fence | hba1c > upper_fence) |>
  select(id, hba1c)

# ── 方法2：Zスコアによる検出 ──
patients |>
  mutate(
    z_score = (hba1c - mean(hba1c, na.rm = TRUE)) /
               sd(hba1c, na.rm = TRUE)
  ) |>
  filter(abs(z_score) > 3) |>            # |Z| > 3 を外れ値とみなす
  select(id, hba1c, z_score)

# ── MAD（中央絶対偏差）：頑健な散らばりの指標 ──
mad(patients$hba1c, na.rm = TRUE)
# デフォルトで定数1.4826をかけてSDと比較可能なスケールにしてくれる
外れ値の扱い方の原則： 外れ値を発見したら、まず「入力ミスか・真の値か」を確認する。削除する場合は理由を明記する。感度分析として「外れ値あり」「外れ値なし」の両方の結果を報告するのが誠実だ。削除の判断を統計的アルゴリズムだけに任せてはならない。

4.6層別集計と group_by ── 「グループ別に見る」基本技

師
先生 全体の統計量を出したら、次は「性別別」「治療群別」「年齢層別」に層別して比較するのが医学研究の常道だ。tidyverse の group_by() + summarise() はそのための道具だ。 
R# ── 性別別の基本統計量 ──
patients |>
  group_by(sex) |>
  summarise(
    n      = n(),
    mean   = round(mean(hba1c, na.rm = TRUE), 2),
    sd     = round(sd(hba1c, na.rm = TRUE), 2),
    median = round(median(hba1c, na.rm = TRUE), 2),
    q1     = quantile(hba1c, 0.25, na.rm = TRUE),
    q3     = quantile(hba1c, 0.75, na.rm = TRUE)
  )

# ── 複数変数をまとめて集計（across）──
patients |>
  group_by(sex) |>
  summarise(
    across(
      c(hba1c, bmi, age),
      list(mean = \(x) round(mean(x, na.rm = TRUE), 1),
           sd   = \(x) round(sd(x, na.rm = TRUE), 1)),
      .names = "{.col}_{.fn}"
    )
  )

# ── 論文の Table 1 スタイルで出力（gtsummary パッケージ）──
library(gtsummary)

patients |>
  select(sex, age, bmi, hba1c, on_insulin) |>
  tbl_summary(
    by = sex,
    statistic = list(
      all_continuous()  ~ "{mean} ({sd})",
      all_categorical() ~ "{n} ({p}%)"
    ),
    digits = all_continuous() ~ 1
  ) |>
  add_p() |>
  add_overall()
gtsummary の威力： tbl_summary() 1行で論文品質の Table 1 が出力できる。add_p() で自動的にt検定・カイ二乗検定・ノンパラメトリック検定が選択されてp値も付く。修士論文・学会発表のスライドに直接貼れる水準の表が生成される。

4.7正規分布の意味 ── なぜこんなに重要なのか

師
先生 統計学でもっとも重要な分布は何かと聞かれたら、迷わず正規分布と答える。なぜかというと——現実の多くの現象がこの形に近づくからだ。そして「平均と標準偏差の2つだけで完全に定義できる」という数学的な美しさがある。

🔔 正規分布の3つの性質

68-95-99.7 則： 平均 ± 1SD の範囲に約68%、± 2SD に約95%、± 3SD に約99.7% のデータが含まれる。

加法性： 正規分布に従う独立した確率変数の和も正規分布に従う。これが多くの統計検定の理論的基盤になる。

中心極限定理： 元の分布が何であれ、標本平均は標本サイズが大きくなるにつれて正規分布に近づく——これがt検定などが広く使える理由だ。

R# ── 正規分布の確率計算 ──
# HbA1c ~ N(7.42, 1.38²) と仮定
m <- 7.42; s <- 1.38

# HbA1c ≥ 10 の確率（コントロール不良の割合）
pnorm(10, mean = m, sd = s, lower.tail = FALSE)
# [1] 0.02383   → 約2.4%

# 上位5%に入るHbA1c値（95パーセンタイル）
qnorm(0.95, mean = m, sd = s)
# [1] 9.690   → 9.7% 以上が上位5%

# ── 正規性の検定（シャピロ・ウィルク検定）──
shapiro.test(patients$hba1c)
# Shapiro-Wilk normality test
# W = 0.9742, p-value = 0.0842
# p > 0.05 → 正規分布から有意に外れているとは言えない

# ── Q-Qプロット（正規性の視覚確認）──
patients |>
  ggplot(aes(sample = hba1c)) +
  stat_qq(color = "#2a6b6e", alpha = 0.7) +
  stat_qq_line(color = "#8b3a2a", linewidth = 0.8) +
  labs(title = "Q-Qプロット（HbA1c）",
       x = "理論分位数", y = "標本分位数") +
  theme_classic()
学
学生 シャピロ検定でp > 0.05なら「正規分布でよい」ということですか？
師
先生 注意が必要だ。「正規分布から有意に外れているとは言えない」というだけで、「正規分布だと証明された」ではない。さらにサンプルサイズが大きくなると、わずかな歪みでも検定が有意になってしまう。だからQ-Qプロットを目視確認することとセットで使うのが正しいやり方だ。

4.8信頼区間の考え方 ── 「95%CI」の本当の意味

師
先生 論文を読むと必ず出てくる「95%信頼区間（95%CI）」。これを「真の値が95%の確率でこの範囲に入る」と解釈している人が多いが、それは誤りだ。
学
学生 え、違うんですか？　ではどういう意味ですか？
師
先生 正確にはこうだ——「同じ方法で100回研究を繰り返したとき、そのうち約95回は構築された信頼区間が真の値を含む」。真の値は固定されていて動かない。動くのは標本と、そこから計算される信頼区間のほうだ。手元の信頼区間が真の値を含んでいるかどうかは、実はわからない。

🎯 信頼区間の正しいイメージ

「的（真の値）は動かない。矢（信頼区間）を100本射たとき、95本が的に当たるように作られたルールで矢を作っている」——これが頻度論的な95%CIの解釈だ。

実際の研究で得られた「7.2–7.8」という区間が当たり（真の値を含む）か外れかは、その1回の結果だけからはわからない。だが研究者たちはこのルールに従った区間を信頼して議論を積み上げる。

R# ── 平均の95%信頼区間（t分布を使用）──
t.test(patients$hba1c)$conf.int
# [1] 7.144 7.696
# attr(,"conf.level")
# [1] 0.95

# ── 手動計算（原理を理解するために）──
n    <- sum(!is.na(patients$hba1c))
m    <- mean(patients$hba1c, na.rm = TRUE)
s    <- sd(patients$hba1c, na.rm = TRUE)
se   <- s / sqrt(n)             # 標準誤差
t_cv <- qt(0.975, df = n - 1)  # t分布の97.5パーセンタイル

c(lower = m - t_cv * se,
  upper = m + t_cv * se)

# ── 層別CIをグラフに（pointrange）──
patients |>
  group_by(sex) |>
  summarise(
    mean = mean(hba1c, na.rm = TRUE),
    se   = sd(hba1c, na.rm = TRUE) / sqrt(n())
  ) |>
  ggplot(aes(x = sex, y = mean, color = sex)) +
  geom_pointrange(aes(ymin = mean - 1.96 * se,
                       ymax = mean + 1.96 * se),
                  size = 0.8) +
  labs(title = "性別別 HbA1c（平均 ± 95%CI）",
       y = "HbA1c (%)", x = NULL) +
  scale_color_manual(values = c(M = "#2a6b6e", F = "#c8922a")) +
  theme_classic() + theme(legend.position = "none")
標準偏差 vs 標準誤差： SD（標準偏差）はデータの「散らばり」を表し、患者個人のばらつきを記述する。SE（標準誤差）は標本平均の「不確かさ」を表し、推定の精度を表す。SE = SD / √n なので、n が大きいほど SE は小さくなる。論文で「平均 ± SE」と書く場合は、個々のばらつきではなく推定精度を強調したいときだ。

🧠 問題2：層別集計 + gtsummary

patients データで、インスリン使用群（on_insulin = TRUE）と非使用群を比較する Table 1 を gtsummary で作成せよ。含める変数は age, bmi, hba1c, sex（カテゴリ）とし、p値も表示すること。

Rlibrary(gtsummary)

patients |>
  filter(!is.na(on_insulin)) |>
  select(on_insulin, age, bmi, hba1c, sex) |>
  tbl_summary(

🧠 問題4（発展）：Bootstrap 信頼区間

HbA1c の中央値について、t分布を使わずに Bootstrap 法で 95%CI を計算せよ（boot パッケージを使用。B = 1000回）。

Rlibrary(boot)

# Bootstrap のための関数（statisticには (data, indices) が必要）
median_fn <- function(data, indices) {
  median(data[indices], na.rm = TRUE)
}

set.seed(42)
boot_result <- boot(
  data      = patients$hba1c[!is.na(patients$hba1c)],
  statistic = median_fn,
  R         = 1000
)

boot.ci(boot_result, type = "perc")
# Percentile CI:
# ( 7.1,  7.5 )  95%

# Bootstrap 分布の可視化
tibble(median_boot = boot_result$t[,1]) |>
  ggplot(aes(x = median_boot)) +
  geom_histogram(bins = 40, fill = "#5a3a8a", alpha = 0.7) +
  geom_vline(xintercept = boot.ci(boot_result, type = "perc")$percent[4:5],
