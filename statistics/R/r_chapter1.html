1.2インデックスの心地よさ ── 1から始まる世界

27.2
36.1

ハイライト（赤）= hba1c[4]。Pythonでは [3] だが、Rでは直感的に [4]。

R# 単一要素

hba1c[4]              # → 8.5

# 範囲指定
hba1c[2:4]            # → 7.2 6.1 8.5

# 複数指定
hba1c[c(1, 3, 5)]     # → 5.8 6.1 6.9

# 除外（マイナスインデックス）※Pythonとは逆の意味
hba1c[-4]             # → 5.8 7.2 6.1 6.9（4番目を除く）

# 条件でインデックス（論理インデックス）
hba1c[hba1c > 7.0]   # → 7.2 8.5

# 末尾の要素
hba1c[length(hba1c)] # → 6.9

1.4データ型の種類 ── 彫刻の素材を知る

numeric
実数（連続値）
c(5.8, 7.2, 6.1)
HbA1c・BMI・体重など。Rの数値のデフォルト型。
integer
整数
c(1L, 2L, 3L)
年齢・回数など。末尾Lで明示。メモリ効率が良い。
character
文字列
c("男性", "女性")
患者名・性別など。必ず引用符で囲む。
logical
論理値
c(TRUE, FALSE)
条件判定の結果。TRUE=1、FALSE=0として計算可。
factor
因子型（カテゴリ）
factor(c("I","II","III"))
病期・群など順序あるカテゴリ。統計解析で重要。
NA / NULL
欠損値 / 空
c(5.8, NA, 6.1)
NA=欠損（どの型にも入る）。NULL=存在しない。
R# 型の確認

class(5L)           # → "integer"
class("HbA1c")     # → "character"
class(TRUE)         # → "logical"

# 型の変換
as.integer(5.8)     # → 5   （小数点以下切り捨て）
as.numeric(TRUE)   # → 1
as.character(7.2)  # → "7.2"

# factor の作成
stage <- factor(c("I", "II", "I", "III"),

1.6tidyverse入門 ── データをパイプでつなぐ

師
先生 tidyverseはRのデータ分析を劇的に変えた革命的なパッケージ群だ。核心にあるのは「パイプ演算子」という概念だ。データを左から右へ水が流れるように処理していく。 
Rlibrary(tidyverse)

# 患者データフレームを作成（詳細は第2章で）
patients <- tibble(
  id        = 1:8,
  sex       = c("M","F","M","F","M","F","M","F"),
  age       = c(52,67,45,71,58,63,49,76),
  hba1c     = c(5.8,7.2,6.1,8.5,6.9,7.8,5.5,9.1),
  bmi       = c(24.3,27.8,22.1,30.5,25.6,28.9,23.4,31.2),
  on_insulin = c(F,T,F,T,F,T,F,T)
)

# ── パイプ演算子 |> でデータを流す ──
# 旧来の書き方（ネストが深くて読みにくい）
mean(filter(patients, hba1c > 7.0)$bmi)

# パイプを使った書き方（左から右に読める）
patients |>
  filter(hba1c > 7.0) |>
  summarise(mean_bmi = mean(bmi),
            n        = n())
# # A tibble: 1 × 2
#   mean_bmi     n
#      <dbl> <int>
#  1     29.5     4

# ── dplyrの主要動詞 ──
# filter()   : 行の絞り込み
# select()   : 列の選択
# mutate()   : 新しい列の作成
# summarise(): 集計
# group_by() : グループ化
# arrange()  : 並び替え

patients |>
  mutate(dm_control = if_else(hba1c <= 7.0, "良好", "不良")) |>
  group_by(dm_control) |>
  summarise(n = n(), mean_bmi = round(mean(bmi), 1))
# # A tibble: 2 × 3
#   dm_control     n mean_bmi
#   <chr>      <int>    <dbl>
# 1 不良           4     29.5
# 2 良好           4     23.9
パイプ演算子 |> について：
R 4.1以降は組み込みパイプ |> が使える。tidyverse伝統の %>%（magrittrパッケージ）も同様の動作。新規コードでは |> の使用を推奨。RStudioでは Ctrl/Cmd + Shift + M でパイプを挿入できる。

1.8ggplot2で可視化 ── グラフは仮説の言語

師
先生 ggplot2は「グラフの文法（Grammar of Graphics）」という概念に基づいている。データ・座標系・幾何学的オブジェクト（点・線・棒）を積み重ねてグラフを構築するんだ。 
Rlibrary(tidyverse)

# ── 基本構造：ggplot() + geom_*() ──
# HbA1cの分布をヒストグラムで
ggplot(patients, aes(x = hba1c)) +
  geom_histogram(binwidth = 0.5, fill = "#8b3a2a", color = "white") +
  labs(title = "HbA1c分布",
       x = "HbA1c (%)", y = "患者数") +
  theme_minimal()

# ── 散布図：HbA1c vs BMI ──
ggplot(patients, aes(x = bmi, y = hba1c,
                      color = on_insulin, shape = sex)) +
  geom_point(size = 3) +
  geom_smooth(method = "lm", se = TRUE, color = "gray40") +
  scale_color_manual(values = c("#2a6b6e", "#c8922a"),
                     labels = c("非インスリン", "インスリン")) +
  labs(title = "BMI と HbA1c の関連",
       x = "BMI (kg/m²)", y = "HbA1c (%)",
       color = "インスリン", shape = "性別") +
  theme_classic()

# ── 箱ひげ図：性別別HbA1c ──
ggplot(patients, aes(x = sex, y = hba1c, fill = sex)) +
  geom_boxplot(alpha = 0.7) +
  geom_jitter(width = 0.1, size = 2) +   # 個別データも重ねる
  scale_fill_manual(values = c("#78b8d8", "#d87898")) +
  labs(title = "性別別 HbA1c", x = "性別", y = "HbA1c (%)") +
  theme_minimal() +
  theme(legend.position = "none")

# ── グラフの保存 ──
ggsave("hba1c_boxplot.png",
       width = 8, height = 5, dpi = 300)
ggplot2の layer 構造：
ggplot(data, aes(...)) でキャンバスを用意 → geom_*() で図形を重ねる → scale_*() で色・軸を調整 → labs() でラベル → theme_*() でデザイン。「＋」でレイヤーを積み上げていくイメージ。

1.10NGS / RNASeq解析入門 ── 高次元の彫刻

師
先生 いよいよ本章の最大の山場だ。RNASeqは「どの遺伝子がどのくらい発現しているか」を網羅的に測定する技術だ。腫瘍vs正常、治療前vs治療後……「2万次元のデータから意味ある差を彫り出す」典型例だよ。

💡 RNASeq解析の全体フロー

① 生データ（FASTQ） → FastQCで品質確認 → ② アライメント（STAR/HISAT2でゲノムにマッピング）→ ③ カウントデータ（featureCountsでgene×sampleの行列）→ ④ Rで差次的発現解析（DESeq2/edgeR）→ ⑤ 可視化・パスウェイ解析（ggplot2, clusterProfiler）

ここではRが登場するステップ③以降を扱う。

R# ── DESeq2によるRNA-Seq差次的発現解析 ──
library(DESeq2)
library(tidyverse)

# カウントマトリクスの読み込み（行=遺伝子, 列=サンプル）
counts <- read_csv("counts.csv", row.names = 1) |>
  as.matrix()
# dim(counts): 20000行（遺伝子数） × 12列（サンプル数）

# サンプル情報テーブル
coldata <- tibble(
  sample    = colnames(counts),
  condition = factor(rep(c("tumor", "normal"), each = 6))
)

# ── DESeq2オブジェクトの作成と正規化 ──
dds <- DESeqDataSetFromMatrix(
  countData = counts,
  colData   = coldata,
  design    = ~condition
)

# 低発現遺伝子のフィルタリング（解析精度向上）
dds <- dds[rowSums(counts(dds)) >= 10, ]

# 差次的発現解析の実行
dds <- DESeq(dds)

# ── 結果の取得と整形 ──
res <- results(dds, contrast = c("condition", "tumor", "normal"))
res_df <- as.data.frame(res) |>
  rownames_to_column("gene") |>
  as_tibble() |>
  arrange(padj)

# 有意に発現変動した遺伝子（DEG）
degs <- res_df |>
  filter(padj < 0.05, abs(log2FoldChange) > 1)
nrow(degs)   # 有意DEG数

# ── Volcano plot（火山図）──
res_df |>
  mutate(
    sig = case_when(
      padj < 0.05 & log2FoldChange >  1 ~ "Up",
      padj < 0.05 & log2FoldChange < -1 ~ "Down",
      TRUE                                ~ "NS"
    )
  ) |>
  ggplot(aes(x = log2FoldChange, y = -log10(pvalue),

RNASeqカウントデータは負の二項分布（Negative Binomial Distribution）に従う。DESeq2はサンプル間のライブラリサイズ差を「サイズ因子（size factor）」で補正し、遺伝子ごとの分散を全遺伝子からshrinkage推定することで、少ないサンプルでも安定した検定を実現している。調整済みp値（padj）はBenjamini-Hochberg法で多重比較補正済み。
