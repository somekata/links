2.1データフレームとは ── ベクトルを束ねた「表」

師
先生 第1章でベクトルを学んだ。患者10人のHbA1c値が1本の線として存在していた。でも現実の臨床データはどうだろう？HbA1c・BMI・年齢・性別・治療法——これらが横に揃った形でないと、患者ごとの比較ができない。これを実現するのがデータフレームだ。

📦 データフレームの本質

データフレームは「同じ長さのベクトルを列として束ねたリスト」である。各列は独立した型を持てる（数値・文字・論理など混在OK）。行は「1つの観察単位（患者・試験・サンプル）」に対応する。

Excelのシート1枚に相当するが、R上では数百万行でも高速に操作でき、再現可能なコードで処理を記録できる点が根本的に異なる。

学
学生 列ごとに型が違っていいんですね。文字列・整数・小数・論理値が混在している……。
師
先生 そう、それがデータフレームの最大の特徴。ベクトルは1種類の型しか持てないが、データフレームは列ごとに型を変えられる。だからこそ「患者ID（文字）・年齢（整数）・HbA1c（小数）・インスリン使用（論理）」という現実の構造をそのまま表現できる。

2.3列と行へのアクセス ── データを取り出す3つの方法

師
先生 データフレームから値を取り出す方法はいくつかある。目的によって使い分けよう。 
R# ── 列へのアクセス（3通り）──
patients$hba1c               # $ 演算子：ベクトルとして取り出す
patients[["hba1c"]]          # [[]] ：同上。変数名も使える
patients |> pull(hba1c)     # pull()：パイプとの相性が良い
# [1] 7.2 9.4 6.8 8.1 5.9

# ── 行へのアクセス（スライス）──
patients[2, ]              # 第2行を取り出す
patients[1:3, ]            # 第1〜3行
patients |> slice(1:3)    # tidyverse版（こちらを推奨）

# ── 行列両方の指定 ──
patients[2, "hba1c"]       # 第2行 hba1c列の値
# [1] 9.4

# ── 複数列を選ぶ ──
patients[, c("id", "hba1c")]          # 旧記法
patients |> select(id, hba1c)         # tidyverse版（後述）
【重要】$ vs [[]] の使い分け： プログラム内で列名を変数に格納して使う場合は必ず [[]] を使うこと。
例：col <- "hba1c"; df[[col]] は正しいが、df$col は「col という列名」を探してしまう。

2.4列を加工する mutate ── 新しい変数を生み出す

師
先生 mutate() は既存の列を元に新しい列を作ったり、既存の列を書き換えたりする動詞だ。Excelで言えば「新しい列に数式を入れる」操作に相当する。 
Rpatients <- patients |>
  mutate(
    # 数値の変換
    hba1c_mmol = hba1c * 10.929,      # HbA1c を mmol/mol に換算

    # 条件分岐で新しいカテゴリ変数
    bmi_cat = case_when(
      bmi <  18.5 ~ "低体重",
      bmi <  25.0 ~ "標準",
      bmi <  30.0 ~ "過体重",
      TRUE         ~ "肥満"
    ),

    # factor 化（順序を明示）
    bmi_cat = factor(bmi_cat,
                     levels = c("低体重", "標準", "過体重", "肥満")),

    # 論理値→文字列
    insulin_label = if_else(on_insulin, "使用", "非使用",
                             missing = "不明"),

    # 年齢を10歳刻みに
    age_decade = (age %/% 10) * 10L
  )

glimpse(patients)
case_when vs if_else
関数使いどころ書き方の例
if_else() 2値分岐（TRUE/FALSE）。NAの扱いを明示できる if_else(x > 0, "正", "非正", missing = "不明")
case_when() 3値以上の分岐。条件を上から評価し、最初にTRUEになった枝を使う case_when(x < 18 ~ "未満", x < 60 ~ "現役", TRUE ~ "高齢")
TRUEが「その他」： case_when() の末尾に TRUE ~ "その他" を置くと「上記いずれにも当てはまらない場合」になる。これを省略すると未分類の行がNAになるので注意。

2.5filter と select ── 行と列を絞る

師
先生 filter() は行を絞る、select() は列を絞る——この2つはデータ整形の基本動作だ。 
R# ── filter：行を絞る ──
patients |> filter(hba1c >= 8.0)               # HbA1c 8以上
patients |> filter(sex == "F", bmi > 25)         # AND条件
patients |> filter(sex == "F" | bmi > 25)         # OR条件
patients |> filter(between(age, 40, 60))           # 範囲指定
patients |> filter(id %in% c("P001", "P003"))     # リスト一致
patients |> filter(!is.na(on_insulin))            # NAを除外

# ── select：列を絞る ──
patients |> select(id, hba1c, bmi)              # 列名で指定
patients |> select(-on_insulin)                 # 列を除外（マイナス）
patients |> select(where(is.numeric))            # 数値列だけ
patients |> select(starts_with("bmi"))           # 列名の前方一致
patients |> select(ends_with("_cat"))            # 列名の後方一致
patients |> rename(patient_id = id)             # 列名を変更

# ── 組み合わせ ──
patients |>
  filter(sex == "F") |>
  select(id, age, hba1c) |>
  arrange(desc(hba1c))   # HbA1c降順で並べ替え
patients |> filter(sex == "F") |> select(id, hba1c) |> arrange(desc(hba1c)) → tibble 結果

2.6group_by で層別化 ── 「グループごと」の集計

師
先生 臨床研究では「男女別」「年代別」「治療群別」という層別比較が頻出する。group_by() + summarise() がそれを担う。 
R# ── 基本的な層別集計 ──
patients |>
  group_by(sex) |>
  summarise(
    n          = n(),
    mean_hba1c = mean(hba1c, na.rm = TRUE),
    sd_hba1c   = sd(hba1c,   na.rm = TRUE),
    med_bmi    = median(bmi, na.rm = TRUE)
  )
# # A tibble: 2 × 5
#   sex       n mean_hba1c sd_hba1c med_bmi
#   <chr> <int>      <dbl>    <dbl>   <dbl>
# 1 F         2       8.75    0.919    29.95
# 2 M         3       6.63    0.655    22.30

# ── 複数変数でグループ化 ──
patients |>
  group_by(sex, bmi_cat) |>
  summarise(n = n(), mean_hba1c = mean(hba1c), .groups = "drop")

# ── グループ内の割合を計算 ──
patients |>
  count(sex, bmi_cat) |>
  group_by(sex) |>
  mutate(pct = round(n / sum(n) * 100, 1))

# ── グループ内最大値の行を取り出す ──
patients |>
  group_by(sex) |>
  slice_max(hba1c, n = 1)   # 性別ごとにHbA1c最大の患者
.groups = "drop" とは： summarise() 後はデフォルトでグループが残る（警告が出る）。.groups = "drop" を指定するとグループを解除した通常の tibble が返る。次の操作に予期しない影響を与えたくない場合は明示的に指定しよう。

2.7縦横変換：pivot ── tidy data の思想

師
先生 データには「横長（wide）」と「縦長（long）」という2つの形がある。Excelでは横長が多いが、Rでグラフを描いたり統計モデルを当てたりするには縦長（tidy形式）が必要になる場合が多い。

🔄 Tidy Data の3原則

① 1行 = 1観測（1人・1時点）
② 1列 = 1変数（測定した1つの値）
③ 1セル = 1値

「患者×複数時点のHbA1c」が横に並んでいたら、それをtidy形式に直すのが pivot_longer() の仕事だ。

R# 横長データ（よくあるExcel形式）
wide_df <- tibble(
  id      = c("P001", "P002", "P003"),
  hba1c_0 = c(9.2, 8.6, 7.8),    # 治療開始前
  hba1c_3 = c(8.1, 7.9, 7.1),    # 3ヶ月後
  hba1c_6 = c(7.4, 7.2, 6.8)     # 6ヶ月後
)

# ── pivot_longer：横→縦 ──
long_df <- wide_df |>
  pivot_longer(
    cols      = starts_with("hba1c_"),  # 変換する列
    names_to  = "time",                   # 列名が入る新しい列
    values_to = "hba1c",                  # 値が入る新しい列
    names_prefix = "hba1c_"              # 列名のプレフィックスを除去
  ) |>
  mutate(time = as.integer(time))

long_df
# # A tibble: 9 × 3
#   id     time hba1c
#   <chr> <int> <dbl>
# 1 P001      0   9.2
# 2 P001      3   8.1
# 3 P001      6   7.4  ...

# ── pivot_wider：縦→横（逆操作）──
long_df |>
  pivot_wider(names_from = time, values_from = hba1c,
              names_prefix = "hba1c_")
学
学生 縦長にすると行が増えますね。でもこの形のほうがggplot2で時系列グラフを描きやすいということですか？
師
先生 その通り。ggplot2はtidy形式を前提として設計されている。時点を x 軸に、HbA1cを y 軸に、患者IDを color に——これが縦長ならaes(x = time, y = hba1c, color = id)の1行で書ける。横長のままだと列ごとに geom_line を追加しなければならなくなる。

2.8テーブルを結合する ── join 操作

師
先生 臨床研究では「患者基本情報テーブル」と「検査値テーブル」が別ファイルになっていることが多い。これらをIDをキーにして結合するのが join 操作だ。SQLに慣れていれば馴染み深いはずだよ。
left_join()
🟧⚪
左テーブルの全行を保持。右に対応がなければNA
inner_join()
⬛🟫
両テーブルに存在するキーの行のみ残す
full_join()
🟧🟧
両テーブルの全行を保持。対応がなければNA
anti_join()
🟧❌
右テーブルに存在しないキーの行のみ残す（欠損チェックに便利）
semi_join()
✅⬛
右テーブルに存在するキーの行を残すが、右の列は追加しない
right_join()
⚪🟦
右テーブルの全行を保持。left_join の左右反転版
R# 2つのテーブルを準備
demographics <- tibble(
  id  = c("P001", "P002", "P003", "P004"),
  age = c(58L, 43L, 67L, 51L),
  sex = c("M", "F", "M", "F")
)

lab_results <- tibble(
  id    = c("P001", "P002", "P005"),  # P003,P004は欠損、P005は新規
  hba1c = c(7.2, 9.4, 8.8),
  ldl   = c(112, 145, 98)
)

# ── left_join（最も頻用）──
merged <- demographics |>
  left_join(lab_results, by = "id")
# P003, P004 の hba1c/ldl は NA になる

# ── キー名が違う場合 ──
left_join(demographics, lab_results,
           by = c("id" = "patient_id"))  # 左=id, 右=patient_id

# ── anti_join で結合できなかった行を確認 ──
demographics |> anti_join(lab_results, by = "id")
# P003, P004 が返る → 検査データ未収集の患者を特定できる
join 前のデータ品質チェック： 結合前に必ず両テーブルのキー列の重複・NA・表記ゆれを確認すること。n_distinct(df$id) == nrow(df) でキーの一意性を確認できる。

2.9欠損値 NA との付き合い方

師
先生 臨床データに欠損値はつきものだ。RはNAを「値が存在しない」という明示的な概念として扱う。NAは伝染する——NA + 1 は NA になる。だから常に意識的に対処する必要がある。 
R# ── NA の基本操作 ──
is.na(NA)            # TRUE
NA == NA            # NA（！== は使えない）

# ── データ全体の欠損を確認 ──
patients |>
  summarise(across(everything(), ~ sum(is.na(.x))))

# パーセンテージで
patients |>
  summarise(across(everything(),
    ~ scales::percent(mean(is.na(.x)), accuracy = 0.1)))

# ── NA を除外して計算（na.rm）──
mean(patients$hba1c, na.rm = TRUE)
sum(patients$hba1c,  na.rm = TRUE)

# ── NA 行を除外 ──
patients |> filter(!is.na(on_insulin))          # 特定列のNA除外
patients |> drop_na()                          # 全列のNA含む行を除外
patients |> drop_na(hba1c, bmi)               # 指定列のNA含む行を除外

# ── NA を置換 ──
patients |>
  mutate(on_insulin = replace_na(on_insulin, FALSE))

# ── 単純補完（前の値で埋める・縦方向）──
patients |> fill(on_insulin, .direction = "down")

# ── 平均値補完（慎重に！）──
patients |>
  mutate(hba1c = if_else(is.na(hba1c),
                          mean(hba1c, na.rm = TRUE), hba1c))
欠損値の種類と対処：
・MCAR（完全無作為欠損）: 単純除外でバイアスなし
・MAR（無作為欠損）: 多重代入法（mice パッケージ）が適切
・MNAR（非無作為欠損）: 欠損自体が情報を持つ（例：副作用で脱落）、感度分析が必要
臨床研究では欠損の理由を確認することが最優先。

2.10臨床データ整形の実例 ── 生データから分析可能な形へ

師
先生 実際の臨床研究では、CSVを読み込んでそのまま分析できることはほぼない。典型的な整形フローをまとめよう。 
Rlibrary(tidyverse)
library(janitor)    # clean_names() で列名を統一

# ── ① CSVの読み込みと列名整形 ──
raw <- read_csv("clinical_data.csv", na = c("", "NA", "N/A", "-")) |>
  clean_names()   # 列名をスネークケースに統一

glimpse(raw)

# ── ② 型変換 ──
df <- raw |>
  mutate(
    enrollment_date = as.Date(enrollment_date, format = "%Y/%m/%d"),
    sex             = factor(sex, levels = c("M", "F")),
    treatment_group = factor(treatment_group,
                              levels = c("Control", "Drug_A", "Drug_B"))
  )

# ── ③ 変数追加 ──
df <- df |>
  mutate(
    age_at_enrollment = as.integer(difftime(enrollment_date, birthdate,
                                    units = "days")) %/% 365L,
    egfr_stage = case_when(
      egfr >= 90              ~ "G1",
      between(egfr, 60, 89)   ~ "G2",
      between(egfr, 30, 59)   ~ "G3",
      between(egfr, 15, 29)   ~ "G4",
      egfr < 15               ~ "G5",
      TRUE                     ~ NA_character_
    ) |> factor(levels = paste0("G", 1:5))
  )

# ── ④ 解析対象の絞り込み ──
analysis_set <- df |>
  filter(
    age_at_enrollment >= 20,
    !is.na(hba1c_baseline),
    !is.na(treatment_group)
  )

# ── ⑤ Table 1 の作成（tableone パッケージ）──
library(tableone)
CreateTableOne(
  vars      = c("age_at_enrollment", "sex", "hba1c_baseline", "egfr_stage"),
  strata    = "treatment_group",
  data      = analysis_set,
  factorVars = c("sex", "egfr_stage")
)
janitor パッケージ： clean_names() は日本語を含む列名・スペース・大文字混在などを一括でスネークケースに変換する。多人数が作ったExcelを読み込む際には必須に近いツール。また tabyl() はクロス集計表を素早く作るのに便利。

2.11練習問題

🧠 問題1：データ整形

以下の patients tibble（10名分）を使って、①BMI別カテゴリ（標準/過体重/肥満）ごとの人数・HbA1c平均・HbA1c標準偏差を計算し、②結果を HbA1c 平均の降順に並べよ。

Rpatients |>
  mutate(bmi_cat = case_when(
    bmi < 25 ~ "標準",
    bmi < 30 ~ "過体重",
    TRUE     ~ "肥満"
  )) |>
  group_by(bmi_cat) |>
  summarise(
    n          = n(),
    mean_hba1c = round(mean(hba1c, na.rm = TRUE), 2),
    sd_hba1c   = round(sd(hba1c, na.rm = TRUE),   2),
    .groups = "drop"
