高次元の向こう側
〜t-SNE/UMAPによる究極の次元削減〜

数万遺伝子の発現データを2次元に圧縮し、目に見えないクラスター構造を浮かび上がらせる——PCAから始まり、t-SNE・UMAPへ。高次元データ解析の最前線を、医学応用とともに学ぶ。

📌 この章の学習目標

  • 次元の呪いとは何か、なぜ高次元データは難しいかを説明できる
  • PCAの原理（分散最大化・固有値分解）を理解し、biplot を描ける
  • t-SNEのアルゴリズム概念（高次元の類似度→低次元への変換）を説明できる
  • UMAPの特徴（速度・大域構造の保持）を t-SNE と比較できる
  • Rで scRNA-seq データに t-SNE/UMAP を適用できる
  • k-means・階層クラスタリングと組み合わせてクラスターを同定できる
  • 次元削減の結果を過信しないための注意点を説明できる

7.1次元の呪いと次元削減 ── なぜ高次元は難しいのか

師
先生 序章でこう言ったのを覚えているか——「100人の患者に50項目のデータがあれば、それは100人×50次元の空間に存在するデータだ」と。50次元でも十分厄介だが、シングルセルRNA-seqでは1細胞あたり2万遺伝子の発現量、つまり2万次元のデータを扱う。
学
学生 2万次元……想像もできません。何が問題なんですか？
師
先生 これが「次元の呪い（Curse of Dimensionality）」だ。次元が増えるにつれて3つの問題が起きる。第一に、データ点間の距離が均一化してしまい「近い・遠い」の区別がつかなくなる。第二に、データを埋めるのに必要なサンプルサイズが指数関数的に増える。第三に、ほとんどの次元はノイズで、真のシグナルは少数の次元に集中していることが多い。次元削減はその「真の低次元構造」を見つける作業だ。

🌌 次元削減の3つのアプローチ

線形（PCA）：データの分散を最大に保つ方向（主成分）を見つけて射影する。計算が速く解釈しやすいが、非線形な構造は捉えられない。

確率的近傍埋め込み（t-SNE）：高次元での「近傍関係」を低次元でも保つように最適化する。局所的なクラスター構造を非常によく可視化できる。

多様体学習（UMAP）：データが「なめらかな多様体の上にある」という仮定のもと、位相的な構造を保ちながら低次元に埋め込む。t-SNEより速く大域構造も保持する。

7.2PCA ── 線形な次元削減の基盤

師
先生 主成分分析（PCA）はt-SNE・UMAPの「前処理」としても使われるほど重要な手法だ。高次元データを少数の主成分に圧縮し、どの変数が寄与しているかも解釈できる。まずここをしっかり理解しよう。 
R# ── PCA の実行 ──
# 数値変数だけ抽出して標準化
patients_num <- patients |>
  select(age, bmi, hba1c, sbp, dbp, egfr, ldl) |>
  drop_na() |>
  scale()                     # 標準化（PCAでは必須）

pca_result <- prcomp(patients_num, center = FALSE, scale. = FALSE)

# ── 分散の説明割合（Scree plot）──
summary(pca_result)$importance[, 1:5]
#                          PC1   PC2   PC3   PC4   PC5
# Standard deviation      1.62  1.20  1.03  0.88  0.74
# Proportion of Variance  0.375 0.205 0.152 0.111 0.079
# Cumulative Proportion   0.375 0.580 0.732 0.843 0.922

# ── Scree plot（ggplot2）──
tibble(
  pc  = paste0("PC", 1:ncol(patients_num)),
  var = pca_result$sdev^2 / sum(pca_result$sdev^2) * 100
) |>
  slice_head(n = 7) |>
  ggplot(aes(x = fct_inorder(pc), y = var)) +
  geom_col(fill = "#2a6b6e", alpha = 0.8) +
  geom_line(aes(group = 1), color = "#c8922a", linewidth = 1) +
  geom_point(color = "#c8922a", size = 2) +
  labs(title = "Scree plot", x = NULL, y = "説明分散割合 (%)") +
  theme_classic()

# ── Biplot（factorextra パッケージ）──
library(factoextra)
fviz_pca_biplot(
  pca_result,
  col.var = "#8b3a2a",     # 変数の矢印の色
  col.ind = patients$sex,  # 個体を性別で色分け
  palette = c("#2a6b6e", "#c8922a"),
  addEllipses = TRUE,
  legend.title = "性別",
  repel = TRUE
) + theme_classic()

# ── 主成分得点の取得 ──
pca_scores <- as_tibble(pca_result$x) |>
  bind_cols(patients |> drop_na(age, bmi, hba1c, sbp, dbp, egfr, ldl) |>
            select(sex, treatment, on_insulin))
PCAの解釈： PC1の「因子負荷量」が大きい変数ほど、その主成分に強く寄与している。たとえばPC1にsbp・dbpの負荷が大きければ「PC1は血圧因子」と解釈できる。Biplotでは変数の矢印の方向・長さでこれを視覚的に読む。

7.3t-SNE ── 局所構造を保つ確率的埋め込み

師
先生 t-SNE（t-distributed Stochastic Neighbor Embedding）は2008年にvan der MaatenとHintonが発表した手法だ。高次元空間での「近傍の類似度」をガウス分布で定義し、低次元でも同じ構造になるようにt分布（自由度1）を使って最適化する。
学
学生 なぜ低次元側だけt分布を使うんですか？
師
先生 鋭い質問だ。t分布はガウス分布より裾が重い——つまり遠い点には「さらに遠く離れろ」という強いプレッシャーをかける。こうすることでクラスター間の「空白」が明確になり、視覚的に美しいクラスター構造が浮かび上がるんだ。ただし、クラスター間の距離は意味を持たないので注意が必要だ。

7.3t-SNE ── 局所構造を保つ確率的埋め込み

    

      
師

      

        先生
        t-SNEは2008年にvan der MaatenとHintonが発表した手法だ。高次元空間での「近傍の類似度」をガウス分布で定義し、低次元でも同じ構造になるようにt分布（自由度1）を使って最適化する。低次元側でt分布を使うのは、裾が重いためクラスター間の空白が明確になるからだ。
      

    


R# ── t-SNE の実行（Rtsne パッケージ）──
library(Rtsne)

# 前処理：PCAで50次元に圧縮してからt-SNEにかける（推奨ワークフロー）
set.seed(42)
tsne_result <- Rtsne(
  patients_num,
  dims        = 2,          # 出力次元
  perplexity  = 30,         # 近傍数の目安（5〜50）
  theta       = 0.5,        # 近似精度（0=完全, 1=高速）
  max_iter    = 1000,       # 反復回数（収束まで）
  pca         = TRUE,        # PCAで前処理
  initial_dims = 20,        # PCAで圧縮する次元数
  check_duplicates = FALSE
)

# 結果をtibbleに変換
tsne_df <- tibble(
  tsne1 = tsne_result$Y[, 1],
  tsne2 = tsne_result$Y[, 2]
) |> bind_cols(
  patients |> drop_na(age, bmi, hba1c) |>
  select(sex, treatment, hba1c, on_insulin)
)

# ── t-SNE プロット ──
tsne_df |>
  ggplot(aes(x = tsne1, y = tsne2,
             color = hba1c, shape = sex)) +
  geom_point(size = 2.5, alpha = 0.8) +
  scale_color_gradient2(
    low = "#2a5080", mid = "#fdfaf3", high = "#8b3a2a",
    midpoint = 7
  ) +
  labs(title = "t-SNE（患者データ）",
       x = "t-SNE 1", y = "t-SNE 2",
       color = "HbA1c") +
  theme_classic()


    

      
パラメータデフォルト効果と推奨値

      
        
perplexity30実効的な近傍数。n/100程度が目安。5〜50の範囲で複数試す

        
max_iter1000反復回数。複雑なデータは2000以上必要なことも

        
theta0.5Barnes-Hut近似の精度。0に近いほど精確だが遅い

        
initial_dims50PCA前処理の次元数。scRNA-seqでは通常50

        
seedランダムset.seed()で固定。結果の再現性のために必須

      
    


    

      t-SNEの3大注意事項： ①クラスター間の距離に意味はない（2つのクラスターが近くても遠くても、それは高次元の距離を反映しない）。②クラスターのサイズに意味はない（元のサイズとは無関係）。③perplexityを変えると図が変わる——複数の値で確認すること。
    

  


  
  

    
7.4UMAP ── 速く・大域構造も保持する

    

      
師

      

        先生
        UMAPはMcInnesが2018年に発表した手法で、理論的な基盤は「リーマン多様体上の均一分布」という位相数学だ——難しく聞こえるが、要は「データが高次元のなめらかな面の上にある」と仮定して、その構造を保ったまま2次元に折りたたむ。t-SNEより10〜100倍速く、大域的な構造（クラスター間の関係）も比較的よく保持する。
      

    


R# ── UMAP の実行（uwot パッケージ）──
library(uwot)

set.seed(42)
umap_result <- umap(
  patients_num,
  n_components  = 2,    # 出力次元
  n_neighbors   = 15,   # 局所構造の近傍数（5〜50）
  min_dist      = 0.1,  # 低次元での点の最小距離（0〜1）
  metric        = "euclidean",
  n_epochs      = 200
)

umap_df <- tibble(
  umap1 = umap_result[, 1],
  umap2 = umap_result[, 2]
) |> bind_cols(
  patients |> drop_na(age, bmi, hba1c) |>
  select(sex, treatment, hba1c, on_insulin)
)

# ── UMAP プロット ──
umap_df |>
  ggplot(aes(x = umap1, y = umap2,
             color = treatment, alpha = hba1c)) +
  geom_point(size = 2) +
  scale_color_manual(values = c("#2a6b6e", "#c8922a")) +
  scale_alpha_continuous(range = c(0.4, 1)) +
  labs(title = "UMAP（患者データ）",
       x = "UMAP 1", y = "UMAP 2") +
  theme_classic()

# ── scRNA-seqへの応用（Seurat パッケージ）──
# 実務ではSeuratがscRNA-seq解析の標準パッケージ
library(Seurat)
seurat_obj <- CreateSeuratObject(counts = gene_counts)
seurat_obj <- seurat_obj |>
  NormalizeData() |>
  FindVariableFeatures() |>
  ScaleData() |>
  RunPCA() |>
  FindNeighbors(dims = 1:20) |>
  FindClusters(resolution = 0.5) |>
  RunUMAP(dims = 1:20)

DimPlot(seurat_obj, reduction = "umap", label = TRUE)

  


  
  

    
7.5PCA・t-SNE・UMAP の比較


    

      

        
PCA

        
prcomp() / factoextra

        
線形・高速・再現性あり。軸が解釈可能。非線形構造は見えない。まず最初に試すべき基本手法。

      

      

        
t-SNE

        
Rtsne::Rtsne()

        
局所構造（クラスター）の可視化に優れる。低速・非決定論的・クラスター間距離に意味なし。perplexityに敏感。

      

      

        
UMAP

        
uwot::umap()

        
t-SNEより高速・大域構造も保持。scRNA-seq解析の現在の標準。パラメータが少なく使いやすい。

      

    


    

      
観点PCAt-SNEUMAP

      
        
速度◎ 非常に速い△ 遅い○ 速い

        
局所構造の保持△ 線形のみ◎ 非常に良い◎ 良い

        
大域構造の保持◎ 良い△ 乏しい○ 比較的良い

        
再現性◎ 完全△ seed固定が必要△ seed固定が必要

        
軸の解釈◎ 可能✕ 不可✕ 不可

        
大規模データ○ 可能△ 限界あり◎ スケール可能

        
主な用途前処理・探索・解釈クラスター可視化scRNA-seq・大規模探索

      
    

  


  
  

    
7.6クラスタリングとの組み合わせ ── 見えた構造に名前をつける


    

      
師

      

        先生
        UMAPで浮かび上がったクラスターは「見えた」だけで、まだ「名前がない」。k-meansや階層クラスタリングでクラスターを定義し、そのラベルをUMAP図に色付けすることで「このクラスターは高HbA1c群」といった解釈につなげる。
      

    



R# ── k-means クラスタリング ──
set.seed(42)
km <- kmeans(patients_num, centers = 3, nstart = 25)

      【注意1】クラスター間の距離に意味はない： t-SNEの2つのクラスターが「近い」からといって、高次元空間でも近いとは限らない。UMAPは比較的改善されているが、絶対視してはいけない。

      【注意3】ランダムシードを固定する： t-SNEもUMAPも確率的アルゴリズムだ。set.seed()を固定しないと、実行のたびに違う図になる。論文に掲載する場合は必ずシードを明記する。
