01

プログラミングとは何か

コンピュータの仕組み

コンピュータは本質的に「0と1しか理解しない」機械です。もっと正確に言えば、電気が流れるか流れないか、それだけです。理解しているわけではなく、0と1の羅列を処理しているにすぎません。

あらゆるデータ——文字も画像も音楽も——この0と1の組み合わせで表現できます。しかし、そのまま私たちが扱うのは不可能です。だから私たちにわかる言葉に翻訳したものがプログラミング言語です。とくに人間の言語に近いものを「高級言語」と呼びます。Rもその一つです。

現代の主流はオブジェクト指向という設計思想です。変数に「型」や「特徴」を持たせることで複雑な処理が簡潔に書けます。今はそこまで知らなくていい——まずは基本から。

変数と代入

数学とプログラミングは似ているようで、決定的に異なる点があります。数学の = は「等しい」という意味ですが、プログラミングの = は「代入」です。

数学 vs プログラミング 
# 数学では「a = a + 1」は矛盾
# プログラミングでは「aの現在の値に1を加えて、aに入れ直す」という意味

a = 1      # a に 1 を代入
a = a + 1  # a を 2 に更新

Rでは代入演算子として <- を使います（= も使えますが、慣習的に <- が好まれます）。

R — 代入演算子 
a <- 1        # a に 1 を代入
a <- a + 1   # a を更新（a は 2 になる）
print(a)     # コンソールに表示 → [1] 2
数学では a = a + 1 はありえませんが、プログラミングでは「今のaに1を足してaに戻す」という操作です。Rの代入演算子 <- はまさに「右から左へ値を流し込む」イメージです。= を「等号」ではなく「代入」と読む癖をつけましょう。

関数と引数

関数とは「何かを受け取って何かを返す処理のまとまり」です。数学の関数 f(x) と同じイメージです。x の部分が「引数（ひきすう）」です。

R — 関数の基本 
# 関数の呼び出し: 関数名(引数)
sqrt(16)         # → 4  （平方根）
log(100, base = 10)  # → 2  （対数、底を指定）
abs(-5)           # → 5  （絶対値）

# 自分で関数を定義することもできる
my_square <- function(x) {
  x ^ 2
}
my_square(4)  # → 16
用語意味例
変数値を入れる「箱」a <- 5
関数処理のまとまりsqrt(x)
引数関数に渡す値sqrt(16) の 16
代入変数に値を入れるa <- 1
演算子計算・比較の記号+ - * / ^ <-

02

Rの基礎

ベクトル：Rの最小単位

Rの最大の特徴のひとつが「ベクトル」という概念です。Rにおける最小のデータ単位は実はすべてベクトルです。a <- 1 と書いても、それは「長さ1のベクトル」にすぎません。

R — ベクトルの作成と操作 
# c() でベクトルを作る（combine の略）
a <- c(1, 2, 3)   # 数値ベクトル
b <- c("A", "B", "C") # 文字列ベクトル

# 1つだけのときは c() を省略できる
x <- 5          # x <- c(5) と同じ

# ベクトルの要素にアクセス（1始まりに注意！）
a[1]    # → 1
a[2]    # → 2
a[2:3]  # → 2 3

# ベクトル演算（要素ごとに計算される）
a * 2   # → 2 4 6
a + a   # → 2 4 6
sum(a)  # → 6
mean(a) # → 2
ベクトルの要素はすべて同じ型でなければなりません。c(1, "a", 3) のように数値と文字列を混ぜると、自動的に全て文字列に変換されてしまいます（型の強制変換）。
R — 型の強制変換（注意！） 
x <- c(1, "a", 3)
class(x)   # → "character"（全部文字列になった！）
x          # → "1" "a" "3"（数値の 1, 3 も文字列に）

データフレーム：表形式のデータ

統計解析では「表」が基本です。Rの「データフレーム」は列と行を持つテーブルで、Excelのシートに相当します。各列がひとつのベクトルになっています。

R — データフレームの作成 
# data.frame() で作成
df <- data.frame(
  id    = c(1, 2, 3),
  age   = c(25, 34, 28),
  group = c("治療", "対照", "治療")
)

# 確認
head(df)      # 最初の数行を表示
str(df)       # 構造を表示
nrow(df)      # 行数 → 3
ncol(df)      # 列数 → 3

# 列へのアクセス
df$age         # → 25 34 28
df[, "age"]   # 同じ
df[1, ]        # 1行目全体

主要なデータ型

型英語名例用途
数値numeric3.14, 100身長・体重・検査値
整数integer1L, 5Lカウントデータ
文字列character"治療群"グループ名・ID
論理値logicalTRUE, FALSEフィルタ条件
因子factorfactor("A")カテゴリ変数

制御構造：条件分岐とループ

R — if文 と for文 
# 条件分岐
x <- 80
if (x >= 60) {
  print("合格")
} else {
  print("不合格")
}

# for ループ
for (i in 1:5) {
  cat("i =", i, "\n")
}

# Rらしいやり方：ベクトル化で書く方が速くて簡潔
scores <- c(55, 72, 88, 40, 95)
ifelse(scores >= 60, "合格", "不合格")
Rはベクトル演算が得意です。for ループより ifelse() や apply() 系の関数の方が高速でコードも短くなることが多いです。慣れてきたら意識してみましょう。

データの操作と可視化

tidyverse：現代Rの標準ツール群

tidyverse はデータ分析に必要なパッケージをまとめたコレクションです。現代のRユーザーの大半がこれを使っています。まず最初にインストールしましょう。

R — tidyverse のインストールと読み込み 
# インストール（初回のみ）
install.packages("tidyverse")

# 毎回の作業開始時に読み込む
library(tidyverse)

dplyrでデータを集計する

dplyr パッケージは「英語の動詞」を使ってデータ操作を書けます。処理を |>（パイプ）でつなぐのが特徴です。

R — dplyrの基本動詞 
library(dplyr)

# サンプルデータを作成
patients <- data.frame(
  id      = 1:6,
  age     = c(45, 52, 38, 61, 29, 55),
  group   = c("治療","対照","治療","対照","治療","対照"),
  outcome = c(1, 0, 1, 0, 1, 1)
)

# パイプでつなぐ（|> は「そして」と読む）
patients |>
  filter(age > 40) |>           # 40歳超だけ
  group_by(group) |>            # グループで分ける
  summarise(
    n           = n(),           # 件数
    mean_age    = mean(age),     # 平均年齢
    event_rate  = mean(outcome)  # イベント率
  )
関数意味
filter()条件を満たす行を選ぶ
select()列を選ぶ
mutate()新しい列を作る
group_by()グループに分ける
summarise()集計する
arrange()並べ替える

ggplot2でデータを可視化する

ggplot2 は「グラフの文法（Grammar of Graphics）」に基づいたグラフ作成パッケージです。層を重ねるように書きます。

R — ggplot2 の基本 
library(ggplot2)

# 箱ひげ図 + データ点
ggplot(patients, aes(x = group, y = age, fill = group)) +
  geom_boxplot(alpha = 0.5) +
  geom_jitter(width = 0.1, size = 2) +
  labs(
    title = "グループ別年齢分布",
    x     = "群",
    y     = "年齢（歳）"
  ) +
  theme_minimal() +
  theme(legend.position = "none")
ggplot2 の書き方は「ggplot(データ) + geom_*(図形の種類) + labs(ラベル) + theme(見た目)」と覚えましょう。+ で層を重ねていくイメージです。

04

メタ解析をRで実施する

メタ解析とは

複数の独立した研究をまとめて統合することで、より大きなサンプルサイズ相当のエビデンスを得る手法がメタ解析です。系統的レビューの定量的な部分にあたります。

個々の研究には「真の効果量」があり、それに研究ごとのランダム変動（sampling error）が加わった値が観察されます。研究間でも真の効果量が異なる（異質性）場合はランダム効果モデルを使います。

モデル前提場面
固定効果モデル全研究の真の効果量が同一非常に均質な研究群
ランダム効果モデル真の効果量が研究間で分布通常の系統的レビュー

metaパッケージによる解析

R — meta パッケージのインストールとデータ準備 
install.packages("meta")
library(meta)

# 二値アウトカムのデータ（イベント数・非イベント数）
study_data <- data.frame(
  study    = c("Smith 2018", "Jones 2019", "Park 2020",
               "Lee 2021", "Tanaka 2022"),
  event_t  = c(15, 22, 8, 30, 12),   # 治療群イベント数
  n_t      = c(80, 100, 60, 120, 75), # 治療群総数
  event_c  = c(25, 35, 15, 45, 20),  # 対照群イベント数
  n_c      = c(80, 100, 60, 120, 75)  # 対照群総数
)
R — metabin() でメタ解析を実行 
# 二値アウトカムのメタ解析（オッズ比）
m1 <- metabin(
  event.e  = event_t,  # 治療群イベント数
  n.e      = n_t,      # 治療群総数
  event.c  = event_c,  # 対照群イベント数
  n.c      = n_c,      # 対照群総数
  studlab  = study,    # 研究ラベル
  data     = study_data,
  sm       = "OR",     # 効果量の種類（OR/RR/RD）
  method   = "MH",    # Mantel-Haenszel 法
  random   = TRUE     # ランダム効果モデル
)

# 結果サマリー
summary(m1)
効果量の指定：sm = "OR"（オッズ比）、sm = "RR"（リスク比）、sm = "RD"（リスク差）。連続アウトカムには metacont() を使います。

異質性と出版バイアスの評価

R — 異質性と出版バイアス 
# I² 統計量（出力から確認）
m1$I2         # I² の値（%）
m1$tau2       # τ²（ランダム効果の分散）

# ファンネルプロット（出版バイアスの視覚的確認）
funnel(m1,
  xlab    = "log オッズ比",
  contour = c(0.9, 0.95, 0.99),  # 有意水準の等高線
  col.contour = c("gray90", "gray70", "gray50")
)
legend("topright", c("p < 0.1", "p < 0.05", "p < 0.01"),
       bty = "n", fill = c("gray90", "gray70", "gray50"))

# Egger の回帰テスト
metabias(m1, method.bias = "Egger")
指標意味目安
I²研究間の異質性の割合0–25% 低、26–50% 中、51%+ 高
τ²真の効果量の分散0 に近いほど均質
Q検定 p値異質性の検定p < 0.1 で異質性あり
I² が高いからといって即「問題あり」ではありません。臨床的文脈や研究デザインの違いを考慮して解釈してください。数字だけでなく「なぜ異質なのか」を考えることが大切です。

R Markdownでレポートを作る

解析結果はちゃんとした形にしないと意味がありません。R Markdownは「解析コード」と「文章」を一つのファイルにまとめて、ボタン一発でPDFやHTMLに出力できる仕組みです。論文の補足資料やレポートに使えます。

Rmdファイルの基本構造

R Markdownファイル（.Rmd）は3つのパートで構成されます。

template.Rmd — ファイルの骨格 
---  ← ここからYAMLヘッダー（文書の設定）

author: "金子幸弘"
date: "`r Sys.Date()`"
output:
  html_document:
    toc: true
    toc_float: true
    theme: "flatly"
---  ← ここまでYAMLヘッダー
