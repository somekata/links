基本セットアップInstallation & Initial Setup

⚙️インストール手順

  1. zotero.org からデスクトップ版（Windows / Mac / Linux）をダウンロード・インストールする。

  2. 同サイトでアカウントを作成し、デスクトップ版の 編集 → 設定 → 同期（Mac: Zotero → 設定 → 同期）でログインする。

  3. ブラウザ拡張 Zotero Connector をインストールする（Chrome / Edge / Firefox / Safari 対応）。

  4. デスクトップ版 Zotero を起動した状態でブラウザを使うと、Connector が論文情報を自動取得できる。

ℹ️
同期について文献情報（メタデータ）は無料で無制限に同期される。PDFなどの添付ファイルのみがクラウド容量を消費する（→ ④ストレージ管理を参照）。

📁コレクション（フォルダ）管理

  1. 左ペインの「My Library」を右クリック → 新しいコレクション でフォルダを作成する。

  2. コレクションはネスト（階層化）可能。プロジェクト → DB別 → スクリーニング段階別の構成が有効。

  3. 1つの文献を複数コレクションに所属させられる（実体は1つ、コピーではない）。

🏷️タグの活用

文献にタグを付けることでコレクションをまたいだ横断管理が可能。スクリーニング管理に特に有効。タグに色を設定すると（右クリック → 色を割り当て）文献一覧に色付きドットが表示される（最大9色）。

🔍重複文献の検出・統合

  1. 左ペインの 「重複文献」（Duplicate Items）をクリックする。

  2. 重複候補がペアで表示される。右ペインで内容を確認し、「文献を統合」（Merge Items）をクリック。

  3. 複数DBから収集したSR/MAでは、スクリーニング前に必ず実施する。

🗒️ノートと関連文献

  • スタンドアロンノートも作成可

🔗 関連（Related）

  • 文献同士をリンク可能

🌐英語系データベース

データベース方法手順・注意点
PubMed Connector対応 検索結果ページでConnectorアイコン（フォルダ形）をクリック → チェックボックスで選択 → OK。OA論文はPDFも自動添付されることがある（容量注意）。
Scopus Connector対応 RIS可 Connectorで直接インポート可。大量件数の場合はDB側で「Export → RIS形式」を選択し、ファイルをZoteroにインポートする方が確実。
Web of Science Connector対応 RIS可 Connectorで取り込み可。500件超の一括エクスポートはDB側でRIS/BibTeX出力が必要。
Embase RIS可 対象文献を選択 → 「Export」→「RIS format (Mendeley, EndNote)」を選択してダウンロード → Zoteroで「ファイル → インポート」。
Cochrane Library RIS可 Connector不安定 Connectorは個別文献には使えるが複数件は不安定。確実な方法：チェックボックスで選択 →「Export selected citation(s)」→「RIS (Reference Manager)」→ Download → Zoteroにインポート。Reviews / Trials 等のカテゴリごとに別々にエクスポートが必要。
CINAHL (EBSCOhost) RIS可 文献を選択 →「Export」→「Direct Export in RIS Format (EndNote, Zotero等)」を選択。EBSCOhostプラットフォームなので、MEDLINEなど他のEBSCOhostDB でも同様の手順。
Ovid (MEDLINE / Embase) RIS可 結果を選択 →「Export」→「Citation Manager (RIS)」を選択してダウンロード。Ovidプラットフォーム経由で複数DBを同時検索する施設向け。
Google Scholar Connector対応 件数制限あり 個別論文ページでConnectorを使用。SR/MAの一次検索には再現性が低く不向き。補完的な使用にとどめる。

🇯🇵日本語データベース

データベース方法手順・注意点
医学中央雑誌
（医中誌Web）		 要工夫 Zoteroへの直接ConnectorエクスポートはデフォルトでなHIStoiは非対応（ダイレクトエクスポート選択肢にZoteroなし）。

方法A（試す価値あり）：医中誌の「ダイレクトエクスポート」→「EndNote」を選択して実行すると、施設・環境によってはZoteroが文献管理ソフトとして認識されZoteroに直接取り込まれることがある。

方法B（確実）：医中誌 →「ダイレクトエクスポート」→「Refworks」または「EndNote XML」でファイルを保存 → Zoteroで「ファイル → インポート」→ 形式を選択してインポート。

※日本語文献のメタデータ（著者名・雑誌名）が文字化けする場合があるためインポート後に確認・修正を要することがある。
CiNii Research Connector対応 RIS可 Connectorで個別論文から取り込み可能。または論文ページの「引用」→「RIS形式でエクスポート」を選択してファイルをインポート。
※日本人著者名の姓・名が逆になる場合がある（Zotero 7以降のバージョンで改善済みの部分あり）。インポート後に確認推奨。
J-STAGE Connector対応 個別論文ページでConnectorを使用。メタデータとPDF（OA論文）が取り込まれる。
※CiNii同様、著者名の姓名順が逆になることがある。
⚠️
日本語文献の著者名についてZoteroはデフォルトでFamily name / First nameの順で格納する。日本人著者名は「姓 名」なので、CiNiiやJ-STAGEからの取り込みでは姓と名が逆に入力されることがある。インポート後に右ペインの著者欄を確認・修正する習慣をつけると良い。

📋RIS / BibTeX ファイルからのインポート手順（共通）

  1. 各データベース側でRISまたはBibTeX形式のファイルをダウンロードする。

  2. Zoteroデスクトップ版で ファイル → インポート を選択する。

  3. 「ファイル（BibTeX, RIS…）」を選択し、ダウンロードしたファイルを指定する。

  4. インポートオプション（新規コレクションを作成するかなど）を確認して「次へ」をクリック。

  5. インポート後、左ペインの「重複文献」で重複確認を実施する。

ℹ️
文献情報（メタデータ）は容量を消費しない。PDFなどの添付ファイルのみが容量を消費する。300MBはPDFを数十〜数百件インポートすると比較的早く圧迫される。

⚙️PDF自動ダウンロードをオフにする

  1. 編集 → 設定 → 一般（Mac: Zotero → 設定 → 一般）を開く。

  2. 「文献情報取得時にPDFを自動的に添付する」のチェックを外す。

  3. 以降は必要な文献のみ手動でPDFを添付する運用にする。

🗑️ごみ箱（Trash）の管理

⚠️
ごみ箱は自動で空にならない削除した文献はTrashに移動するだけで、手動で「ごみ箱を空にする」（Empty Trash）するまでクラウド容量は解放されない。容量不足を感じたらTrashも必ず確認する。左ペインのTrashを右クリック → 「ごみ箱を空にする」。

☁️PDFをローカル保存してクラウド容量を節約する

  1. 編集 → 設定 → 同期 を開く。

  2. 「添付ファイルをZoteroストレージで同期する」のチェックを外す。

  3. PDFを任意のクラウドフォルダ（Dropbox / OneDrive等）に保存し、Zoteroからは「リンク添付ファイル」として参照する。
    （Ctrl+Shift を押しながらPDFをドラッグするとリンク添付になる）

💡
Zotero 7以降、ZotFileの代替として Zotmoov プラグインが推奨されている。PDFの自動リネーム・移動をDropboxなど任意フォルダに対して設定できる。

論文執筆・引用Word Integration & Citation Workflow

📝Wordプラグインのセットアップ

  1. Zoteroをインストールすると、通常はWordプラグインが自動的にインストールされる。Wordを再起動すると「Zotero」タブが表示される。

  2. 自動インストールされない場合：Zotero → 編集 → 設定 → 引用 → ワードプロセッサ から手動インストール。

  3. Google Docs でも同様に利用可能（Zotero Connector経由）。

✍️引用の挿入ワークフロー

操作Wordメニュー内容
引用を挿入Zotero → Add/Edit Citation引用を挿入したい箇所にカーソルを置き、検索バーで文献を選択。複数文献も同時に追加できる。
参考文献リストを作成Zotero → Add/Edit Bibliography文書末尾など任意の位置に参考文献リストを自動生成。引用を追加・削除すると自動更新される。
引用スタイルの変更Zotero → Document Preferences引用スタイルを一括変更。全引用と参考文献リストが自動で再フォーマットされる。
フィールドを更新Zotero → RefreshZoteroライブラリで文献情報を修正した後、Wordに反映するために実行する。
Zoteroリンクを解除Zotero → Unlink Citations投稿前の最終段階で実行。フィールドコードが普通のテキストに変換される（不可逆・要バックアップ）。
⚠️
Unlink Citationsは不可逆実行するとZoteroとのリンクが永久に切れる。必ず作業前にファイルをバックアップすること。投稿直前の最終作業として行う。

💡効率的な引用挿入のコツ

  • 引用検索バー（Add/Edit Citation）では著者名・タイトル・年で検索可能。部分一致でもよい。
  • 確定前にページ番号の追加も可能（書籍引用で有用）。
  • Vancouverスタイル（番号）設定時、番号は参考文献リストの出現順に自動対応する。
  • 文献情報の誤りはZoteroライブラリ側で修正し、Wordで「Refresh」を実行すれば全引用に反映される。

引用スタイル（CSL）Citation Styles & Journal Formats

🎨引用スタイルとは

Zoteroは CSL（Citation Style Language） という標準形式でスタイルを管理している。11,000以上のジャーナル・機関スタイルが公式リポジトリに登録されており、無料でインストール可能。

カテゴリ代表的なスタイル
番号形式（Vancouver系）Vancouver, ICMJE準拠ジャーナル（Lancet, NEJM, BMJ, JAMA等）
著者-年形式APA, Harvard, Chicago Author-Date
ジャーナル独自PLOS ONE, Nature, Science, Emerging Infectious Diseases, Clinical Infectious Diseases, AAC等

📦ジャーナルのCSLスタイルをインポートする方法

方法A：Zotero Style Repository から検索・インストール（推奨）

  1. Zoteroデスクトップ版で 編集 → 設定 → 引用 → スタイル を開く。

  2. 右下の 「スタイルを追加…」 をクリックする。

  3. Zotero Style Repository のページ（zotero.org/styles）が開く。

  4. ジャーナル名を検索（例：「Emerging Infectious Diseases」「Clinical Infectious Diseases」「Antimicrobial Agents and Chemotherapy」）し、目的のスタイルをクリックすると自動インストールされる。

方法B：CSLファイルを手動インポート（リポジトリにない場合）

  1. ジャーナル公式サイトまたは GitHub（citation-style-language/styles）から .csl ファイルをダウンロードする。

  2. Zotero設定 → 引用 → スタイル → 「＋」ボタン でファイルを選択してインポートする。

✅
感染症系ジャーナルの検索例
Emerging Infectious Diseases（CDC），Clinical Infectious Diseases，Journal of Infection，International Journal of Infectious Diseases，Antimicrobial Agents and Chemotherapy—いずれも Style Repository に収載されている。

🔄投稿先変更時のスタイル一括変更

  1. Wordの「Zotero」タブ → Document Preferences を開く。

  2. 現在のスタイルが表示される。「スタイルを変更…」から新しいスタイルを選択する。

  3. OKをクリックすると、文書内の全引用と参考文献リストが新しいスタイルで自動再フォーマットされる。

💡
ジャーナルを変更して再投稿する際も、このステップだけで参考文献を全て書き直す必要がなくなる。Zoteroを使う最大のメリットの一つ。
