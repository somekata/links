パート2: 問76-175 | アルゴリズム・データ構造・C++/C#応用

🎯 アルゴリズム基礎（問76-100）
問76 易
アルゴリズムの定義として最も正確なものは？
正解: B) 問題を解くための手順や計算方法
良いアルゴリズムは高速かつメモリ効率的です。
問77 中
時間計算量 O(n) のアルゴリズムの特徴は？
正解: B) 入力サイズに比例して時間が増加
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)。
問78 中
空間計算量とは何か？
正解: A) 実行に必要なメモリ量
時間計算量と同等の O(n), O(n²) などで表現。
問79 中
線形探索（Linear Search）の時間計算量は？
正解: C) O(n)
最悪の場合、すべての要素を調べる必要があります。
問80 中
二分探索（Binary Search）の時間計算量は？
正解: B) O(log n)
ソート済み配列で、毎回半分に絞り込むため高速。
問81 中
バブルソート（Bubble Sort）の特徴として正しいものは？
正解: B) 隣同士を比較・交換、O(n²)
単純だが遅いソートアルゴリズム。教育目的で使われます。
問82 中
クイックソート（Quick Sort）の時間計算量は？
正解: C) O(n log n)（平均）
分割統治で効率的。最悪は O(n²) ですが実践的です。
問83 易
動的計画法（Dynamic Programming）の基本概念は？
正解: B) 部分問題の結果を記憶して再利用
フィボナッチ数列、最短経路など、多くの問題で使われます。
問84 難
分割統治法（Divide and Conquer）の特徴は？
正解: A) 問題を小さく分割し、再帰的に解く
マージソート、クイックソートなどで使われます。
問85 中
貪欲法（Greedy Algorithm）の欠点は？
正解: B) 局所最適でも全体最適とは限らない
常に最良の選択をするが、全体では最適でない場合があります。
📊 データ構造基礎（問86-125）
問86 易
リスト（List）データ構造の特徴は？
正解: B) 動的にサイズ変更可能
配列とは異なり、要素を自由に追加・削除できます。
問87 中
スタック（Stack）のデータ構造における操作は？
正解: B) 後入先出（LIFO）
push（追加）、pop（取り出し）操作。関数呼び出し、戻るボタンなど。
問88 易
キュー（Queue）の特徴は？
正解: B) 先入先出（FIFO）
enqueue（追加）、dequeue（取り出し）。待ち行列のイメージ。
問89 中
リンク単一リスト（Singly Linked List）の特徴は？
正解: B) 各ノードが次ノードのポインタのみ保持
前後探索が非効率な代わり、メモリ効率が良い。
問90 中
二重リンク単一リスト（Doubly Linked List）の利点は？
正解: B) 前後両方向に移動可能
単一リストより柔軟ですが、メモリ消費が増加。
問91 中
木（Tree）構造の基本用語で正しい説明は？
正解: B) 根 = トップノード、葉 = 子を持たないノード
深さ（レベル）と高さ（最大深さ）が基本概念。
問92 中
二分探索木（Binary Search Tree）の性質は？
正解: B) 左 < 親 < 右 の関係
この性質により、効率的な検索・挿入が可能。
問93 中
ハッシュテーブル（Hash Table）の利点は？
正解: B) O(1) の高速な検索・挿入
ハッシュ関数でキーを座標に変換、即座にアクセス。
問94 難
ハッシュ衝突（Hash Collision）の対策として最も一般的なものは？
正解: B) チェーン法または開きアドレス法
チェーン法が最も実装が簡単で実用的。
問95 中
グラフ（Graph）とは何か？
正解: B) ノードと枝で構成
木は有向非環グラフ、一般的なグラフはサイクル可能。
問96 中
幅優先探索（BFS）の特徴は？
正解: B) キューを使って層ごとに探索
最短経路探索で使われます。
問97 中
深さ優先探索（DFS）の特徴は？
正解: B) スタックまたは再帰で深く探索
迷路の探索、バックトラッキングに使われます。
問98 難
ダイクストラアルゴリズム（Dijkstra Algorithm）の用途は？
正解: A) 最短経路を見つける
GPS ナビゲーション、ネットワークルーティングで使用。
問99 易
ソーティング（Sorting）の用途として最も一般的なものは？
正解: B) データを順序立てて配列
多くのアプリケーションで基本操作です。
問100 中
マージソート（Merge Sort）の特徴は？
正解: B) 分割統治、安定ソート、O(n log n)
実装複雑ですが、常に安定な実行時間。
🔧 C/C++ 応用（問101-125）
問101 中
C++ のコンストラクタ（Constructor）の役割は？
正解: B) オブジェクト生成時に初期化
デストラクタと対になり、オブジェクトのライフサイクルを管理。
問102 中
C++ のデストラクタ（Destructor）の役割は？
正解: B) リソース解放
メモリリーク防止のため、デストラクタで malloc/new で確保した領域を解放。
問103 難
RAII（Resource Acquisition Is Initialization）パターンとは？
正解: B) 取得と解放の自動化
C++ の安全なリソース管理の標準パターン。例外安全性も提供。
問104 中
C++ のテンプレート（Template）の利点は？
正解: B) 型を指定せず汎用的にコード記述
同じロジックを異なる型に適用（STL コンテナなど）。
問105 中
C++ の STL（Standard Template Library）の機能は？
正解: B) コンテナ、イテレータ、アルゴリズムを提供
vector, map, sort, find など豊富な機能。
問106 中
C++ の vector の特徴は？
正解: B) 動的配列、O(1) のランダムアクセス
末尾追加は償却 O(1)、削除は遅い。
問107 中
C++ の map の特徴は？
正解: B) キー・値ペア、O(log n) 高速検索
二分木ベース、順序保証。unordered_map なら O(1)。
問108 中
C の void* ポインタを C++ で型安全に扱う方法は？
正解: B) テンプレートとジェネリック
template<class T> で型を固定することで安全性確保。
問109 難
C++ のメモリリークを防ぐベストプラクティスは？
正解: B) スマートポインタ使用
RAII により自動的にメモリを解放。現代 C++ の標準。
問110 中
C++ の unique_ptr と shared_ptr の違いは？
正解: B) unique_ptr = 単一所有、shared_ptr = 複数所有
unique_ptr が効率的だが、複数の所有が必要な場合は shared_ptr。
問111 中
C では動的メモリを確保するが、最も安全な方法は？
正解: B) 確保・保持・解放の対を確実に
malloc/free の数をカウント、メモリリーク検出ツール使用推奨。
問112 易
C の strcpy() が危険な理由は？
正解: B) バッファオーバーフロー脆弱性
strncpy() または C++ の std::string を使用すべき。
問113 中
C++ の std::string のメリットは？
正解: B) 動的サイズ、安全な操作
char 配列より安全で便利。C スタイル文字列より推奨。
問114 中
C の printf() の脆弱性「フォーマット文字列攻撃」とは？
正解: B) 入力を直接フォーマット指定で情報漏洩
printf(user_input) はダメ、printf("%s", user_input) とすべき。
問115 中
C++ のイテレータ（Iterator）の役割は？
正解: B) コンテナの要素をポインタのように訪問
for (auto it = vec.begin(); it != vec.end(); ++it) のように使用。
問116 中
C++ の範囲ベース for ループ（Range-based for）の利点は？
正解: B) 簡潔で安全な反復処理
for (auto val : vec) { } で従来の for より読みやすい（C++11）。
問117 中
C の struct と C++ のクラスの最大の違いは？
正解: B) データ集合 vs オブジェクト指向
C++ では struct もクラスと同じ（デフォルト public）。
問118 易
C++ の constexpr の役割は？
正解: B) コンパイル時定数計算
実行時オーバーヘッド 0、埋め込み処理に最適。
問119 中
C++ のムーブセマンティクス（Move Semantics）とは？
正解: B) 所有権を移動
std::move() で一時オブジェクトの所有権を新しい変数に移動。
問120 中
C++ で operator overload（演算子オーバーロード）の用途は？
正解: B) 演算子をカスタム定義
例：Complex a+b を定義して複素数計算を自然に。
問121 易
C++ の << と >> 演算子のオーバーロード用途は？
正解: B) ストリーム入出力
friend ostream& operator<<(...) でカスタムクラスを表示可能。
問122 中
C++ の std::function の用途は？
正解: B) 関数・ラムダを汎用保存
std::function<int(int)> f = ... で色々な呼び出し対象を統一的に扱える。
問123 難
C++ の perfect forwarding とは？
正解: B) 参照の種類を保存したまま転送
template<class T> f(T&& arg) { g(std::forward<T>(arg)); } で左・右値参照を正確に渡す。
問124 中
C の goto 文が現代でも使われない理由は？
正解: B) スパゲッティコード化
構造化プログラミングの推奨により、goto は避けるべき。
問125 中
C++ の inline 関数の目的は？
正解: B) 呼び出しオーバーヘッド削減
コンパイラがソースコード上で関数本体を展開（実際の展開は最適化次第）。
