📚 コンピューター・プログラミング復習問題集

パート1: 問1-75 | 基礎概念・メモリ管理・言語基礎

📖 基礎概念（問1-25）
問1 易
コンピュータの「5大装置」に含まれないものはどれか？
正解: C) 冷却装置
5大装置は「制御装置・演算装置・記憶装置・入力装置・出力装置」です。冷却装置は補助的なコンポーネントで、5大装置には含まれません。
問2 易
CPUが行うべき仕事は次のうちどれか？
正解: B) 制御と演算の両方
CPUは「中央演算処理装置」で、制御装置と演算装置の両方の役割を担います。
問3 易
主記憶装置（メモリ）の特徴として最も正確なものはどれか？
正解: B) 作業中のデータを一時的に保持する
メモリはCPUの「机」の役割で、高速ですが電源遮断でデータが失われます。
問4 易
補助記憶装置（HDD/SSD）の主な用途は何か？
正解: B) データの長期保存
HDD/SSDは「本棚」の役割で、電源が切れてもデータが保存されます。
問5 中
TCP/IPの4階層で、データの信頼性を確保するのはどの層か？
正解: C) トランスポート層
TCP（トランスポート層）は、データが順番通りに届き、壊れていないことを確認します。IPは単に「住所」を付けるだけ。
問6 易
TCPとUDPの最大の違いは何か？
正解: B) TCPは信頼性がある、UDPは速度重視
TCP = 確認応答あり（メール、Web）、UDP = 確認応答なし（動画配信、オンラインゲーム）。
問7 中
IPアドレスの主な役割は何か？
正解: B) 目的地を特定する「住所」
IPアドレスはネットワーク上の「住所」で、データがどこに届けられるかを指定します。
問8 易
2進数の「1」と「0」は、機械的に何で表現されるか？
正解: A) 電圧の有無
デジタル回路では、電圧がある状態を「1」、ない状態を「0」で表します。
問9 中
AND（論理積）ゲートの出力が「1」になる条件は？
正解: B) 両方とも1
AND = 両方の条件が揃わないと出力1にならない（シーソーの両側を押す）。
問10 易
NOT（論理反転）ゲートの動作は？
正解: B) 入力を反転させる（1→0、0→1）
NOT = 「あまのじゃく」で、入力と逆の出力を出す。
問11 中
半加算器で「1+1」を計算した場合、出力は？
正解: B) 和=0, 進位=1
2進数の計算で「1+1=10」なので、和は0、進位（キャリー）は1になります。
問12 易
オペレーティングシステム（OS）の主な役割は？
正解: B) リソースを管理し、アプリが動作する環境を提供
OSは「工場長」で、メモリ、CPU、ディスクなどを効率的に管理します。
問13 中
プロセスとスレッドの違いとして正しいものは？
正解: B) プロセスは独立した実行単位、スレッドは軽量な実行単位
プロセスは独立したメモリ空間を持ち、スレッドは同じプロセス内でメモリを共有します。
問14 易
ファイルシステムの主な役割は？
正解: B) ディスク上のデータを整理・管理する
ファイルシステムは階層的な構造でファイルを管理し、効率的にアクセスできます。
問15 中
コンテキストスイッチとは何か？
正解: A) CPUがあるプロセスから別のプロセスに切り替える
マルチタスキングの実現に必須で、CPUの処理状態（コンテキスト）を保存・復元します。
問16 易
クロック周波数（GHz）が表すものは？
正解: A) 1秒間に何回の信号を出すか
2.4GHzなら1秒間に24億回のクロック信号を出します。周波数が高いほど高速。
問17 中
パイプラインの利点として最も正確なものは？
正解: B) 計算を並列に進めて効率を上げる
回転寿司のように、複数の段階を同時に進めることで、全体のスループットを増加させます。
問18 易
キャッシュメモリの目的は？
正解: B) メモリアクセスを高速化
CPUに近い小容量・高速のメモリで、よく使うデータを保持します。L1, L2, L3キャッシュがあります。
問19 中
キャッシュミスが増加する場合の対策として不適切なものは？
正解: C) CPUクロック周波数を下げる
クロック周波数を下げると、むしろ全体の性能が低下します。キャッシュミスの根本原因に対処すべき。
問20 易
割り込み（Interrupt）の役割は？
正解: B) 周辺機器からの緊急の信号に対応する
割り込みにより、OSは現在の処理を中断して優先度の高い処理に対応できます。
🧠 メモリ管理（問21-50）
問21 中
物理メモリ（実メモリ）とは？
正解: B) コンピュータに実装されたRAM
物理メモリは、実際にマザーボードに装着されたメモリモジュール（DRAM）です。
問22 中
仮想メモリ（Virtual Memory）の目的は？
正解: B) 物理メモリより大きなメモリ空間を提供する
ディスクの一部をメモリの拡張として使い、8GB物理メモリを16GBのように見せることができます。
問23 中
ページング（Paging）の仕組みで正しい説明は？
正解: A) メモリを固定サイズブロック（ページ）に分割し、ディスクと交換
通常4KB単位のページに分け、不要なページをディスク（スワップ領域）に退避させます。
問24 難
スワップ（Swapping）とページング（Paging）の違いは？
正解: B) スワップはプロセス全体、ページングはページ単位
スワップは古い手法（プロセス全体をディスクに退避）、ページングは現代的（部分的に退避）。
問25 中
メモリのメインアドレスとは何か？
正解: A) メモリ内の「住所」を示す番号
物理アドレス（PA）ともいい、実メモリ上の位置を示します。例: 0x12345678（16進数）。
問26 中
仮想アドレス（Virtual Address）の役割は？
正解: A) プログラムが認識するメモリ上の位置
プログラムは仮想アドレスでメモリにアクセスし、MMUが物理アドレスに翻訳します。
問27 難
MMU（メモリ管理ユニット）の主な役割は？
正解: B) 仮想アドレスを物理アドレスに変換
MMUはページテーブルを参照して、仮想アドレスを実メモリ上の物理アドレスに翻訳します。
問28 中
ページテーブルの役割は？
正解: A) 仮想ページから物理ページへのマッピング情報を保持
ページテーブルはMMUが参照して、仮想→物理アドレス変換を行います。
問29 中
TLB（Translation Lookaside Buffer）の目的は？
正解: A) メモリのアドレス翻訳をキャッシュして高速化
TLBはMMU内の高速キャッシュで、頻繁に使う仮想→物理アドレス変換を保持します。
問30 難
ページフォルト（Page Fault）が発生する状況は？
正解: A) アクセスしたページがメモリにない
ページフォルト発生時、OSはディスクからページを読み込んでメモリに置きます。
問31 中
メモリ保護（Memory Protection）の目的は？
正解: A) あるプロセスが他のプロセスのメモリを破壊するのを防ぐ
メモリ保護により、バグやセキュリティ脅威からシステムを守ります。
問32 中
ガベッジコレクション（GC）とは何か？
正解: B) 使用されなくなったメモリを自動回収
GC により、プログラマが明示的に メモリを解放する手間を省けます（Java、Python等）。
問33 難
Mark and Sweep（マーク・スウィープ）GCアルゴリズムの動作は？
正解: B) 到達可能なオブジェクトをマークし、不要なものを削除
Mark フェーズで到達可能なオブジェクトに印、Sweep フェーズで他を削除。
問34 中
世代別GC（Generational GC）の利点は？
正解: B) 若い世代が高頻度で削除される傾向を利用して効率化
多くのオブジェクトはすぐに削除されるため、世代分けで GC 頻度を最適化。
問35 中
Reference Counting（参照カウント）GCの欠点は？
正解: B) 循環参照で削除されないメモリ（メモリリーク）が発生
AがBを参照、BがAを参照 → 両方カウント > 0 となり、削除されません。
問36 易
メモリリークとは何か？
正解: B) 使用されなくなったメモリが回収されない
メモリリークが続くと、メモリ不足でプログラムがクラッシュします。
問37 中
ヒープメモリの用途は？
正解: B) 動的に割り当てられるオブジェクト
new で作成したオブジェクト、String（Java）、動的配列などはヒープに確保。
問38 易
スタックメモリの特徴は？
正解: B) 小容量で高速、LIFO（後入先出）
スタックはローカル変数、関数呼び出しを管理し、関数終了で自動回収。
問39 難
スタックオーバーフロー（Stack Overflow）が発生する状況は？
正解: B) 無限再帰で呼び出しが積み重なる
スタックサイズは限定的なため、深すぎる再帰でオーバーフロー。
問40 中
ポインタ（Pointer）の役割は？
正解: A) メモリのアドレスを保持する変数
ポインタは他の変数のメモリアドレスを格納し、間接的なデータアクセスを可能にします。
問41 難
ダブルフリー（Double Free）エラーの危険性は？
正解: B) 既に解放されたメモリを再度解放、メモリ管理が破壊
セキュリティ脅威で、ゼロ化攻撃などに利用される可能性あり。
問42 易
メモリの DRAM と SRAM の違いは？
正解: B) SRAM は高速だが高コスト、DRAM は低速だが安価
SRAM = キャッシュ（L1, L2, L3）、DRAM = メインメモリ。
問43 中
セグメンテーションフォルト（Segmentation Fault）の原因として最も一般的なものは？
正解: B) 不正なメモリアクセス
NULL ポインタ参照、確保していないメモリへのアクセスなど。
問44 中
メモリマッピングファイル（Memory-Mapped File）の利点は？
正解: A) ファイルをメモリ空間にマップして高速アクセス
OS が自動的にページング・キャッシング管理でき、大容量ファイル処理が効率化。
問45 難
ASLR（Address Space Layout Randomization）の目的は？
正解: B) メモリ配置をランダム化してセキュリティを強化
ASLR により、バッファオーバーフローなどの攻撃が成功しにくくなります。
問46 中
メモリコンプレッション（Memory Compression）の目的は？
正解: B) メモリを圧縮して容量を伸ばす
Windows 10 等で採用、スワップ減少でパフォーマンス向上。
問47 易
NULLポインタとは何か？
正解: A) メモリアドレス 0 を指すポインタ
NULL ポインタはポインタが初期化されていないか、無効であることを示します。
問48 中
void ポインタの特徴は？
正解: A) 任意の型のデータを指せる
void * は汎用ポインタで、キャストして使用します（malloc の戻り値など）。
問49 難
メモリの境界チェック（Bounds Checking）がない言語の危険性は？
正解: B) バッファオーバーフローにより不正メモリアクセス可能
C、C++ は境界チェックがなく、セキュリティ脅威の原因。
問50 中
Out Of Memory（OOM）エラーの原因として最も一般的なものは？
正解: B) メモリリークで使用可能メモリが枯渇
長時間実行されるアプリケーションでメモリリークが蓄積し、OOM 発生。
🔤 プログラミング言語基礎（問51-75）
問51 易
C言語の最大の特徴は？
正解: B) ハードウェアに近い制御と効率性
C言語はポインタとメモリ管理を直接操作できる低レベル言語です。
問52 易
C言語でメモリを動的に確保する関数は？
正解: B) malloc()
malloc はメモリブロック確保、free で解放。手動管理が必要。
問53 中
C言語のポインタ演算の用途として正しいものは？
正解: B) メモリ配列へのアクセス
ptr++、ptr[i]、*(ptr+i) などで配列要素にアクセス。
問54 中
C言語のヘッダファイル（.h）の役割は？
正解: B) 関数宣言、マクロ、型定義を提供
#include でヘッダを読み込み、関数や定数を利用。
問55 易
C++ の主な改善点は？
正解: B) オブジェクト指向、テンプレート、標準ライブラリの追加
C++ = C の上位互換で、より高度な機能を提供。
問56 中
C++ のクラスとは何か？
正解: B) データと機能をまとめたテンプレート
クラスはオブジェクト指向の基本単位で、メンバ変数とメンバ関数を含みます。
問57 中
C++ における継承（Inheritance）の利点は？
正解: B) コード再利用と機能拡張
親クラスの機能を子クラスが引き継ぎ、拡張できます。
問58 中
C++ のポリモーフィズム（多態性）とは？
正解: B) 同名の関数が異なる動作をする機能
演算子オーバーロード、仮想関数（virtual）によって実現。
問59 難
C++ の virtual キーワードの役割は？
正解: B) 実行時に実際のオブジェクトの関数を呼び出す
仮想関数により、基底クラスポインタ経由で子クラスの関数が呼ばれます。
問60 易
C# の主な特徴は？
正解: B) 自動メモリ管理と完全なオブジェクト指向
C# は Java に似た言語で、.NET Framework 上で動作。
問61 中
C# の GC（ガベッジコレクション）の利点は？
正解: B) メモリリークを自動防止
C# では手動の free/delete が不要で、GC が自動回収。
問62 中
C# の using ステートメントの役割は？
正解: B) ネームスペース参照とリソース自動解放
using (resource) { } でスコープを抜ける際、Dispose() が自動呼び出し。
問63 中
C++ とC# の最大の違いは？
正解: B) 手動メモリ管理 vs 自動メモリ管理
C++ は効率性、C# は開発速度と安全性を優先。
問64 易
データ型とは何か？
正解: B) 変数が保持できるデータの種類とサイズ
int（4byte）、char（1byte）、double（8byte）など。
問65 易
配列（Array）とは何か？
正解: B) 同じ型の複数データを順序付けて格納
arr[0], arr[1], ... でアクセス。インデックスは 0 から開始。
問66 中
構造体（Struct）の役割は？
正解: B) 異なる型の複数データをまとめる
struct Person { int age; char name[50]; }; のように定義。
問67 中
スコープ（Scope）の意味は？
正解: B) 変数が有効な範囲
スコープを抜けると変数は破棄される。{ } で区切られた範囲。
問68 易
関数（Function）の目的は？
正解: B) 処理を整理・再利用する
同じ処理を何度も書かず、関数にまとめて呼び出す。
問69 中
再帰関数（Recursive Function）の条件として正しいものは？
正解: A) 終了条件と再帰呼び出し
終了条件がないと無限再帰 → スタックオーバーフロー。
問70 中
カプセル化（Encapsulation）とは何か？
正解: B) データとメソッドを隠し、公開インターフェースのみ提供
private/public などのアクセス修飾子で実現。
問71 中
例外（Exception）ハンドリングの目的は？
正解: B) 予期しないエラーに対応し、プログラム継続
try-catch で例外をキャッチし、エラーに対応。
問72 易
型キャスト（Type Cast）とは何か？
正解: B) あるデータ型を別の型に変換
(int)3.14 = 3、(double)5 = 5.0 など。
問73 中
ジェネリック（Generic）プログラミングの利点は？
正解: B) 型安全性を保ちながらコード再利用
template<class T> や List<T> で異なる型でも同じロジック使用。
問74 中
オーバーロード（Overload）とオーバーライド（Override）の違いは？
正解: B) オーバーロード = 同名異なる引数、オーバーライド = 継承関数再定義
コンパイル時結合 vs 実行時結合の違い。
問75 難
ラムダ式（Lambda Expression）の用途は？
正解: B) 無名関数として簡潔に処理を記述
C++11: [](int x) { return x*2; }、Java/C#: x => x*2。
