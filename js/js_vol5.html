Ch.1 — DOM

DOM ― ブラウザが HTML を「ツリー」として管理する仕組み

DOM とは
→
Document Object Model。ブラウザが HTML を解釈して作る「木構造のオブジェクト」。JS はこのツリーを通じて HTML を読み書きする
なぜ重要？
→
ブラウザ上で「文字を書き換える」「要素を追加する」「色を変える」はすべて DOM 操作。このスライドのナビも DOM + JS
HTML → DOMツリーのイメージ
/* HTML */ /* DOM ツリー */ <body> document <div id="app"> └── body <h1>結果</h1> └── div#app <p id="out"></p> ├── h1 "結果" </div> └── p#out "" </body> // JS からアクセス document.getElementById("out") // p#out を取得 document.querySelector("#app h1") // h1 を取得
F12 → Elements タブ = DOMツリーをブラウザでリアルタイムに見られる
Ch.1 — 要素の取得

要素を取得する ― getElementById と querySelector

取得メソッドの使い分け
// ① getElementById ― id で1つ取得（最速・最多用） const outEl = document.getElementById("result"); // ② querySelector ― CSSセレクタで1つ取得 const btn = document.querySelector("button"); const inp = document.querySelector("#weight"); // id const warn = document.querySelector(".warning"); // class const hba = document.querySelector("input[type=number]"); // ③ querySelectorAll ― 複数取得（NodeList で返る） const cards = document.querySelectorAll(".patient-card"); cards.forEach(card => console.log(card.textContent)); // ④ 子要素の中から探す const form = document.getElementById("form"); const inputs = form.querySelectorAll("input"); // form内のinputのみ // 取得できなかった場合は null → 事前チェックが重要 if (outEl) { outEl.textContent = "表示"; }
🔑 使い分けの目安
getElementById
→ id が決まってる・速い・最多用

querySelector
→ class・属性・複合セレクタ

querySelectorAll
→ 同じ種類の要素を複数まとめて

id が存在しない場合は null が返る。
null.textContent はエラーになる。

Ch.2 — DOM 読み書き

要素を読み書きする ― textContent / innerHTML / value

テキスト・HTML・入力値の読み書き
const p = document.getElementById("output"); const inp = document.getElementById("weight"); // ── 書き込み ───────────────────────────── p.textContent = "BMI: 22.9"; // テキストのみ（XSS安全） p.innerHTML = "<strong>BMI: 22.9</strong>"; // HTML も書ける p.innerHTML = ""; // 中身を空にする // ── 読み取り ───────────────────────────── console.log(p.textContent); // 現在のテキストを取得 const w = parseFloat(inp.value); // input の値（必ず文字列） // ── 属性 ────────────────────────────────── inp.setAttribute("placeholder", "体重(kg)"); inp.getAttribute("type"); // "number" inp.disabled = true; // 無効化 // ── スタイル ─────────────────────────────── p.style.color = "#f87171"; // 文字色を赤に p.style.display = "none"; // 非表示 p.style.fontWeight = "700"; // 太字
🔑 textContent vs innerHTML
  • textContent：テキストのみ。タグは文字として表示。XSSに安全
  • innerHTML：HTMLとして解釈。ユーザー入力を直接渡すと危険
  • ユーザー入力の表示は必ず textContent

Ch.2 — 要素の作成

要素を作って追加・削除する

患者カードを動的に生成する
// 要素を作成 const card = document.createElement("div"); card.className = "patient-card"; card.textContent = "山田太郎 | HbA1c: 7.2%"; // 親要素に追加 const list = document.getElementById("patient-list"); list.appendChild(card); // 末尾に追加 list.prepend(card); // 先頭に追加 // innerHTML でまとめて生成（患者リストを配列から） const patients = [ { name:"山田", hba1c:6.5 }, { name:"鈴木", hba1c:8.2 }, ]; list.innerHTML = patients.map(p => `<div class="card ${p.hba1c >= 7 ? 'alert' : 'ok'}"> ${p.name}: ${p.hba1c}% </div>` ).join(""); // 要素を削除 card.remove(); // 要素自身を削除 list.innerHTML = ""; // 子要素を全消去
💡 innerHTML + map().join("") パターン：配列データから HTML リストを一気に生成する最もよく使われる手法。Vol.3 の map と組み合わせて使う

Ch.3 — イベント

イベント ― ユーザーの操作に反応する

イベントとは
→
ユーザーが「クリック・入力・スクロール」した瞬間にブラウザが発火させる通知。JS はこれを「リスンして」関数を実行する
医療での例
→
「判定ボタンを押す」→ HbA1c判定実行
「体重を入力する」→ リアルタイムBMI計算
「Enter キー」→ フォーム送信
よく使うイベント： click dblclick mouseover keydown keyup input change submit focus blur DOMContentLoaded resize scroll
イベント駆動 = 「何かが起きたら何かをする」― Webアプリの根本原理
Ch.3 — addEventListener

addEventListener ― イベントを「聴く」

クリック・入力・キーボードイベント
// ① クリック：ボタンを押したら計算する const btn = document.getElementById("calcBtn"); btn.addEventListener("click", () => { const w = parseFloat(document.getElementById("weight").value); const h = parseFloat(document.getElementById("height").value) / 100; document.getElementById("result").textContent = `BMI: ${(w/h**2).toFixed(1)}`; }); // ② input：入力のたびにリアルタイム処理 const inp = document.getElementById("hba1c"); inp.addEventListener("input", (e) => { const v = parseFloat(e.target.value); // e.target = 入力要素 const msg = v >= 7.0 ? "🔴 要管理" : "✅ 良好"; document.getElementById("judge").textContent = isNaN(v) ? "" : msg; }); // ③ keydown：Enter キーで実行 inp.addEventListener("keydown", (e) => { if (e.key === "Enter") calculate(); });
🔑 構文の形
要素.addEventListener(
  "イベント名",
  コールバック関数
);
💡 イベントオブジェクト e
  • e.target：イベントが起きた要素
  • e.target.value：input の値
  • e.key：押されたキー名
  • e.preventDefault()：デフォルト動作を止める

Ch.3 — フォーム

フォームの値を取得・バリデーション・クリア

問診フォームの値を取得・検証する
// 各フォーム要素から値を取得 const name = document.getElementById("name").value.trim(); const age = parseInt(document.getElementById("age").value); const hba1c = parseFloat(document.getElementById("hba1c").value); const smok = document.getElementById("smoking").checked; // checkbox const sex = document.querySelector( 'input[name="sex"]:checked').value; // radio const ward = document.getElementById("ward").value; // select // バリデーション（入力チェック） function validate() { const errs = []; if (!name) errs.push("名前を入力してください"); if (isNaN(age)) errs.push("年齢を数値で入力してください"); if (isNaN(hba1c)) errs.push("HbA1cを入力してください"); return errs; } // フォームをクリア document.getElementById("form").reset();
🔑 フォーム要素の .value
input[text] → .value（string）
input[number] → .value（string！）
input[checkbox] → .checked（boolean）
input[radio] → querySelector(':checked').value
select → .value（選択中の値）
⚠ input[number] の .value は必ず文字列
計算前に必ず parseFloat() か parseInt() で変換する

Ch.3 — スタイル操作

classList でスタイルを動的に切り替える

検査値に応じて警告スタイルを切り替える
/* CSS（あらかじめ定義しておく） */ .result-good { color: #4ade80; font-weight: 700; } .result-warn { color: #fbbf24; font-weight: 700; } .result-alert { color: #f87171; font-weight: 700; } .hidden { display: none; } // JS でクラスを切り替える function applyStyle(el, hba1c) { // まず全部外す el.classList.remove("result-good", "result-warn", "result-alert"); // 値に応じて付ける if (hba1c >= 8.0) el.classList.add("result-alert"); else if (hba1c >= 7.0) el.classList.add("result-warn"); else el.classList.add("result-good"); } // toggle：あれば外す・なければ付ける panel.classList.toggle("hidden"); // 表示・非表示を切り替え
💡 style直書きより classList が推奨
スタイルは CSS で管理、制御は JS で切り替えるだけ。保守しやすい
✅ よく使うパターン
classList.toggle("active") でメニュー開閉
classList.remove(...all) → add(one) で排他切り替え
Ch.4 — 全体構造

HTML + JS の連携パターン ― 正しい書き方

推奨パターン：HTML → JS の順で書く
<!-- HTML（構造）--> <body> <input id="hba1c" type="number" step="0.1" placeholder="HbA1c"> <button id="judgeBtn">判定</button> <p id="result"></p> <!-- JS は </body> 直前に置く（DOMが先に読み込まれるため）--> <script> // ① 要素を取得 const inp = document.getElementById("hba1c"); const btn = document.getElementById("judgeBtn"); const out = document.getElementById("result"); // ② ロジック関数を定義（Vol.4 の純粋関数） const judge = v => v >= 7.0 ? "🔴 要管理" : "✅ 良好"; // ③ イベントリスナーで繋ぐ btn.addEventListener("click", () => { const v = parseFloat(inp.value); out.textContent = isNaN(v) ? "数値を入力" : judge(v); }); </script> </body>
🔑 3段構成の鉄則
① 要素を取得
getElementById / querySelector

② ロジック関数を定義
純粋関数（Vol.4）でビジネスロジック

③ イベントで繋ぐ
addEventListener で ① と ② を繋ぐ
⚠ script は body の末尾に
head に書くと DOM が読み込まれる前に実行されて null エラーになる
取得 → ロジック → イベント接続。この3段が Web ツールの骨格

Ch.3 応用

イベント委任 ― 動的に追加した要素にもイベントを効かせる

動的リストへのイベント登録
// ❌ 問題：appendChild で追加した要素にはリスナーが効かない document.querySelectorAll(".patient-card") .forEach(el => el.addEventListener("click", handler)); // 後から追加したカードにはリスナーが付いていない！ // ✅ 解決：親要素でリスンして子を判別する（イベント委任） const list = document.getElementById("patient-list"); list.addEventListener("click", (e) => { // e.target：実際にクリックされた要素 if (e.target.classList.contains("patient-card")) { console.log(`${e.target.textContent} を選択`); } if (e.target.classList.contains("delete-btn")) { e.target.closest(".patient-card").remove(); } }); // 後から追加したカードにも自動で効く！
💡 e.target vs e.currentTarget
e.target：実際にクリックされた要素
e.currentTarget：リスナーが登録された要素（list）
委任では e.target を使う
✅ e.target.closest()
.closest(".card") は自分か祖先で最初に条件に一致する要素を返す。「削除ボタン」の親カードを取得するのに便利
よくあるミスと対処

DOM・イベントでハマりがちな落とし穴

① script の位置（最頻出エラー）
<head> <script> // ❌ DOMが読まれる前に実行 → null エラー document.getElementById("btn").addEventListener(…); </script> </head> <body> <button id="btn">…</button> <script> // ✅ body末尾ならDOMは読み込み済み document.getElementById("btn").addEventListener(…); </script> </body>
② input の値は必ず文字列 → 変換忘れ
const v = document.getElementById("hba1c").value; // ❌ "7.2" + 1 = "7.21" （文字列結合になる！） console.log(v + 1); // ✅ parseFloat してから計算 console.log(parseFloat(v) + 1); // 8.2
③ innerHTML に直接ユーザー入力を渡す
// ❌ XSSリスク！ユーザーがスクリプトを注入できる out.innerHTML = userInput; // ✅ テキストだけなら textContent を使う out.textContent = userInput;
④ null チェック忘れ
// ❌ id が存在しない場合 document.getElementById("xyz") .textContent = "…"; // TypeError: null // ✅ 取得後に存在チェック const el = document.getElementById("xyz"); if (el) el.textContent = "…";
