QUIZ 1 ― ウォームアップ

条件分岐って何に使う？

「プログラムの中で条件分岐（if 文）が一番使われる場面はどれだと思う？」
正解は D ― if 文はあらゆるプログラムの基礎中の基礎
Ch.1 — if / else

if / else if / else ― 条件分岐の基本形

HbA1c 判定 ― if / else if / else
const hba1c = 7.8; if (hba1c >= 10.0) { console.log("⚠ 重篤：即時介入が必要"); } else if (hba1c >= 8.0) { console.log("🔴 不良：治療強化を検討"); } else if (hba1c >= 7.0) { console.log("🟡 要管理：生活指導を継続"); } else { console.log("✅ 良好：現在の治療を継続"); } // hba1c = 7.8 なので → "🟡 要管理：生活指導を継続"
🔑 構造のポイント
  • if の条件が true → そのブロックを実行して終了
  • 上から順に評価し、最初に true になった箇所だけ実行
  • else は「それ以外すべて」のキャッチオール
  • 条件は { } のブロックで囲む
⚠ よくあるミス
// = は代入、== は比較（使わない）、=== が正解 if (hba1c = 7.0) { // ❌ 代入になってしまう！ if (hba1c == 7.0) { // ⚠ 動くが推奨しない if (hba1c === 7.0) { // ✅ 厳格比較
Ch.1 — 三項演算子

三項演算子 ― 1行で書ける if / else

構文：条件 ? 真のとき : 偽のとき
const hba1c = 7.2; // if / else で書くと ↓ let status; if (hba1c >= 7.0) { status = "要管理"; } else { status = "良好"; } // 三項演算子で書くと ↓（同じ意味） const status2 = hba1c >= 7.0 ? "要管理" : "良好"; console.log(`HbA1c ${hba1c}% → ${status2}`); // → "HbA1c 7.2% → 要管理"
ネスト（入れ子）― 多段階判定
const bmi = 28; const grade = bmi >= 35 ? "高度肥満" : bmi >= 30 ? "肥満2度" : bmi >= 25 ? "肥満1度" : bmi >= 18.5 ? "普通体重" : "低体重"; // bmi=28 → "肥満1度"
🔑 使い分けの目安
  • 三項演算子：シンプルな2択・値を変数に代入したいとき
  • if / else：3択以上・処理が複数行・読みやすさ優先
  • ネストしすぎると読みにくくなる → 3段以上は if / else に
💡 テンプレートリテラルと相性抜群
`結果：${bmi >= 25 ? "肥満" : "正常"}`
"結果：肥満1度"
三項演算子は「値の選択」に使う ― 処理の分岐には if / else を使う
Ch.1 — 複合条件

複合条件 ― && と || で条件を組み合わせる

生活習慣病リスク判定 ― 複数条件
const bmi = 27; const hba1c = 6.8; const smoking = true; const age = 58; // && ： 全部 true のとき true if (bmi >= 25 && hba1c >= 6.5 && age >= 50) { console.log("🔴 高リスク：精密検査を推奨"); } // || ： どれか1つでも true なら true if (hba1c >= 10 || bmi >= 35) { console.log("🚨 即時介入が必要"); } // ! ： 真偽を反転 if (!smoking) { console.log("非喫煙者"); } else { console.log("喫煙あり：禁煙指導を検討"); // ← これが出力 }
💡 短絡評価（Short-circuit）
A && B：A が false なら B は評価しない
A || B：A が true なら B は評価しない
処理速度の最適化と副作用防止に使われる
⚠ 括弧で優先順位を明示する
(bmi>=25 && age>=50) || smoking
複雑な条件は ( ) で意図を明確に
Ch.1 — switch

switch ― 値が一致したケースを実行する

血液型判定 ― switch の典型例
const bloodType = "A"; switch (bloodType) { case "A": console.log("A型：A または O 型が輸血可"); break; // ← break を忘れると次の case も実行される！ case "B": console.log("B型：B または O 型が輸血可"); break; case "AB": console.log("AB型：全血液型から輸血可能"); break; case "O": console.log("O型：O型からのみ輸血可"); break; default: // どれにも一致しない場合 console.log("不明な血液型"); }
🔑 switch の使いどころ
  • 同じ変数を複数の固定値と比較するとき
  • if / else if が4つ以上続くなら switch を検討
  • break を忘れると次のcaseも実行される
  • default はどれにも一致しない場合のフォールバック
⚠ fall-through の罠
case "A": console.log("A型"); // break を忘れると↓も実行！ case "B": console.log("B型"); // A型でも出る

function judgeHbA1c(val) { // parseFloat で文字列→数値（Vol.1） const v = parseFloat(val); // NaN チェック（Vol.1） if (isNaN(v)) return "⚠ 数値を入力してください"; // 多段階 if / else if（今日） if (v >= 10.0) return "🚨 重篤（≥10.0）：即時介入"; else if (v >= 8.0) return "🔴 不良（8.0〜9.9）：治療強化"; else if (v >= 7.0) return "🟡 要管理（7.0〜7.9）"; else if (v >= 6.0) return "🟢 ほぼ良好（6.0〜6.9）"; else return "✅ 良好（<6.0）"; }

Ch.2 — for ループ

for ループ ― 決まった回数を繰り返す

構文：for (初期化; 条件; 更新)
// ①初期化 ②条件 ③更新 for (let i = 0; i < 5; i++) { console.log(`患者番号 ${i + 1} を処理中`); } // → 患者番号 1 を処理中 // → 患者番号 2 を処理中 // → 患者番号 3 を処理中 // → 患者番号 4 を処理中 // → 患者番号 5 を処理中
for × if の組み合わせ
const hba1cList = [6.5, 7.8, 5.9, 9.2, 6.8]; let alertCount = 0; for (let i = 0; i < hba1cList.length; i++) { if (hba1cList[i] >= 7.0) { alertCount++; console.log(`⚠ 患者${i+1}: HbA1c ${hba1cList[i]}%`); } } console.log(`要管理: ${alertCount} 名`); // → "要管理: 2 名"
🔑 for ループの3つの部分
let i = 0　　① 開始値（i を 0 に）
i < 5　　　 ② 継続条件（i が 5 未満の間）
i++　　　　 ③ 更新（毎回 i を 1 増やす）
💡 .length で配列の長さを取得
[6.5, 7.8, 5.9].length
3
配列の要素数 = ループの回数
Ch.2 — while

while ― 条件が true の間ずっと繰り返す

while ― 薬の残数カウントダウン
let pills = 5; // 残薬 5 錠 while (pills > 0) { console.log(`💊 服薬 → 残り ${pills - 1} 錠`); pills--; // ← これがないと無限ループ！ } console.log("薬がなくなりました。処方を依頼してください"); // → 💊 服薬 → 残り 4 錠 // → 💊 服薬 → 残り 3 錠 ... (省略) // → 薬がなくなりました。処方を依頼してください
for vs while ― 使い分け
// 繰り返し回数が決まっている → for for (let i = 0; i < 10; i++) { … } // 条件が満たされるまで → while while (血圧 > 140) { // 投薬・再測定を繰り返す }
⚠ 無限ループに注意
while (true) { // 更新がないと永遠に回る // ブラウザがフリーズする！ }
必ず条件が false になる更新を忘れずに
💡 do...while（後判定）
do { // 最低1回は必ず実行 } while (条件);
「問診票を最低1回は表示する」などに使う
for → 回数決まってる、while → 条件で終わる
Ch.2 — for...of

for...of ― 配列の要素を順に取り出す

for vs for...of の比較
const patients = ["山田", "鈴木", "田中", "佐藤"]; // 旧来の for ループ（インデックスが必要なとき） for (let i = 0; i < patients.length; i++) { console.log(i, patients[i]); } // for...of （要素だけ取り出す、シンプル） for (const name of patients) { console.log(`患者：${name}さん`); } // → 患者：山田さん / 鈴木さん / 田中さん / 佐藤さん // インデックスも欲しいとき → entries() for (const [i, name] of patients.entries()) { console.log(`${i+1}番：${name}`); }
💡 使い分けの目安
for (let i=0...)：インデックスが必要 / 逆順 / 飛ばしたい
for...of：全要素を順に処理するだけ（こちらが読みやすい）
✅ Vol.3（次回）で学ぶ配列メソッド
.forEach()・.map()・.filter() を使うとさらに簡潔になる
Ch.2 — break / continue

break と continue ― ループを制御する

break ― ループを途中で抜ける
const hba1cList = [6.2, 6.8, 9.5, 7.1, 8.0]; // 重篤な患者を見つけたら即座に通知して終了 for (const v of hba1cList) { if (v >= 9.0) { console.log(`🚨 緊急 HbA1c ${v} → ループ終了`); break; // ← ここでループを抜ける } console.log(`✅ ${v} 正常範囲`); } // 6.2, 6.8 は表示 → 9.5 で break
continue ― その回だけスキップ
const data = [6.5, null, 7.2, null, 8.1]; // null（未記入）をスキップして処理 for (const v of data) { if (v === null) continue; // スキップ console.log(`処理: HbA1c ${v}`); } // → 6.5 / 7.2 / 8.1 のみ処理（null は飛ばす）
🔑 まとめ
break　　ループ全体を終了
continue　その回だけスキップして次へ
return　　関数ごと終了（Vol.4 で詳しく）
🩺 医療データ処理での使い方
  • break：緊急度の高い患者を発見したら即通知
  • continue：欠損値（null / undefined）をスキップ
  • continue：測定範囲外の値を除外して集計

Ch.3 — 組み合わせ

条件分岐 × ループ ― 患者データを集計する

5人の患者データを分析する
const patients = [ { name: "山田", hba1c: 6.5, bmi: 24 }, { name: "鈴木", hba1c: 8.2, bmi: 28 }, { name: "田中", hba1c: 5.9, bmi: 22 }, { name: "佐藤", hba1c: 9.5, bmi: 31 }, { name: "伊藤", hba1c: 7.1, bmi: 26 }, ]; let alertList = []; // 要管理患者の名前リスト let totalHbA1c = 0; // 合計（平均を計算するため） for (const p of patients) { totalHbA1c += p.hba1c; if (p.hba1c >= 7.0 || p.bmi >= 30) { alertList.push(p.name); } } const avg = (totalHbA1c / patients.length).toFixed(1); console.log(`平均 HbA1c: ${avg}%`); // "7.4%" console.log(`要管理: ${alertList.join("・")}`); // "鈴木・佐藤・伊藤"
📌 新しく登場した概念
  • { name, hba1c, bmi }：オブジェクト（Vol.3 で詳しく）
  • p.hba1c：ドット記法でプロパティを取得
  • [].push()：配列に要素を追加
  • .join("・")：配列を文字列に変換

Ch.3 — ネスト

ネストしたループ ― ループの中にループ

週 × 曜日 ― 2重ループ
const weeks = ["第1週", "第2週"]; const weekdays = ["月", "水", "金"]; // 外来診察日 for (const week of weeks) { for (const day of weekdays) { console.log(`${week} ${day}曜日 外来`); } } // → 第1週 月曜日 外来 // → 第1週 水曜日 外来 // → 第1週 金曜日 外来 // → 第2週 月曜日 外来 ...
⚠ ネストは2〜3重まで
深くなるほど読みにくく処理も遅くなる。
4重以上は関数に切り出す（Vol.4 で学ぶ）
💡 計算量に注意
外側 n 回 × 内側 m 回 = n × m 回実行される
1000×1000 = 100万回 → ブラウザが遅くなる
ネストは便利だが、深さには要注意
よくあるミスと対処

ループ・分岐でハマりがちなミス

① off-by-one（境界値ミス）
const arr = ["a", "b", "c"]; // 要素は 0,1,2 // ❌ i <= arr.length は範囲外アクセス for (let i=0; i <= arr.length; i++) { console.log(arr[i]); // 最後は undefined } // ✅ i < arr.length が正解 for (let i=0; i < arr.length; i++) { … }
② 条件の = vs ==/ === の混同
let flag = false; // ❌ = は代入。条件が常に true になる if (flag = true) { console.log("常に実行"); } // ✅ === で比較する if (flag === true) { console.log("正しい比較"); }
③ switch の break 忘れ
case "A": console.log("A型"); // break を書かないと case "B": console.log("B型"); // ← A型でも出る
④ while の無限ループ
let i = 0; while (i < 5) { console.log(i); // i++ を忘れると無限ループ！ }
エラーが出たら「境界値・= vs ===・break・更新処理」を確認
実践パターン

よく使うパターン ― コピペして使える雛形

医療でよく使うパターン集
// ① 配列の合計・平均 const sum = values.reduce((a, b) => a + b, 0); const avg = sum / values.length; // ② 配列から条件に合う要素を取り出す（filter） const highRisk = patients.filter(p => p.hba1c >= 7.0); // ③ 最大値・最小値 const maxHbA1c = Math.max(...values); const minHbA1c = Math.min(...values); // ④ 特定の要素が存在するか const hasCritical = patients.some(p => p.hba1c >= 10); // ⑤ 全員が条件を満たすか const allGood = patients.every(p => p.hba1c < 7.0);
💡 filter・some・every・reduce は配列メソッド。for ループを内部で持ち、1行で処理できる。Vol.3 で詳しく学ぶ
