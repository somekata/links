Ch.1 — 配列の基本

配列 ― データを順番に並べる「箱の列」

配列の作成・アクセス・基本操作
// 作成：[ ] でくくる const hba1cList = [6.5, 7.8, 5.9, 9.2, 6.8]; // アクセス：0始まりのインデックス console.log(hba1cList[0]); // 6.5（先頭） console.log(hba1cList[4]); // 6.8（末尾） console.log(hba1cList.length); // 5（要素数） console.log(hba1cList.at(-1)); // 6.8（末尾を-1で取得） // 追加・削除 hba1cList.push(7.1); // 末尾に追加 → [6.5,7.8,5.9,9.2,6.8,7.1] hba1cList.pop(); // 末尾を削除 → [6.5,7.8,5.9,9.2,6.8] hba1cList.unshift(8.0); // 先頭に追加 hba1cList.shift(); // 先頭を削除 // 一部を取り出す（元の配列は変化しない） const top3 = hba1cList.slice(0, 3); // [6.5, 7.8, 5.9]
🔑 インデックスは 0 始まり
index: 0　　1　　2　　3　　4
value: [6.5, 7.8, 5.9, 9.2, 6.8]
length=5 だが最後のインデックスは 4
arr[arr.length] は undefined になる
💡 push vs unshift の速度
  • push（末尾追加）→ 高速
  • unshift（先頭追加）→ 全要素をずらすので遅い
  • 大量データでは push を優先する

Ch.1 — 配列メソッド

map / filter / reduce ― 配列を加工・集計する

患者の HbA1c データを加工する
const values = [6.5, 7.8, 5.9, 9.2, 6.8]; // map：全要素を変換した新しい配列を返す const labels = values.map(v => v >= 7.0 ? `${v} 🔴` : `${v} ✅` ); // ["6.5 ✅","7.8 🔴","5.9 ✅","9.2 🔴","6.8 ✅"] // filter：条件を満たす要素だけを取り出す const highRisk = values.filter(v => v >= 7.0); // [7.8, 9.2] // reduce：全要素を1つの値に集約する const sum = values.reduce((acc, v) => acc + v, 0); const avg = (sum / values.length).toFixed(1); // sum=36.2, avg="7.2" // メソッドチェーン：連続して使える const highAvg = values .filter(v => v >= 7.0) .reduce((a,b) => a+b, 0) / 2; // 高リスク患者の平均: (7.8+9.2)/2 = 8.5
🔑 3メソッドの違い
map　　→ 同じ長さの新配列
filter → 短い新配列（条件一致のみ）
reduce → 1つの値（合計・最大・文字列等）
💡 元の配列は変化しない
map/filter/reduce は非破壊的
新しい配列を返す。元データは安全
⇒ アロー関数 v => ...
.map(v => v * 2) の v => ... は
関数の省略形。Vol.4 で詳しく学ぶ
Ch.1 — その他の配列メソッド

find / some / every / sort ― 検索・判定・整列

患者データでの使い方
const vals = [6.5, 7.8, 5.9, 9.2, 6.8]; // find：条件を満たす最初の要素を返す（なければ undefined） const firstHigh = vals.find(v => v >= 9.0); // 9.2 // findIndex：条件を満たす最初のインデックスを返す const idx = vals.findIndex(v => v >= 9.0); // 3 // some：1件でも条件を満たすか（true/false） const hasCritical = vals.some(v => v >= 10); // false // every：全件が条件を満たすか（true/false） const allGood = vals.every(v => v < 7.0); // false（7.8, 9.2 あり） // sort：並び替え（数値は比較関数が必要） const sorted = [...vals].sort((a,b) => a - b); // 昇順 // [5.9, 6.5, 6.8, 7.8, 9.2] // includes：値が含まれるか const has78 = vals.includes(7.8); // true
⚠ sort() は文字列ソートがデフォルト：数値を並べ替えるには (a,b) => a-b の比較関数が必要。[10,9,2].sort() → [10,2,9]（文字列順になる）

Ch.2 — オブジェクト

オブジェクト ― 名前付きデータの集合

患者オブジェクトの作成・操作
// オブジェクト：{ キー: 値 } の形 const patient = { id: 1001, name: "山田太郎", age: 65, hba1c: 7.2, bmi: 24.5, isDiabetic: true, }; // アクセス方法 ①：ドット記法（推奨） console.log(patient.name); // "山田太郎" console.log(patient.hba1c); // 7.2 // アクセス方法 ②：ブラケット記法（キーが変数のとき） const key = "age"; console.log(patient[key]); // 65 // プロパティの追加・更新・削除 patient.lastVisit = "2025-04"; // 追加 patient.hba1c = 7.5; // 更新 delete patient.isDiabetic; // 削除
🔑 配列 vs オブジェクト
配列：順序が重要・インデックスでアクセス
オブジェクト：名前が重要・キーでアクセス

「HbA1cの測定値リスト」→ 配列
「患者の情報まとめ」→ オブジェクト

Object.keys(patient)

Ch.2 — 分割代入・スプレッド

分割代入 と スプレッド構文 ― 現代的な書き方

分割代入（Destructuring）― オブジェクト・配列から一気に取り出す
const patient = { name:"山田", age:65, hba1c:7.2, bmi:24 }; // 通常の取り出し方（冗長） const name = patient.name; const hba1c = patient.hba1c; // 分割代入（スッキリ） const { name, age, hba1c } = patient; console.log(`${name}さん（${age}歳）HbA1c: ${hba1c}`); // 配列の分割代入 const [first, second, ...rest] = [6.5, 7.8, 5.9, 9.2]; // first=6.5, second=7.8, rest=[5.9, 9.2]
スプレッド構文（...）― コピー・マージ
// オブジェクトのコピー（元を変えずに更新） const updated = { ...patient, hba1c: 7.5 }; // hba1c だけ上書き // 配列のコピー・結合 const copy = [...vals]; // 配列の浅いコピー const merged = [...vals1, ...vals2]; // 2配列を結合 // Math.max に配列を渡す const max = Math.max(...vals); // 9.2
分割代入とスプレッドは「読みやすいコード」のための現代的構文
Ch.2 — オブジェクト配列

オブジェクト配列 ― 患者リストを本格的に扱う

map / filter / sort をオブジェクト配列に適用
const patients = [ { name:"山田", hba1c:6.5, bmi:24 }, { name:"鈴木", hba1c:8.2, bmi:28 }, { name:"田中", hba1c:5.9, bmi:22 }, { name:"佐藤", hba1c:9.5, bmi:31 }, ]; // map：全患者にリスクラベルを追加 const withLabel = patients.map(p => ({ ...p, risk: p.hba1c >= 7.0 ? "高" : "低" })); // filter：要管理患者だけ抽出 const alerts = patients.filter( p => p.hba1c >= 7.0 || p.bmi >= 30 ); // [鈴木, 佐藤] // sort：HbA1c 降順で並べ替え const ranked = [...patients] .sort((a,b) => b.hba1c - a.hba1c); // 佐藤(9.5), 鈴木(8.2), 山田(6.5), 田中(5.9) // reduce：平均 HbA1c const avgHbA1c = (patients .reduce((s,p) => s + p.hba1c, 0) / patients.length).toFixed(1); // "7.5"
✅ これで何ができるか
  • 100人の患者データを
    1行で要管理者だけ抽出
  • 全患者にリスクスコアを付与
  • HbA1c 順でランキング表示
  • グループ別平均の計算
💡 map の中のスプレッド
{ ...p, risk: "高" }
元のプロパティを全部コピーしつつ risk を追加。元オブジェクトは変化しない

Ch.3 — JSON

JSON ― データを「文字列」として持ち運ぶ

stringify（JS→文字列）と parse（文字列→JS）
const patient = { name: "山田太郎", age: 65, hba1c: 7.2 }; // JS オブジェクト → JSON 文字列 const json = JSON.stringify(patient); console.log(json); // '{"name":"山田太郎","age":65,"hba1c":7.2}' // 見やすく整形（インデント付き） console.log(JSON.stringify(patient, null, 2)); // { // "name": "山田太郎", // "age": 65, // "hba1c": 7.2 // } // JSON 文字列 → JS オブジェクト（API から受け取ったデータ） const received = '{"name":"鈴木","hba1c":8.2}'; const obj = JSON.parse(received); console.log(obj.name); // "鈴木"
🔑 JSON が使われる場面
  • APIからデータを受け取るとき（→ parse）
  • データをサーバーに送るとき（→ stringify）
  • localStorage に保存するとき
  • 設定ファイル（package.json 等）
⚠ JSON の制約
  • キーは必ず ""（ダブルクォート）
  • 関数・undefined・Symbol は含めない
  • コメントが書けない
  • parse 失敗時は SyntaxError → try/catch で捕まえる
JSONはJS・Python・R・SQL すべてが読める「共通語」

よくあるミスと注意点

配列・オブジェクトでハマりがちな落とし穴

① 参照渡しの罠（浅いコピー）
const a = { name: "山田", hba1c: 6.5 }; const b = a; // ❌ コピーでなく同じ参照 b.hba1c = 9.0; console.log(a.hba1c); // 9.0 ← a も変わってしまう！ // ✅ スプレッドでコピーする const c = { ...a }; // 独立したコピー c.hba1c = 9.0; console.log(a.hba1c); // 6.5 ← a は変わらない
② sort() の罠 ＋ map の返り値忘れ
// sort はデフォルトで文字列比較 [10, 9, 2, 100].sort(); // [10,100,2,9] ← 意図しない [10, 9, 2, 100].sort((a,b)=>a-b); // [2,9,10,100] ← 正解 // map で return を忘れるとすべて undefined const bad = [1,2,3].map(n => { n * 2 }); // [undef,undef,undef] const ok = [1,2,3].map(n => n * 2 ); // [2,4,6]
③ JSON.parse のエラー処理
// 不正な JSON は SyntaxError try { const d = JSON.parse("壊れたJSON"); } catch (e) { console.error("parse失敗", e); }
④ const 配列への push は可能
const arr = [1, 2, 3]; arr.push(4); // ✅ これは OK arr = [5]; // ❌ 再代入はダメ // const は「参照」を固定するため
実践パターン

医療データ処理のパターン集

コピペして使える雛形集
const pts = [/* オブジェクト配列 */]; // ① 要管理患者を抽出して名前リスト化 const alertNames = pts .filter(p => p.hba1c >= 7.0) .map(p => p.name); // ② 全患者の平均 HbA1c const avg = pts .reduce((s, p) => s + p.hba1c, 0) / pts.length; // ③ HbA1c が最大の患者 const worst = pts.reduce( (max, p) => p.hba1c > max.hba1c ? p : max ); // ④ リスクラベルを全員に付与して JSON 出力 const report = JSON.stringify( pts.map(p => ({ ...p, risk: p.hba1c >= 7.0 ? "高" : "低" })), null, 2 );
filter → map のチェーンは「絞る→変換」の黄金パターン
