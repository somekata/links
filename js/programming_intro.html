Chapter 1 解説

2種類の言語：インタープリタ vs コンパイラ

🔄 インタープリタ型
  • コードを書いたらすぐ実行できる
  • 翻訳しながら動く（その場で解釈）
  • 起動が速い・試しやすい
  • 代表：JavaScript・Python・R
ブラウザの開発者ツールで即実行
> 1 + 1
← 2
vs
⚙️ コンパイル型
  • 実行前に機械語に「翻訳」が必要
  • 翻訳済みなので動作が非常に速い
  • 型・構造を厳密に宣言する必要あり
  • 代表：C・C++・C#
コンパイル → 実行ファイル生成
$ gcc hello.c -o hello
$ ./hello
医学部生が最初に学ぶなら→インタープリタ型がおすすめ

Chapter 2

共通概念②　条件分岐 ― if / else

お題：HbA1c 7.0以上なら「要管理」
JavaScript
const hba1c = 7.2; if (hba1c >= 7.0) { console.log("要管理"); } else { console.log("良好"); }
R
hba1c <- 7.2 if (hba1c >= 7.0) { print("要管理") } else { print("良好") }
Python（比較参照）
Python
hba1c = 7.2 if hba1c >= 7.0: print("要管理") else: print("良好") # ※ {} の代わりにインデント（字下げ）で構造を表す
🔑 3言語の共通点と違い
  • if / else という構造はどれも同じ
  • JS・R はブロックを { } で囲む
  • Python はインデント（字下げ）が文法
  • R の代入は <-、JS は const/let
Chapter 2

共通概念③　ループ ― 繰り返し処理

お題：5人の患者名を順に表示する
JavaScript
const patients = [ "山田", "鈴木", "田中", "佐藤", "伊藤" ]; for (const name of patients) { console.log(name); } // → 山田、鈴木、田中… と順に表示
R
patients <- c("山田", "鈴木", "田中", "佐藤", "伊藤") for (name in patients) { print(name) } # R のベクトルは c() で作る
Python（比較参照）
Python
patients = [ "山田", "鈴木", "田中", "佐藤", "伊藤" ] for name in patients: print(name)
🔑 Rが特に得意なこと
  • Rはループよりベクトル演算が本領
  • patients[1:3] で先頭3人を一気に取得
  • 100人分のBMIを一行で計算できる
  • forループを書かなくて済む場面が多い
「繰り返し作業の自動化」はプログラミングの最強の武器
Chapter 2

共通概念④　関数 ― 処理をまとめて再利用

お題：BMI を計算する関数を作る
JavaScript
function calcBMI(weight, height) { // height は メートル単位 return weight / (height ** 2); } console.log( calcBMI(70, 1.75) ); // → 22.857…
R
calcBMI <- function(weight, height) { weight / height^2 # Rは最後の評価式が自動的に返り値になる } calcBMI(70, 1.75) # → 22.857… # ベクトル化：一度に複数患者を計算 w <- c(70, 55, 90) h <- c(1.75, 1.60, 1.80) calcBMI(w, h) # 3人分まとめて計算！
Python（比較参照）
Python
def calc_bmi(weight, height): return weight / (height ** 2) print( calc_bmi(70, 1.75) ) # → 22.857…
🔑 JS と R の関数の違い
  • JS：function キーワード + return
  • R：function() を変数に代入、returnは省略可
  • R最大の強み：関数にベクトルを渡すと全要素に自動適用
  • 100人分も1行で処理できる = 医学研究向き
💡 エラーは敵ではなく道しるべ
Error: could not find function → タイポを疑う
unexpected '}' → 括弧の対応を確認

Chapter 4 — R

R でデータ解析 ― 一気通貫の例

R — 糖尿病患者データ解析
# ── 1. パッケージ読み込み ────────── library(tidyverse) # ── 2. データ作成（架空の患者10名）── df <- tibble( id = 1:10, group = c(rep("治療群", 5), rep("対照群", 5)), hba1c = c(6.8, 7.1, 6.5, 7.3, 6.9, 8.1, 8.5, 7.9, 8.3, 8.0) ) # ── 3. グループ別平均 ───────────── df |> group_by(group) |> summarise(mean_hba1c = mean(hba1c)) # ── 4. t 検定 ───────────────────── t.test(hba1c ~ group, data = df) # ── 5. ggplot2 で箱ひげ図 ───────── ggplot(df, aes(x=group, y=hba1c, fill=group)) + geom_boxplot() + labs(title="HbA1c 治療比較", y="HbA1c (%)") + theme_bw()
📌 読み解きポイント
  • tibble()：Rのデータフレーム（Excelのシートに相当）
  • |>：パイプ演算子。前の結果を次の関数に渡す
  • group_by → summarise：グループ集計（SQL の GROUP BY と同じ発想）
  • t.test()：t検定が1行で完結
  • ggplot2：レイヤーを + でつなぐ独自文法
この5ステップで何ができるか

① データ整形
② 記述統計
③ 統計検定
④ 論文用グラフ生成

ExcelでやるとVBA必須の作業が
Rなら10行以下で書ける

Chapter 5 — SQL

SQL ― 電子カルテは全部データベース

SQL
-- HbA1c > 7.0 の患者を抽出 SELECT patient_id, name, hba1c FROM patients WHERE hba1c > 7.0; -- 診断ごとの平均 HbA1c を集計 SELECT diagnosis, AVG(hba1c) AS mean_hba1c, COUNT(*) AS n FROM patients GROUP BY diagnosis ORDER BY mean_hba1c DESC; -- 処方テーブルと結合 SELECT p.name, p.hba1c, rx.drug FROM patients p JOIN prescriptions rx ON p.patient_id = rx.patient_id WHERE rx.drug = 'メトホルミン';
📌 SQL の4大命令
SELECT ── 読む（最重要）
WHERE　 ── 条件を絞る
GROUP BY ── 集計する
JOIN　　 ── テーブルを結合する
