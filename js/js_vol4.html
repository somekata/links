Ch.1 — 関数とは

関数 ― 処理に「名前」をつけて再利用する

なぜ必要？
→
同じ処理を何度も書かずに済む。バグを直すのも1か所だけ。コードが読みやすくなる
医療の例
→
「BMI計算」「HbA1c判定」「リスクスコア計算」を関数にすれば、何人の患者データにも使い回せる
アナロジー
→
関数 = 「クリニカルパス」。入力（患者情報）を受け取り → 処理して → 結果（判断・指示）を返す
関数の3要素：入力（引数）・処理・出力（返り値）
入力 処理 出力 ↓ ↓ ↓ function calcBMI(weight, height) { const bmi = weight / (height ** 2); return bmi.toFixed(1); // ← return で結果を返す } calcBMI(70, 1.75); // → "22.9" calcBMI(55, 1.60); // → "21.5"（同じ関数を使い回す）
Ch.1 — function 宣言

function 宣言 ― 基本の書き方と細かいルール

引数・返り値・デフォルト引数・早期 return
// ① 基本形 function greet(name) { return `こんにちは、${name}さん`; } // ② デフォルト引数（引数が省略されたときの初期値） function judgeHbA1c(value, threshold = 7.0) { return value >= threshold ? "要管理" : "良好"; } judgeHbA1c(7.2); // "要管理"（threshold=7.0） judgeHbA1c(7.2, 8.0); // "良好"（threshold=8.0） // ③ 早期 return（バリデーション） function calcBMI(weight, height) { if (isNaN(weight) || isNaN(height) || height <= 0) { return null; // 不正な入力 → 即終了 } return weight / (height ** 2); } // ④ return がない関数 → undefined を返す function logResult(val) { console.log(`結果: ${val}`); // 副作用のみ }
🔑 return のルール
  • return に達した時点で関数が終了する
  • 返り値がない場合は undefined が返る
  • 複数の return を持てる（早期 return）
  • return;（値なし）も書ける
💡 巻き上げ（Hoisting）
// function宣言は宣言前に呼べる greet("山田"); // ✅ エラーにならない function greet(n) { return `こんにちは`; }
Ch.2 — アロー関数

アロー関数 => ― 関数の省略記法

function → アロー関数 への変換
// ❶ function 宣言（長い） function double(n) { return n * 2; } // ❷ 関数式（変数に代入） const double = function(n) { return n * 2; }; // ❸ アロー関数（{ } と return を省略） const double = (n) => { return n * 2; }; // ❹ 式が1つなら { } と return も省略 const double = n => n * 2; // ← 最短形 // 引数が0個や2個以上のとき ( ) は必要 const greet = () => "こんにちは"; const add = (a, b) => a + b;
🔑 省略ルールまとめ
引数が1つ → ( ) 省略可
本体が1式 → { } と return 省略可
オブジェクトを返すとき → ( ) でくくる
n => ({ bmi: n })
💡 map に渡す関数 ← これが正体
// この v => ... はアロー関数！ [6.5, 7.8].map(v => v >= 7 ? "🔴":"✅") // 同じ意味： [6.5, 7.8].map(function(v) { return v >= 7 ? "🔴" : "✅"; });
アロー関数 = 「使い捨ての小さな関数」を書くための省略記法
Ch.2 — 高階関数

関数は「値」― 変数に入れて、引数に渡す

関数を引数に渡す ― コールバック関数
// 関数を変数に代入できる const isHighRisk = p => p.hba1c >= 7.0; const getName = p => p.name; const addLabel = p => ({ ...p, risk: p.hba1c >= 7.0 ? "高" : "低" }); // 関数を引数として渡す（コールバック） const patients = [ { name:"山田", hba1c:6.5 }, { name:"鈴木", hba1c:8.2 }, ]; const alerts = patients.filter(isHighRisk); // [鈴木] const names = patients.map(getName); // ["山田","鈴木"] const labeled = patients.map(addLabel); // リスクラベル付き // 関数を返す関数（高階関数） function makeThreshold(threshold) { return v => v >= threshold; // 関数を返す！ } const isOver7 = makeThreshold(7.0); const isOver10 = makeThreshold(10.0); patients.filter(isOver7); // HbA1c ≥ 7.0 の患者
💡 高階関数：「関数を引数に受け取る」か「関数を返す」関数。map / filter / reduce はすべて高階関数。自分で関数を渡せると設計の幅が広がる

関数の動き可視化 ― 引数→処理→返り値を追う

実行する関数
function riskScore(hba1c, bmi, age) { let score = 0; if (hba1c >= 7.0) score += 3; if (bmi >= 25) score += 2; if (age >= 50) score += 1; return score; }
▶ 値を入力して riskScore() を呼ぶ

BMI ≥ 25 → +2点
年齢 ≥ 50 → +1点

スコープ ― 変数が「見える」範囲

3種類のスコープ
// ① グローバルスコープ：どこからでも見える const HOSPITAL = "大阪公立大学病院"; // グローバル function checkPatient(hba1c) { // ② 関数スコープ：関数の中だけ見える const msg = `${HOSPITAL} 判定`; // グローバルは見える if (hba1c >= 7.0) { // ③ ブロックスコープ：{ } の中だけ（let/const） const alert = "要管理"; console.log(alert); // ✅ ここでは見える } // console.log(alert); // ❌ ブロック外 → ReferenceError } // console.log(msg); // ❌ 関数外 → ReferenceError

Ch.4 — クロージャ

クロージャ ― 関数が変数を「記憶する」

カウンター ― クロージャの典型例
// makeCounter を呼ぶたびに独立したカウンターが生まれる function makeCounter(label) { let count = 0; // この変数が「記憶される」 return function() { count++; return `${label}: ${count} 件`; }; } // 2つの独立したカウンター const visitCount = makeCounter("外来来院"); const admitCount = makeCounter("入院受付"); console.log(visitCount()); // "外来来院: 1 件" console.log(visitCount()); // "外来来院: 2 件" console.log(admitCount()); // "入院受付: 1 件"（別カウント） console.log(visitCount()); // "外来来院: 3 件"
🔑 クロージャとは
  • 内側の関数が、外側の変数を「閉じ込めて」覚え続ける
  • 外から count に直接アクセスできない → 安全
  • makeCounter を呼ぶたびに独立した count が生まれる
💡 医療での使い方
  • 患者ごとのカウンター
  • 設定値（閾値）を内包した判定関数
  • 累積スコアを持つリスク評価器
クロージャ = 「プライベートな変数」を持つ関数の仕組み
Ch.4 — クロージャ応用

設定を「記憶する」関数 ― クロージャの実践

閾値を設定で変えられる判定器
// 閾値を「記憶する」判定関数ファクトリ function makeJudge(name, threshold) { return (value) => ({ label: name, value, flag: value >= threshold, msg: value >= threshold ? `${name} ${value} ≥ ${threshold} → 🔴 要管理` : `${name} ${value} < ${threshold} → ✅ 良好` }); } // それぞれの閾値を覚えた専用関数を生成 const judgeHbA1c = makeJudge("HbA1c", 7.0); const judgeBMI = makeJudge("BMI", 25.0); const judgeBP = makeJudge("収縮期血圧", 140); console.log(judgeHbA1c(7.8).msg); // "HbA1c 7.8 ≥ 7.0 → 🔴 要管理" console.log(judgeBMI(22).msg); // "BMI 22 < 25.0 → ✅ 良好" console.log(judgeBP(155).msg); // "収縮期血圧 155 ≥ 140 → 🔴 要管理"
💡 makeJudge を一度定義すれば、どんな指標・どんな閾値にも使い回せる。これが「関数を返す関数」の力
「設定を覚えた関数を量産する」― それがクロージャの真価

よくあるミスと注意点

関数・スコープでハマりがちな落とし穴

① return を忘れる → undefined
// ❌ { } を使うと return が必要 const double = n => { n * 2; }; // undefined // ✅ return を書く const double = n => { return n * 2; }; // ✅ または { } を省略 const double = n => n * 2;
② オブジェクトを返すとき ( ) が必要
// ❌ { } をブロックと解釈してしまう const mk = n => { value: n }; // SyntaxError // ✅ ( ) でオブジェクトリテラルをくくる const mk = n => ({ value: n }); // OK
③ 関数式は巻き上げされない
greet(); // ✅ function宣言はOK function greet() { return "hello"; } bye(); // ❌ ReferenceError const bye = () => "bye";
④ 引数の数が違っても動く
function add(a, b) { return a + b; } add(1); // NaN（b=undefined） add(1,2,3); // 3（余分な引数は無視）
設計の考え方

純粋関数 ― 「同じ入力 → 同じ出力」を目指す

純粋関数 vs 副作用のある関数
// ❌ 副作用あり：外部の変数を変える（テスト困難） let totalRisk = 0; function addRisk(score) { totalRisk += score; // 外部変数を変更 ← 副作用 } // ✅ 純粋関数：引数だけを使って結果を返す（テスト容易） function calcRisk(hba1c, bmi, age) { let score = 0; if (hba1c >= 7.0) score += 3; if (bmi >= 25) score += 2; if (age >= 50) score += 1; return score; // 外に影響しない } // 純粋関数は必ず同じ結果になる → テストが簡単 console.log(calcRisk(7.2, 28, 55)); // 常に 6
💡 純粋関数のメリット
  • テストが簡単（入力→出力だけ確認）
  • 再利用しやすい
  • バグが起きにくい
  • map/filter に渡せる
⚠ 副作用が必要な場面もある
画面更新・ファイル保存・API呼び出しは副作用。
ただしコアのロジックは純粋関数で書くのがベスト
