Ch.1 — 変数

変数とは ― データに「名前」をつける

アナロジー
→
変数は「ラベル付きの箱」。中身（値）を入れておき、名前で呼び出せる。
患者の体重を毎回 70 と書く代わりに weight と名前をつける
医療例
→
患者ID・体重・HbA1c・診断名 ― すべてプログラムの中では変数として扱う
JavaScript — はじめての変数
// 変数を宣言して値を代入する let patientName = "山田太郎"; let weight = 70; let hba1c = 7.2; let isDiabetic = true; // 名前で呼び出す console.log(patientName); // → "山田太郎" console.log(weight); // → 70
変数名は「中身が何かわかる名前」にする ― これが読みやすいコードの第一歩
Ch.1 — 変数宣言

let と const ― どちらを使うか

let ― 再代入あり
let temperature = 37.0; console.log(temperature); // 37 // 値を書き換えられる temperature = 38.5; console.log(temperature); // 38.5
const ― 再代入なし（定数）
const HOSPITAL = "大阪公立大学病院"; console.log(HOSPITAL); // OK // 書き換えようとするとエラー ↓ HOSPITAL = "他の病院"; // ✗ TypeError: Assignment to constant
🔑 使い分けの鉄則
  • const をデフォルトで使う
  • 値が変わる必要があるときだけ let
  • var は古い書き方 ― 新規では使わない
💡 命名規則 ― camelCase
// ✅ 良い例（camelCase）
patientAge, bloodPressure, isHospitalized
// ❌ 悪い例
PatientAge, blood_pressure, 体温
⚠ 変数名に使えない文字：数字始まり・スペース・-（ハイフン）
使えるのは英数字・_・$
Ch.2 — データ型

データ型 ― 値には「種類」がある

number
数値（整数・小数どちらも同じ型）
let age = 25　　let hba1c = 7.2　　let pi = 3.14
string
文字列（クォートで囲む）
"山田太郎"　'糖尿病'　`HbA1c: ${hba1c}%`
boolean
真偽値（true か false の2択）
let isDiabetic = true　　let isFasting = false
null
意図的な「空」（値がないと明示する）
let nextAppointment = null　← まだ決まっていない
undefined
未定義（宣言だけして値を入れていない）
let diagnosis;　← 宣言のみ → 自動的に undefined
医療データにも「未記入（null）」と「項目なし（undefined）」は別概念 ― 型で表現できる
Ch.2 — typeof

typeof ― 型を調べる演算子

JavaScript
console.log( typeof 7.2 ); // "number" console.log( typeof "山田太郎" ); // "string" console.log( typeof true ); // "boolean" console.log( typeof null ); // "object" ← 有名なバグ！ console.log( typeof undefined ); // "undefined" const age = 25; console.log( typeof age ); // "number" // 型を確認してから処理する（実務的な使い方） if (typeof age === "number") { console.log("年齢は数値です"); }
⚠ JSの有名な罠
typeof null
"object"
null は「オブジェクト型」と表示されるが、これはJSの初期設計ミス。null を本当に確認したいときは === null で比較する
🔧 デモ：Console で試す
F12 → Console を開いて
typeof "hello" と入力 → Enter
結果が "string" と返ってくる
型を把握することがバグを防ぐ第一歩
Ch.2 — 型変換

型変換の罠 ― == と === の違い

== （緩い比較・型変換あり）
console.log( 5 == "5" ); // true ← 数値と文字列が等しい!? console.log( 0 == false ); // true ← 0とfalseが等しい!? console.log( "" == false ); // true ← 空文字とfalseが等しい!? // JSが自動で型を変換するので直感に反する結果になる
=== （厳格な比較・型変換なし）
console.log( 5 === "5" ); // false ← 型が違うので false console.log( 0 === false ); // false ← number と boolean は別 console.log( 5 === 5 ); // true ← 値も型も同じなら true // 型まで含めて比較する → 予測しやすい
❌ == は使わない
型変換が暗黙的に起きて、
バグの温床になる
✅ === を常に使う
「値も型も同じ」のときだけ true。
予測しやすく安全
同様に != より !== を使う
5 !== "5"
true
5 !== 5
false
JSでは == を使うのは悪習慣。常に === を使う

型チェッカー ― 値の型をリアルタイムで確認

JavaScript — 内部コード（参考）
function checkType(input) { // 入力文字列を JavaScript の値に変換して判定 let val; if (input === "true") val = true; else if (input === "false") val = false; else if (input === "null") val = null; else if (input === "") val = undefined; else if (!isNaN(input)) val = Number(input); else val = input; return typeof val; }
▶ 値を入力して型を確認する
試してみよう：
42 → number
hello → string
true → boolean
null → object（罠！）
（空欄） → undefined
Ch.3 — 演算子

算術演算子 ― 数値を計算する

+
加算：weight + 5 → 75
文字列に使うと「結合」になる（後述）
-
減算：100 - weight → 30
*
乗算：height * height → 身長²
/
除算：weight / (height ** 2) → BMI
%
剰余（余り）：10 % 3 → 1
偶数・奇数判定に使う：n % 2 === 0
**
べき乗：height ** 2 → height²
BMI計算でよく使う
算術演算子の実例
const weight = 70; // kg const height = 1.75; // m const bmi = weight / (height ** 2); console.log(bmi.toFixed(1)); // "22.9"（小数1桁に丸める）
Ch.3 — 比較・論理

比較演算子と論理演算子

比較演算子 ― boolean を返す
const hba1c = 7.2; console.log( hba1c > 7.0 ); // true （より大きい） console.log( hba1c >= 7.0 ); // true （以上） console.log( hba1c < 7.0 ); // false （より小さい） console.log( hba1c === 7.2 ); // true （厳格な等値） console.log( hba1c !== 7.0 ); // true （不等値）
論理演算子 ― 条件を組み合わせる
const bmi = 28; const age = 55; const smok = true; // && (AND)：両方 true のとき true console.log( bmi >= 25 && age >= 50 ); // true // || (OR)：どちらか true のとき true console.log( bmi >= 30 || smok ); // true // ! (NOT)：真偽を反転 console.log( !smok ); // false
🩺 医療での使い方
// 生活習慣病リスク判定 const highRisk = (bmi >= 25) && (age >= 50) && (smok === true); console.log(highRisk); // true
🔑 演算子の優先順位（抜粋）
** → 高
* / %
+ -
> >= < <=
=== !==
&&
|| → 低
迷ったら ( ) で明示
Ch.3 — 文字列

文字列の操作 ― よく使うメソッド

.length
文字数を返す："山田太郎".length → 4
.toUpperCase
大文字変換："hba1c".toUpperCase() → "HBA1C"
.includes
含むか判定："糖尿病性腎症".includes("腎症") → true
.startsWith
始まりを判定："E11.9".startsWith("E") → true
ICD-10コードの判定に使える
.replace
文字を置換："体重:70kg".replace("70", "72") → "体重:72kg"
.trim
前後の空白除去：" 山田 ".trim() → "山田"
フォーム入力のクリーニングに必須
文字列結合（+ は古い書き方）
const name = "山田太郎"; const age = 65; console.log("患者名：" + name + "　年齢：" + age); // 古い書き方 ← 読みにくい console.log(`患者名：${name}　年齢：${age}`); // テンプレートリテラル ← こちらを使う
Ch.4 — テンプレートリテラル

テンプレートリテラル ― 文字列と変数を混ぜる

バッククォート ` で囲む → ${ } で変数を展開
const name = "山田太郎"; const age = 65; const hba1c = 7.2; // ${ } の中には変数だけでなく式も書ける const msg = `患者：${name}（${age}歳） HbA1c：${hba1c}%　${hba1c >= 7.0 ? "要管理" : "良好"} BMI：${(70 / 1.75 ** 2).toFixed(1)}`; console.log(msg); // 患者：山田太郎（65歳） // HbA1c：7.2%　要管理 // BMI：22.9
式を埋め込む ― ${} の中で計算できる
const w = 70, h = 1.75; console.log(`BMI = ${(w/h**2).toFixed(1)}`); // "BMI = 22.9" console.log(`今は ${new Date().getFullYear()} 年`); // "今は 2025 年"
🔑 3種類のクォートの違い
"..."　ダブルクォート（普通の文字列）
'...'　シングルクォート（同上）
` ... `　バッククォート ← テンプレートリテラル
改行も \n なしでそのまま書ける
💡 よく使うパターン
`${x.toFixed(2)} mg/dL`
"7.20 mg/dL"
`${a} + ${b} = ${a+b}`
"3 + 4 = 7"
テンプレートリテラルは「報告文の自動生成」に強い ― 医療記録フォーマットにも応用できる
Ch.3 — 代入演算子

代入演算子 ― 変数を更新するショートカット

複合代入演算子
+=
count += 1　→　count = count + 1
-=
score -= 5　→　score = score - 5
*=
dose *= 2　→　dose = dose * 2
/=
total /= n　→　total = total / n
インクリメント・デクリメント
++
count++　→　1ずつ増やす（患者カウントなど）
--
remaining--　→　1ずつ減らす（残薬カウントなど）
実例 ― 外来患者カウンター
let visitCount = 0; // 本日の来院数 visitCount++; // 患者1人来院 → 1 visitCount++; // 患者2人来院 → 2 visitCount += 3; // 3人まとめて処理 → 5 console.log(`本日の来院数：${visitCount} 人`); // "本日の来院数：5 人"
++ は for ループ（Vol.2）で毎回使う ― 今のうちに慣れておこう

Ch.2 補足 — 型変換

明示的な型変換 ― 数値 ↔ 文字列

文字列 → 数値
// フォームの入力値は必ず「文字列」で来る → 計算前に変換が必要 const raw = "70"; // type: string const num1 = Number(raw); // 70 (number) const num2 = parseInt(raw); // 70 (整数) const num3 = parseFloat("7.2kg"); // 7.2 (小数・単位も除去) const bad = Number("山田"); // NaN（変換失敗） console.log( isNaN(bad) ); // true ← NaN チェック
数値 → 文字列
const bmi = 22.857; const s1 = String(bmi); // "22.857" const s2 = bmi.toFixed(1); // "22.9"（丸めて文字列） const s3 = bmi.toString(); // "22.857"
⚠ NaN（Not a Number）：変換できなかった数値の表現。isNaN() で確認できる。演算に NaN が混ざると結果がすべて NaN になるので要注意
よくあるミスとデバッグ

エラーは敵でなく「道しるべ」

よくあるエラー ① ― 変数の再宣言
const age = 25; const age = 30; // ✗ SyntaxError: 再宣言不可 // 解決 → let を使うか変数名を変える let age = 25; age = 30; // ✅ let なら再代入 OK
よくあるエラー ② ― 文字列の足し算
const w = "70"; // string（フォーム入力はstring!） const h = 1.75; // number console.log(w + h); // ✗ "701.75"（文字列結合になる） console.log(Number(w) + h); // ✅ 71.75（数値として加算）
🔧 デバッグの基本手順
  • エラーメッセージを読む（翻訳して理解する）
  • console.log(変数名) で値を確認
  • typeof 変数名 で型を確認
  • 1行ずつコメントアウトして絞り込む
💡 エラーメッセージ対訳
ReferenceError：変数名が見つからない
TypeError：型が違う操作をした
SyntaxError：文法が間違っている
NaN：数値変換に失敗した
エラーを出した数だけ上達する ― 消えないエラーはない
