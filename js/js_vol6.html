非同期処理とAPI連携

〜「待ち時間」をスマートに扱う〜

Concept

なぜ「非同期」が必要か？

通常のプログラムは、1つずつ順番に処理が終わるのを待ちます。しかし...

同期処理（通常）

通信が終わるまで画面がフリーズ。ユーザーは何も操作できません。

非同期処理

通信の完了を待ちつつ、画面の他の操作を可能にします。裏側で処理を進めるイメージです。

API

APIとは何か？

Application Programming Interface。簡単に言うと、「外部サーバーにあるデータや機能を使わせてもらうための窓口」です。

  • 「郵便番号を送ると、住所を返してくれる」
  • 「緯度経度を送ると、今の天気を返してくれる」

現代のWebアプリは、これらを組み合わせて作られています。

Basic

Promise（約束）

非同期処理の結果を「将来返すこと」を約束するオブジェクトです。

// 3つの状態があります 1. Pending (保留中：処理が終わるのを待っている) 2. Fulfilled (成功：約束を果たした) 3. Rejected (失敗：エラーが起きた)

昔はこの .then() を繋げて書いていましたが、今はもっと良い書き方があります。

Modern JS

async / await の基本

非同期処理を「まるで上から下に順番に動いている」かのように書ける魔法の構文です。

async function fetchData() { const result = await 処理; // 終わるまでここで一時停止 console.log(result); // 終わってから実行される }
  • await を使う関数には async を付けます。
Fetch

データを取得する fetch()

async function getWeather() { // 1. サーバーにリクエストを送る const response = await fetch("https://api.example.com/weather"); // 2. レスポンスをJSON形式に変換する const data = await response.json(); console.log(data); }

fetch と .json()、どちらも時間がかかるので await が必要です。

Practice

実践：PubMed風検索（イメージ）

async function searchArticles(keyword) { const url = `https://api.pubmed.example/search?term=${keyword}`; try { const res = await fetch(url); const results = await res.json(); renderList(results); } catch (error) { console.error("通信に失敗しました", error); } }
Try-Catch

try...catch：失敗への備え

インターネット通信は必ずしも成功しません。失敗した時の処理を必ず書きます。

try { // 成功するはずの処理 const data = await fetch(...); } catch (err) { // ネットワークエラーなどが起きた時にここに来る alert("データの取得に失敗しました"); }
UX

「ロード中」を表示する

通信がいつ終わるかユーザーは不安です。視覚的なフィードバックを与えましょう。

loadingElement.style.display = "block"; // 表示 await fetchData(); // 通信 loadingElement.style.display = "none"; // 消去

Data Format

JSON形式とは？

JavaScriptのオブジェクトそっくりのデータ形式です。世界中で使われています。

{ "id": 101, "name": "Acinetobacter", "type": ["Gram-negative", "Coccobacilli"] }

キーも値も " " (ダブルクォート) で囲むのがルールです。

DevTools

開発者ツールの「Network」

通信の様子は、ブラウザのデベロッパーツールで丸見えです。

  • どのURLに送ったか？
  • どんなデータが返ってきたか？
  • 何秒かかったか？

F12キーを押して「Network」タブを開き、Fetch/XHR フィルタを確認する習慣をつけましょう。

Advanced

タイムアウトの設定

あまりに遅い通信は、途中で諦める設定が必要です。

const controller = new AbortController(); setTimeout(() => controller.abort(), 5000); // 5秒で中断 try { const res = await fetch(url, { signal: controller.signal }); } catch(e) { ... }
Syntax

URLパラメータを作る

検索条件をURLに含める正しい作り方です。

const params = new URLSearchParams({ apiKey: "12345", city: "osaka", lang: "ja" }); const fullUrl = "https://api.com/v1?" + params.toString();

スペースや日本語が含まれても、自動的に「安全なURL用文字列」に変換してくれます。

Optimization

並列実行（Promise.all）

2つのデータを取得する際、1つずつ待つより同時に送った方が速いです。

// 同時にスタート！ const [user, post] = await Promise.all([ fetch("/user").then(r => r.json()), fetch("/post").then(r => r.json()) ]);

どちらも完了したタイミングで結果が返ってきます。

Security

APIキーの扱い

多くのAPIは「APIキー（合言葉）」を必要とします。

fetch(url, { headers: { "Authorization": "Bearer YOUR_API_KEY" } });

※ APIキーをGitHubなどにそのまま公開してはいけません！

Methods

データを送る（POST送信）

取得(GET)ではなく、サーバーに情報を保存(POST)する時の書き方です。

fetch(url, { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ name: "金子", score: 100 }) });
Troubleshooting

最大の敵：CORSエラー

セキュリティ上の理由で、ブラウザが勝手なサイト間通信をブロックすることがあります。

Access to fetch at '...' from origin '...' has been blocked by CORS policy...

これは「サーバー側の設定」で許可されていないと解決できません。自作サーバーの場合は注意が必要です。

Library

Axios (アキシオス)

fetch をもっと便利に使いやすくしたライブラリです。現場ではよく使われます。

const res = await axios.get(url); console.log(res.data); // 最初からパースされている！
Review

まとめ：非同期処理の流れ

  • async / await をセットで使うのが基本。
  • fetch() で通信し、.json() でデータ化する。
  • どちらも時間がかかるので await を付ける。
  • 失敗に備えて try...catch で囲む。
Advice

開発効率を上げるコツ

  • ダミーデータを用意する: 通信がなくても開発できるよう const dummy = {...} を作っておく。
  • console.log(data): 取得した中身の階層（配列かオブジェクトか）を必ず確認する。
