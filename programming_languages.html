プログラミング言語の
種類と考え方

C系・JavaScript・Python・R——それぞれどう違うのか、どこで使うのか。比較しながら理解して、JavaScriptは実際に動くコードで体感しよう。

なぜ言語がこんなにたくさんあるのか。最初は混乱するかもしれません。でも、料理と同じで、包丁もあれば菜箸もあれば鍋もある——目的に合った道具を使うだけです。全部できなくていい。まず1つをちゃんとわかれば、他の言語も「ああ、これはあの言語の◯◯に似ているな」と自然に読めるようになります。
01

言語の全体像

なぜ複数の言語が存在するのか

プログラミング言語は「道具」です。ハンマーとメスは別の道具です。それと同じで、Webページを動かしたい、データを解析したい、基板を制御したい——目的が違えば適した言語も違います。

大きな分類として「コンパイル言語」と「スクリプト言語（インタープリタ言語）」があります。コンパイル言語（C、C++など）は実行前に機械語に変換するため高速ですが準備が必要。スクリプト言語（Python、JavaScript、Rなど）はコードをその場で解釈して実行するため手軽です。

区分 C / C++ JavaScript Python R
種類 コンパイル言語 スクリプト言語 スクリプト言語 スクリプト言語
実行速度 非常に速い 速い（JIT最適化） 中程度 遅め（ベクトル演算は速い）
主な用途 OS・組込み・ゲームエンジン Web・アプリ全般 AI・機械学習・汎用 統計解析・研究
型 静的型（厳格） 動的型（緩い） 動的型 動的型
添え字の始まり 0 0 0 1
学習難易度 難しい（メモリ管理要） 比較的易しい 易しい 統計知識あれば易しい

4言語の用途まとめ

C/C++ コンピュータの最も「素」に近い言語。OSのLinuxもCで書かれています。速度が最優先の組み込み機器・ゲームエンジン・医療機器のファームウェアなど。メモリを自分で管理しなければならず難しいですが、「プログラミングの根っこ」を学ぶには最適です。

JavaScript もともとはWebブラウザの中だけで動く言語でした。今はサーバーでも動き（Node.js）、スマホアプリも作れる（React Native）、デスクトップアプリも作れる（Electron）——最も守備範囲が広い言語のひとつです。Webに関わる仕事では避けて通れません。

Python 「読みやすさ」を最優先に設計された言語。AIブームで一躍主役になりました。TensorFlow・PyTorchなどの深層学習ライブラリはPython製です。医学研究でもPythonで機械学習を使うケースが増えています。

R 統計学者が統計学者のために作った言語。統計手法の実装が豊富で、メタ解析・生存時間分析・混合効果モデルなど医学研究に必要な手法がすぐ使えます。グラフの品質も高く、論文品質の図が作れます。

同じ処理を4言語で書き比べる

「1から5の合計を求める」という単純な処理を4言語で比べてみましょう。

1から5の合計 C 
// C言語
#include <stdio.h>

int main() {
    int sum = 0;
    for (int i = 1; i <= 5; i++) {
        sum += i;
    }
    printf("合計: %d\n", sum);  // → 合計: 15
    return 0;
}
1から5の合計 JavaScript 
// JavaScript
const nums = [1, 2, 3, 4, 5];
const sum = nums.reduce((acc, n) => acc + n, 0);
console.log(`合計: ${sum}`);  // → 合計: 15
1から5の合計 Python 
# Python
nums = [1, 2, 3, 4, 5]
total = sum(nums)
print(f"合計: {total}")  # → 合計: 15
1から5の合計 R 
# R
nums <- c(1, 2, 3, 4, 5)
total <- sum(nums)
cat("合計:", total, "\n")  # → 合計: 15
Cは「型の宣言」と「ヘッダのインクルード」が必要。JavaScriptとPythonはほぼ同じ書き方。Rは代入に <- を使い、配列に c() を使う点が特徴的。

フロントエンドとバックエンド

Webの仕組みを理解する

Webアプリケーションは「見える部分（フロントエンド）」と「見えない部分（バックエンド）」に分かれています。レストランに例えると、フロントエンドは「客席・メニュー・給仕」、バックエンドは「厨房・食材倉庫・レジシステム」です。

フロントエンド

ブラウザの中で動く。ユーザーが直接見て操作する部分。

  • JS ページを動かすロジック
  • HTML — ページの構造
  • CSS — 見た目・デザイン
⇄ HTTP リクエスト /レスポンス
バックエンド

サーバーの中で動く。データ処理・認証・DB操作など。

  • JS Node.js
  • Py Django / Flask
  • R Plumber / Shiny
  • C++ 高速処理

どの言語がどこで動くか

言語フロントエンドバックエンド補足
C/C++ ❌ ブラウザ不可 ✅ 可能 主にOS・組込み・ゲーム
JavaScript ✅ 唯一の標準言語 ✅ Node.js で可能 フロント・バック両方書ける唯一の言語
Python ❌（PyodideなどWASM経由で一部可） ✅ Django・FastAPI AI/MLのバックエンド処理に強い
R ❌ ✅ Shiny・Plumber 研究者向けWebダッシュボードに使われる
ブラウザ上で動かせる言語は事実上 JavaScript（とHTML/CSS）だけです。Webページを「動かす」ためにはJavaScriptが必須です。これが「JavaScriptを学ぶ意味」のひとつです。
医学研究でもWebを使う場面は増えています。R Shinyでインタラクティブなダッシュボードを作ったり、Pythonでデータ収集APIを作ったり。「どこで動くか」を知っているだけで、設計の選択肢がぐっと広がります。

添え字（インデックス）の考え方の違い

0始まりと1始まり——なぜ違うのか

プログラミングを始めたばかりの人が最初にハマる罠のひとつが「添え字（インデックス）」の問題です。配列の最初の要素は [0] なのか [1] なのか——言語によって異なります。

C言語はコンピュータのメモリ構造に近い設計で、配列の先頭アドレスからの「ずれ（オフセット）」が0だから [0] が先頭になります。これがC系言語（C、C++、Java、JavaScript、Python）の伝統です。一方Rは数学・統計の世界の慣習を優先して [1] から始めました。

言語別の書き方比較

配列の先頭・末尾・スライス C 
int arr[] = {10, 20, 30, 40, 50};
arr[0]  // → 10  （先頭）
arr[4]  // → 50  （末尾：長さ5なので[4]）
// C は範囲外アクセスしてもエラーにならず危険！
配列の先頭・末尾・スライス JavaScript 
const arr = [10, 20, 30, 40, 50];
arr[0]           // → 10  （先頭）
arr[arr.length - 1]  // → 50  （末尾）
arr.at(-1)        // → 50  （末尾：最新の書き方）
arr.slice(1, 3)   // → [20, 30]  （1番目から3番目の手前まで）
配列の先頭・末尾・スライス Python 
arr = [10, 20, 30, 40, 50]
arr[0]      # → 10  （先頭）
arr[-1]     # → 50  （末尾：マイナスインデックスが使える）
arr[1:3]    # → [20, 30]  （1番目から3番目の手前まで）
配列の先頭・末尾・スライス R 
arr <- c(10, 20, 30, 40, 50)
arr[1]         # → 10  （先頭）※ 1始まり！
arr[length(arr)] # → 50  （末尾）
arr[2:3]        # → 20 30  （2番目から3番目まで、両端含む）
RからJavaScriptに移るとき（またはその逆）、インデックスの「ずれ」によるバグは本当によく起きます。特にスライスの書き方：Rの 2:3 は「2番目と3番目」を含む両端込みですが、JSの .slice(1, 3) は「1番目から3番目の手前まで」です。意図せぬ1ずれに注意しましょう。

04

JavaScript 入門

JavaScriptは「ゆるい」言語です。型を間違えてもエラーにならず勝手に変換する。これは初心者には優しいのですが、慣れてくるとこの「ゆるさ」が思わぬバグを生むことがわかってきます。その特性を知りながら使うことが大切です。

変数宣言の3種類：var / let / const

JavaScriptには変数宣言の方法が3つあります。歴史的な経緯で増えたので、それぞれの違いを知っておく必要があります。

var / let / const の違い JavaScript 
// var: 古い書き方。ブロックスコープがなく予期しない動作をすることがある
var x = 1;
var x = 2;  // 同名で再宣言できてしまう（危険）

// let: 再代入できる変数
let count = 0;
count = count + 1;  // OK
// let count = 5;  // 同名再宣言はエラー（安全）

// const: 再代入できない定数（推奨）
const PI = 3.14159;
// PI = 3;  // エラー！

// ただし const のオブジェクトや配列の中身は変更できる
const arr = [1, 2, 3];
arr.push(4);  // OK（中身の変更は可）
// arr = [5, 6];  // エラー（再代入は不可）
現代のJavaScriptでは var は使わないのが鉄則です。基本は const、値を変える必要があるときだけ let を使います。

データ型と型の「ゆるさ」

JSの主要データ型 JavaScript 
// 基本型（プリミティブ）
const n  = 42;           // number（整数も小数も同じ型）
const s  = "hello";      // string
const b  = true;         // boolean
const u  = undefined;   // 値がまだ設定されていない
const nu = null;         // 意図的に「空」を表す

// 型の確認
typeof 42        // → "number"
typeof "hello"  // → "string"
typeof true     // → "boolean"
typeof null     // → "object" ← 有名なバグ！歴史的な仕様ミス

// 型の自動変換（暗黙的型変換）
"3" + 1     // → "31"  （文字列の連結になる！）
"3" - 1     // → 2     （引き算は数値に変換される）
"3" * 2     // → 6
true + 1   // → 2     （true は 1 として扱われる）
+ 演算子は「数値の足し算」と「文字列の連結」を兼ねています。片方が文字列だと文字列連結になります。"3" + 1 が "31" になるのはJSあるあるバグです。明示的に変換するには Number("3") または parseInt("3") を使いましょう。

配列とオブジェクト

JavaScriptのデータ構造の中心は「配列（Array）」と「オブジェクト（Object）」です。Rのベクトルとデータフレームに対応すると考えると理解しやすいです。

配列（Array） JavaScript 
const scores = [85, 72, 91, 68, 77];

// 基本操作
scores.push(90);          // 末尾に追加
scores.pop();             // 末尾を取り出し
scores.length             // → 5（長さ）

// 配列の変換・絞り込み（高階関数）
scores.map(s => s * 1.1)        // 全要素を1.1倍
scores.filter(s => s >= 80)     // 80点以上だけ抽出
scores.reduce((a, b) => a + b)  // 合計
scores.find(s => s > 90)        // 最初の90超を返す

// ソート（注意：デフォルトは文字列順！）
[10, 9, 100].sort()              // → [10, 100, 9] （文字列順！）
[10, 9, 100].sort((a, b) => a - b) // → [9, 10, 100] （数値順）
オブジェクト（Object） JavaScript 
// オブジェクト：キーと値のペア（Rの名前付きリストに似ている）
const patient = {
  id:      1001,
  name:    "田中太郎",
  age:     45,
  group:   "治療群",
  scores:  [72, 85, 91]
};

// プロパティへのアクセス
patient.name           // → "田中太郎"
patient["age"]         // → 45（ブラケット記法）
patient.scores[0]     // → 72

// 分割代入（便利な構文）
const { name, age } = patient;
console.log(name, age);  // → "田中太郎" 45

// スプレッド構文（コピー・マージ）
const updated = { ...patient, age: 46 };  // age だけ更新したコピー

関数の4つの書き方

JavaScriptは関数の書き方が複数あります。歴史的に増えてきたので、それぞれを知っておきましょう。

関数の書き方いろいろ JavaScript 
// ① 関数宣言（hoisting される：定義前に呼べる）
function add(a, b) {
  return a + b;
}

// ② 関数式（変数に代入）
const add = function(a, b) {
  return a + b;
};

// ③ アロー関数（現代の主流。短く書ける）
const add = (a, b) => a + b;  // 1行ならreturn省略可

const greet = name => {        // 引数1つなら() 省略可
  return `こんにちは、${name}さん`;
};

// ④ デフォルト引数・残余引数
const power = (base, exp = 2) => base ** exp;  // exp 省略時は 2
power(3)     // → 9
power(3, 3)  // → 27

const sumAll = (...nums) => nums.reduce((a, b) => a + b);
sumAll(1, 2, 3, 4)  // → 10
アロー関数 => は短く書けるだけでなく、this の扱いが通常の関数と異なります。クラスやイベントリスナーの中で使う場合はこの違いが重要になります。最初は「短く書ける関数」と覚えておけばOK。

非同期処理：Promise と async/await

JavaScriptの最大の特徴のひとつが「非同期処理」です。データをサーバーから取得する・ファイルを読む・タイマーを使う——これらは「結果が来るまで待つ」処理です。他の言語では処理が終わるまで止まりますが、JavaScriptは「頼んでおいて、他の処理を続けて、来たら処理する」という仕組みを持っています。

async / await の基本 JavaScript 
// 昔のやり方（コールバック地獄になりがち）
fetch(url)
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

// 現代のやり方（async/await）
async function getData(url) {
  try {
    const response = await fetch(url);     // 待つ
    const data     = await response.json();  // 待つ
    console.log(data);
    return data;
  } catch (error) {
    console.error("エラー:", error);
  }
}

getData("https://api.example.com/patients");
async をつけた関数の中でだけ await が使えます。await はPromiseの完了を待ち、その結果を受け取ります。エラー処理は必ず try/catch で行いましょう。

DOMを操作する

DOM（Document Object Model）はHTMLをJavaScriptから操作するための仕組みです。HTMLの各要素をオブジェクトとして扱えます。

DOM操作の基本 HTML+JS 
<!-- HTML側 -->
<h1 id="title">こんにちは</h1>
<p  class="message">メッセージ</p>
<ul id="list"></ul>

/* JavaScript側 */
// 要素を取得
const title = document.getElementById("title");
const msg   = document.querySelector(".message");

// テキストを変更
title.textContent = "更新されました";
msg.textContent   = "こんにちは世界";

// スタイルを変更
title.style.color    = "steelblue";
title.style.fontSize = "2rem";

// 要素を動的に追加
const list  = document.getElementById("list");
const items = ["田中", "佐藤", "鈴木"];

items.forEach(name => {
  const li = document.createElement("li");
  li.textContent = name;
  list.appendChild(li);
});

イベント処理

ユーザーの操作（クリック・入力・マウス移動など）に反応するのが「イベント」です。

イベントリスナーの基本 HTML+JS 
<button id="btn">クリックしてください</button>
<input  id="nameInput" type="text" placeholder="名前を入力">
<p      id="output"></p>

const btn       = document.getElementById("btn");
const nameInput = document.getElementById("nameInput");
const output    = document.getElementById("output");

// クリックイベント
btn.addEventListener("click", () => {
  const name = nameInput.value || "名無し";
  output.textContent = `こんにちは、${name}さん！`;
  output.style.color = "steelblue";
});

// キー入力イベント（リアルタイム）
nameInput.addEventListener("input", (e) => {
  output.textContent = `入力中: ${e.target.value}`;
});

fetch でデータを取得する

実際のWebアプリではサーバーからデータを取得します。fetch APIを使います。

fetch で外部APIからデータ取得 JavaScript 
// JSONPlaceholderは練習用の無料APIサーバー
async function loadUsers() {
  const url = "https://jsonplaceholder.typicode.com/users";

  try {
    const response = await fetch(url);

    // HTTPエラーチェック
    if (!response.ok) {
      throw new Error(`HTTP error: ${response.status}`);
    }

    const users = await response.json();

    // 取得したデータを使う
    users.forEach(user => {
      console.log(`${user.name}: ${user.email}`);
    });

    // データを加工してHTMLに表示
    const list = document.getElementById("userList");
    list.innerHTML = users
      .map(u => `<li><strong>${u.name}</strong> — ${u.email}</li>`)
      .join("");

  } catch (err) {
    console.error("取得失敗:", err.message);
  }
}

loadUsers();

          return;
        }

        const bmi = w / ((h / 100) ** 2);
        const { label, cls } = getBMICategory(bmi);

        result.style.display = "block";
        result.className = cls;
        result.innerHTML = `
          BMI: <strong>${bmi.toFixed(1)}</strong>　
          判定: <strong>${label}</strong>
        `;
      });
  </script>
</body>
</html>
このコードをそのままメモ帳に貼り付けて bmi.html という名前で保存し、ブラウザで開けば実際に動きます。ぜひ試してみてください。
コードは「読む」と「書く」は全然違います。読めばわかった気がしても、白紙から書こうとすると手が止まる。それは正常です。最初はこのコードをそのままコピーして動かし、次に少し変えてみる——その繰り返しが力になります。エラーが出たら「なぜ出たか」を調べる。それがプログラミングの学び方です。
