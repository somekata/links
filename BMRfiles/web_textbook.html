HTML 骨格を作る HyperText Markup Language
H-01

HTML 文書の全体構造

HTMLファイルは「決まった骨格」から始まります。毎回必ず書くテンプレートとして覚えましょう。

基本テンプレート
<!-- DOCTYPE 宣言：これは HTML5 です、とブラウザに伝える -->
<!DOCTYPE html>
<html lang="ja">

<head>                              <!-- ブラウザに表示されない設定情報 -->
  <meta charset="UTF-8">          <!-- 文字コード（日本語を使うなら必須）-->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>ページタイトル</title>
  <link rel="stylesheet" href="style.css">
</head>

<body>                              <!-- ここに表示したいコンテンツを書く -->
  <h1>見出し</h1>
  <p>段落のテキスト</p>
  <script src="script.js"></script>  <!-- JS は body の末尾に -->
</body>

</html>
<head>
ブラウザに表示されない「設定情報」。文字コード・タイトル・CSS読み込みなど。
<body>
ブラウザに表示されるすべてのコンテンツを書く場所。
lang="ja"
日本語ページと宣言。スクリーンリーダーや検索エンジンに影響する。
meta viewport
スマートフォン表示を適切にするための設定。モバイル対応に必須。

<h1>〜<h6>

H-03

属性（Attribute）

属性はタグに追加情報を付ける仕組みです。<タグ名 属性名="値"> の形で書きます。

属性の書き方例
<!-- href で行き先を指定するリンク -->
<a href="https://pubmed.ncbi.nlm.nih.gov/" target="_blank">PubMed</a>

<!-- id と class で CSS・JS から操作 -->
<div id="main-box" class="card highlight">コンテンツ</div>

<!-- src と alt で画像表示 -->
<img src="cell.jpg" alt="好中球の顕微鏡写真" width="400">
属性使うタグ説明
href<a>リンク先のURL
src<img><script>画像・スクリプトのパス
alt<img>代替テキスト。アクセシビリティ必須
idすべて一意な名前。1ページで重複不可
classすべてグループ名。複数指定可
target="_blank"<a>新しいタブで開く
type<input>入力欄の種類（text/email/number等）
placeholder<input>薄く表示されるヒントテキスト
⚠ id と class の違い： id は1ページに1つだけ（固有名）。class は何個でも同じものを使える（グループ名）。CSSでは #id名、.class名 で区別します。
H-04

セマンティック HTML

「意味のある」タグを使うことで、検索エンジンやスクリーンリーダーが文書構造を正しく理解できます。

✗ div だらけ（意味なし）
<div class="header">…</div>
<div class="nav">…</div>
<div class="main">
  <div class="article">
    <div class="title">…</div>
  </div>
</div>
<div class="footer">…</div>
✓ セマンティックタグを使う
<header>…</header>
<nav>…</nav>
<main>
  <article>
    <h1>…</h1>
    <p>…</p>
  </article>
</main>
<footer>…</footer>
<header>
ページ・記事のヘッダー（ロゴ・タイトル・ナビ）。
<nav>
ナビゲーションリンク集。サイトメニューに使う。
<main>
ページの主要コンテンツ。1ページに1つ。
<article>
単体で意味が完結する内容（ブログ記事・論文要旨）。
<section>
ひとまとまりの内容のセクション。
<footer>
フッター（著作権・連絡先など）。
H-05

リンクと画像

リンクのパターン
<!-- 外部サイト（新タブで開く）-->
<a href="https://pubmed.ncbi.nlm.nih.gov/"
   target="_blank" rel="noopener noreferrer">PubMed</a>

<!-- 同じページ内のジャンプ -->
<a href="#h-form">フォームセクションへ</a>

<!-- メール送信 -->
<a href="mailto:example@osaka-u.ac.jp">メールを送る</a>

<!-- figure / figcaption でキャプション付き画像 -->
<figure>
  <img src="cell.jpg" alt="好中球の顕微鏡写真">
  <figcaption>図1：好中球（×1000倍）</figcaption>
</figure>
💡 alt 属性は省略しない： 画像が表示できない状況（低速回線・スクリーンリーダー）でも意味が伝わるよう、必ず画像の内容を説明するテキストを書きましょう。装飾目的の画像のみ alt=""（空）でOK。

CSS 見た目を作る Cascading Style Sheets
C-01

CSS の書き方と読み込み方

CSSは「何に（セレクタ）・何を（プロパティ）・どうする（値）」の3要素で構成されます。

CSS の基本構文
/* セレクタ { プロパティ: 値; } の形が基本 */

p {
  color: navy;
  font-size: 16px;
  line-height: 1.8;
}

.card {
  background-color: #fff;
  border: 1px solid #ccc;
  border-radius: 8px;
  padding: 16px;
}
外部CSS（推奨）
.cssファイルを別に作り、<link>タグで読み込む。複数のHTMLで共有できる。
<style>タグ
<head>内に書く。そのHTMLファイルだけに適用される。
インラインstyle
タグに直接 style="..."。優先度が最も高いが、緊急時以外は使わない。
CSS変数
:root{--color:#fff} で定義し、var(--color) で使う。色やサイズを一元管理できる。
C-02

セレクタ — 「どこに」当てるか

セレクタ例説明
タグ名p { }すべての <p> に適用
.クラス名.card { }class="card" を持つすべての要素
#id名#header { }id="header" を持つ要素（1つだけ）
A Bdiv p { }div の中の p（子孫セレクタ）
:hovera:hover { }マウスが乗ったとき
:nth-child(n)tr:nth-child(2n) { }偶数行（ゼブラストライプ）
[属性名]input[type="text"] { }属性値を持つ要素
詳細度のルール（強い順）：
!important > インラインstyle > #id > .class > タグ名。同じ詳細度なら後から書いたものが優先されます。
C-03

ボックスモデル — すべての要素は「箱」

HTMLのすべての要素は、見えない「箱」として扱われます。この箱は4つの層で構成されています。

margin（外側の余白）
border（枠線）
padding（内側の余白）
content（内容）
ボックスモデルのプロパティ
.box {
  width: 300px;
  padding: 16px;              /* 内側の余白 */
  border: 1px solid #ccc;    /* 枠線 */
  margin: 0 auto;             /* 上下0、左右auto → 中央揃え */
  border-radius: 8px;         /* 角丸 */
  box-shadow: 0 4px 16px rgba(0,0,0,0.1);

  /* ★重要：常に指定する */
  box-sizing: border-box;     /* widthにpadding・borderを含める */
}
⚠ box-sizing: border-box を必ず指定しよう： デフォルトでは padding・border の分だけ実際の幅が広がります。*, *::before, *::after { box-sizing: border-box; } を先頭に書くのが現代のベストプラクティスです。

C-05

Flexbox — 横並びレイアウト

親要素に display: flex を指定するだけで、子要素が横に並びます。ナビゲーション・カード並び・ボタン配置など、あらゆる「横並び」に使います。

Flexbox の主なプロパティ
.container {
  display: flex;
  flex-direction: row;          /* 方向（row=横、column=縦）*/
  justify-content: space-between; /* 主軸の揃え */
  align-items: center;          /* 交差軸の揃え */
  gap: 16px;                     /* 要素間の余白 */
  flex-wrap: wrap;              /* 折り返す */
}

J-01

変数とデータ型

変数の3種類
// const — 再代入不可（推奨。変える必要がないなら const）
const patientId = 'P001';

// let — 再代入できる
let score = 0;
score = 85;          // OK
score += 10;          // score = score + 10

// var — 古い書き方。現代では使わない
var old = 'avoid';

// テンプレートリテラル（バッククォートで変数を埋め込む）
const name = '太郎';
const msg  = `こんにちは、${name}さん`;   // → "こんにちは、太郎さん"
string
'テキスト' または "テキスト"。テンプレートリテラルで変数埋め込み可。
number
整数・小数を区別しない。NaN（非数）・Infinityも含む。
boolean
true か false の2値。条件分岐に使う。
null / undefined
null=意図的に「値なし」。undefined=値が代入されていない。

J-05

配列とオブジェクト

配列（Array）
const symptoms = ['発熱', '頭痛', '倦怠感'];

symptoms[0]               // → '発熱'（0始まり）
symptoms.length           // → 3
symptoms.push('咳嗽')     // → 末尾に追加
symptoms.pop()            // → 末尾から削除
symptoms.includes('頭痛') // → true

// map — 各要素を変換して新配列
const scores = [72, 85, 64, 90, 58];
const labels = scores.map(s => s >= 60 ? '合格' : '不合格');

// filter — 条件を満たすものだけ取り出す
const passed = scores.filter(s => s >= 60);

// reduce — 1つの値にまとめる
const total = scores.reduce((acc, s) => acc + s, 0);
オブジェクト（Object）
const patient = {
  id:    'P001',
  name:  '田中 太郎',
  age:   45,
  hba1c: 7.2
};

patient.name        // → '田中 太郎'（ドット記法）
patient['age']      // → 45（ブラケット記法）
patient.blood = 'A'; // プロパティを追加

// オブジェクトの配列（最もよく使うパターン）
const patients = [

];
