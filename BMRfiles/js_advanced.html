非同期処理 Asynchronous JavaScript Promise · async/await · fetch
A-1

なぜ「非同期」が必要か — コールバック

JavaScriptは基本的に1行ずつ順番に実行されます（同期処理）。しかしサーバーへのデータ取得や大きなファイルの読み込みは時間がかかるため、待っている間も他の処理ができるよう非同期処理の仕組みがあります。

昔は「処理が終わったらこれを呼んで」というコールバック関数を使っていました。

コールバックの例（setTimeout）
// setTimeout(コールバック, ミリ秒) — 時間が経ったら実行
console.log('① 開始');

setTimeout(() => {
  console.log('③ 2秒後に実行（非同期）');
}, 2000);

console.log('② すぐに実行（③ より先！）');
// 出力順: ① → ② → ③
⚠ コールバック地獄（Callback Hell）：
複数の非同期処理を順番に行うと、コールバックがネストされてコードが読みにくくなります。これを解決するために Promise が生まれました。 
// コールバック地獄の例（読みにくい！）
getUser(id, (user) => {
  getRecords(user, (records) => {
    analyzeData(records, (result) => {
      saveResult(result, () => {
        console.log('完了'); // 深すぎる…
      });
    });
  });
});
A-2

Promise — 「約束」オブジェクト

Promise は「非同期処理の結果を将来受け取る約束」を表すオブジェクトです。3つの状態を持ちます。

Promise の3つの状態
pending（待機中）
fulfilled（成功）
rejected（失敗）

pending → fulfilled（resolve）または rejected（reject）に遷移します。一度遷移したら変わりません。

Promise の作り方と使い方
// Promise を作る
const myPromise = new Promise((resolve, reject) => {
  // 非同期処理（例：1秒後に結果を返す）
  setTimeout(() => {
    const success = true;
    if (success) {
      resolve('取得成功: 田中太郎');  // 成功
    } else {
      reject(new Error('患者が見つかりません')); // 失敗
    }
  }, 1000);
});

// Promise を使う（.then / .catch / .finally）
myPromise
  .then(result => console.log(result))   // 成功したら
  .catch(err  => console.log(err.message)) // 失敗したら
  .finally(() => console.log('必ず実行'));  // 常に実行
複数の Promise を扱う
// Promise.all — 全部成功したら次へ（1つでも失敗したら catch へ）
const [user, records] = await Promise.all([
  fetchUser(id),
  fetchRecords(id),
]);

// Promise.allSettled — 成功・失敗に関わらず全部待つ
const results = await Promise.allSettled([p1, p2, p3]);
results.forEach(r => {
  if (r.status === 'fulfilled') console.log(r.value);
  else                            console.log(r.reason);
});

// Promise.race — 最初に完了したものの結果を使う
const fastest = await Promise.race([server1, server2]);

A-3

async / await — Promise をもっと読みやすく

async/await は Promise をまるで同期処理のように書ける構文です。内部では Promise のまま動いています。

Promise チェーン（読みにくい）
fetchPatient(id)
  .then(p => fetchRecords(p.id))
  .then(r => analyzeData(r))
  .then(a => saveResult(a))
  .catch(e => console.log(e));
async/await（読みやすい）
async function process(id) {
  const p = await fetchPatient(id);
  const r = await fetchRecords(p.id);
  const a = await analyzeData(r);
  await saveResult(a);
}
async / await の書き方ルール
// async をつけた関数は必ず Promise を返す
async function getPatientData(patientId) {

  // await は async 関数の中だけで使える
  // → Promise が解決するまでここで「待つ」
  const patient = await fetchPatient(patientId);

  // 通常の処理と混ぜて書ける
  const bmi = patient.weight / (patient.height / 100) ** 2;
  console.log(`${patient.name}: BMI = ${bmi.toFixed(1)}`);

  return { patient, bmi }; // return した値が Promise の resolve 値になる
}

// 呼び出し方（トップレベルでも await が使える環境では）
const result = await getPatientData('P001');

// または .then() で受け取る
getPatientData('P001').then(result => console.log(result));

A-4

fetch API — サーバーからデータを取得する

fetch() はブラウザ標準のHTTP通信関数です。Promise を返します。実際のWeb開発でよく使う「外部APIからデータを取得する」パターンを学びます。

fetch の基本パターン
// fetch は Promise を返す → async/await と組み合わせるのが定番
async function loadData(url) {
  try {
    const response = await fetch(url);

    // ① HTTP エラーチェック（fetch は 404 でも reject しない！）
    if (!response.ok) {
      throw new Error(`HTTP error: ${response.status}`);
    }

    // ② レスポンスを JSON として解析
    const data = await response.json();
    return data;

  } catch (err) {
    console.error('取得エラー:', err.message);
    throw err;  // 呼び出し元にも伝える
  }
}

// 使い方
const patients = await loadData('https://api.example.com/patients');
POST でデータを送信する場合
const response = await fetch('/api/patients', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({    // オブジェクトを JSON 文字列に変換
    name:   '田中 太郎',
    age:    45,
    hba1c:  7.2,
  }),
});

const result = await response.json();
console.log(`患者登録: ID=${result.id}`);

⚠ fetch の注意点：
fetch は 404・500 などのHTTPエラーでも reject（例外）を投げません。response.ok（200〜299ならtrue）を必ずチェックしましょう。ネットワーク断絶のときだけ reject されます。
クラス Class・継承・this Object Oriented Programming
C-1

クラスの基本

クラスはオブジェクトの「設計図」です。同じ構造を持つオブジェクトを何度も作りたいときに使います。

クラスの書き方
class Patient {

  // constructor — new したときに自動で呼ばれる初期化処理
  constructor(id, name, age, hba1c) {
    this.id    = id;      // this.プロパティ名 で値を保存
    this.name  = name;
    this.age   = age;
    this.hba1c = hba1c;
  }

  // メソッド（クラスに属する関数）
  diagnose() {
    if (this.hba1c < 5.7)  return '正常';
    if (this.hba1c < 6.5)  return '糖尿病前症';
    return '糖尿病';
  }

  summary() {
    return `[${this.id}] ${this.name}（${this.age}歳）: ${this.diagnose()}`;
  }

  // static メソッド — インスタンスを作らずに使える
  static calcBMI(weight, height) {
    return (weight / (height / 100) ** 2).toFixed(1);
  }
}

// インスタンスを作る（new キーワード）
const p1 = new Patient('P001', '田中 太郎', 45, 7.2);
const p2 = new Patient('P002', '山田 花子', 62, 5.4);

console.log(p1.summary()); // → [P001] 田中 太郎（45歳）: 糖尿病
console.log(p2.summary()); // → [P002] 山田 花子（62歳）: 正常

// static メソッドの呼び方（インスタンス不要）
console.log(Patient.calcBMI(65, 170)); // → "22.5"

C-2

継承 — クラスを拡張する

extends で既存のクラスを継承し、機能を追加・変更できます。共通部分を親クラスにまとめることで重複を避けられます。

Patient（親クラス）
├ id, name, age, hba1c
└ diagnose(), summary()

↑ extends

InpatientPatient（子クラス） — Patientを継承
├ ward（病棟）, admitDate（入院日）
└ summary()（オーバーライド）, daysAdmitted()
extends と super の使い方
class InpatientPatient extends Patient {

  constructor(id, name, age, hba1c, ward, admitDate) {
    // super() — 親クラスの constructor を呼ぶ（必須）
    super(id, name, age, hba1c);
    this.ward      = ward;
    this.admitDate = new Date(admitDate);
  }

  // メソッドのオーバーライド（親の summary を上書き）
  summary() {
    const base = super.summary();  // 親の summary を使いつつ
    return `${base}【入院中：${this.ward}病棟】`;
  }

  // 入院日数を計算
  daysAdmitted() {
    const diff = Date.now() - this.admitDate.getTime();
    return Math.floor(diff / (1000 * 60 * 60 * 24));
  }
}

const ip = new InpatientPatient('P010', '木村 三郎', 70, 8.2, '3A', '2025-03-01');
console.log(ip.summary());         // 親＋子の情報が表示される
console.log(ip.daysAdmitted());    // 入院日数
console.log(ip instanceof Patient); // → true（Patient の一種）
C-3

this の注意点

this は「このオブジェクト自身」を指しますが、呼ばれ方によって値が変わるという落とし穴があります。

this が変わる問題と対処法
class Counter {
  constructor() {
    this.count = 0;
  }
  increment() {
    this.count++;
    console.log(this.count);
  }
}

const c = new Counter();
c.increment();   // → 1 ✅ 正常

// ❌ メソッドを変数に代入すると this が失われる
const inc = c.increment;
inc();           // → エラー（this が undefined）

// ❌ setTimeout のコールバックでも this が失われる
setTimeout(c.increment, 1000); // エラー

// ✅ 解決策1: アロー関数でラップする
setTimeout(() => c.increment(), 1000);

// ✅ 解決策2: bind() で this を固定する
const boundInc = c.increment.bind(c);
boundInc();      // → 2 ✅

// ✅ 解決策3: クラスフィールドでアロー関数として定義
class Counter2 {
  count = 0;
  increment = () => {   // アロー関数はthisを外側から受け継ぐ
    this.count++;
    console.log(this.count);
  };
}
💡 アロー関数と this：
アロー関数は自分自身の this を持ちません。代わりに定義された時点の外側の this を使います。クラスのメソッドをコールバックとして渡す場合はアロー関数でラップするか、クラスフィールドで定義するのが現代のベストプラクティスです。
エラー処理 try / catch / throw Robust Error Handling
E-1

try / catch / finally

エラーが発生してもプログラムが止まらないようにするのが try/catch です。「試してみて、ダメだったら対処する」という構造です。

基本構文
try {
  // エラーが起きるかもしれない処理
  const data = JSON.parse(userInput);   // 不正なJSONならエラー
  console.log(data.name);

} catch (err) {
  // エラーが起きたら実行される
  console.error('エラーの種類:', err.name);     // → "SyntaxError"
  console.error('エラーのメッセージ:', err.message);
  console.error('スタックトレース:', err.stack);

} finally {
  // エラーがあってもなくても必ず実行
  console.log('処理終了（リソース解放など）');
}
throw — 自分でエラーを投げる
// throw でエラーを意図的に発生させる
function validateHbA1c(value) {
  if (typeof value !== 'number') {
    throw new TypeError('HbA1c は数値で入力してください');
  }
  if (value < 0 || value > 20) {
    throw new RangeError(`HbA1c の値 ${value} は範囲外です（0〜20）`);
  }
  return true;
}

try {
  validateHbA1c('高め');   // → TypeError が投げられる
} catch (err) {
  // エラーの種類によって処理を分岐
  if (err instanceof TypeError)  console.log('型エラー:', err.message);
  if (err instanceof RangeError) console.log('範囲エラー:', err.message);
}

E-2

エラーの種類

SyntaxError
構文エラー。JSON.parse に不正な文字列を渡したときなど。
TypeError
型エラー。null のプロパティにアクセスしようとしたときなど最頻出。
ReferenceError
未定義の変数を参照したとき。スペルミスで起きやすい。
RangeError
配列の長さに負の数を指定するなど、範囲外の値のとき。
URIError
encodeURI / decodeURI に不正な文字列を渡したとき。
Error（カスタム）
class AppError extends Error で自分専用のエラークラスを作れる。
カスタムエラークラスの作り方
// 独自のエラークラス（クラスの継承を使う）
class ValidationError extends Error {
  constructor(field, message) {
    super(message);
    this.name  = 'ValidationError';
    this.field = field;  // どのフィールドでエラーが起きたか
  }
}

class NetworkError extends Error {
  constructor(statusCode, message) {
    super(message);
    this.name       = 'NetworkError';
    this.statusCode = statusCode;
  }
}

// 使い方
try {
  throw new ValidationError('hba1c', '値が範囲外です');
} catch (err) {
  if (err instanceof ValidationError) {
    console.log(`${err.field}: ${err.message}`); // → hba1c: 値が範囲外です
  }
}
E-3

非同期処理のエラーを捕まえる

非同期処理（async/await）のエラーも try/catch で捕まえられます。非同期関数の中で発生した例外は、await した側に伝わります。

async/await + try/catch の組み合わせ
async function fetchPatient(id) {
  try {
    const res = await fetch(`/api/patients/${id}`);
    if (!res.ok) throw new NetworkError(res.status, '患者データの取得に失敗');
    return await res.json();

  } catch (err) {
    if (err instanceof NetworkError) {
      console.error(`通信エラー (${err.statusCode}): ${err.message}`);
      return null;   // エラー時は null を返して呼び出し元に委ねる
    }
    throw err;       // 想定外のエラーは再スロー
  }
}

// Promise.allSettled と組み合わせて「一部失敗を許容する」
const ids = ['P001', 'P002', 'INVALID'];
const results = await Promise.allSettled(ids.map(id => fetchPatient(id)));

results.forEach((r, i) => {
  if (r.status === 'fulfilled') console.log(`${ids[i]}: 取得成功`);
  else                            console.log(`${ids[i]}: 取得失敗`);
});
🚨 よくある間違い — catch し忘れ：
async 関数の中で await するとき、try/catch がないと エラーが「未処理のPromise rejection」となり、本番環境ではクラッシュの原因になります。
呼び出しチェーンのどこかで必ずエラーを捕まえましょう。
モジュール＋構文 Modules & Modern Syntax import · export · ES2015+
X-1

なぜモジュールが必要か

プログラムが大きくなると、すべてのコードを1つのファイルに書くと読みにくくなります。モジュールは「ファイルを機能単位に分割して、必要なものだけを共有する」仕組みです。

⚠ モジュールなしの問題点：
モジュールがないと、すべての変数・関数がグローバル（window）に乗るため、名前の衝突が起きやすくなります。また、ファイルの読み込み順に依存するバグも発生します。 
<!-- 問題: utils.jsとlib.jsの両方にcalculate()があったら？ -->
<script src="utils.js"></script>
<script src="lib.js"></script>   <!-- 後から読んだ方で上書きされる! -->

モジュールを使うと、各ファイルは自分のスコープを持ち、明示的に export したものだけが外から見えます。

💡 ブラウザでのモジュールの使い方：
<script type="module" src="main.js"></script>
type="module" を指定するだけで、そのファイル内で import/export が使えます。ローカルファイル（file://）では動かないため、VSCodeのLive Serverなどで動作確認してください。
X-2

export と import の書き方

名前付きエクスポート（Named Export）— 複数を出せる

patient-utils.js — エクスポート側
// ① 宣言と同時に export
export const NORMAL_RANGE = { min: 4.0, max: 5.6 };

export function diagnoseHbA1c(value) {
  if (value < 5.7) return '正常';
  if (value < 6.5) return '糖尿病前症';
  return '糖尿病';
}

export function calcBMI(weight, height) {
  return (weight / (height / 100) ** 2).toFixed(1);
}

// ② まとめて export（後で宣言したものを一括で）
function formatDate(d) { return new Date(d).toLocaleDateString('ja-JP'); }
export { formatDate };
main.js — インポート側
// 名前付き import（{}で指定）
import { diagnoseHbA1c, calcBMI } from './patient-utils.js';

// 別名（as）で受け取る
import { diagnoseHbA1c as diagnose } from './patient-utils.js';

// 全部まとめて受け取る（名前空間インポート）
import * as PatientUtils from './patient-utils.js';
PatientUtils.calcBMI(65, 170);   // → "22.5"

// 使う
console.log(diagnoseHbA1c(7.2));  // → "糖尿病"
console.log(calcBMI(65, 170));   // → "22.5"

デフォルトエクスポート（Default Export）— 1ファイル1つ

Patient.js
// export default — ファイルの「主役」を1つだけ出す
export default class Patient {
  constructor(id, name, age, hba1c) {
    this.id = id; this.name = name;
    this.age = age; this.hba1c = hba1c;
  }
  summary() { return `[${this.id}] ${this.name}（${this.age}歳）`; }
}
main.js — デフォルトインポート
// デフォルト import は {} 不要・名前は自由
import Patient from './Patient.js';

// 名前付き + デフォルトを同時に import
import Patient, { diagnoseHbA1c, NORMAL_RANGE } from './patient-utils.js';

const p = new Patient('P001', '田中 太郎', 45, 7.2);
console.log(p.summary());   // → "[P001] 田中 太郎（45歳）"
💡 named vs default — どちらを使う？
Named export：ユーティリティ関数・定数など、複数を1ファイルにまとめる場合。
Default export：クラス・コンポーネントなど、「このファイルの主役」が1つの場合。
迷ったら Named export を使う方が、リファクタリング時に追跡しやすくなります。

M-1

分割代入（Destructuring）

配列やオブジェクトの値を、すっきりした書き方で変数に取り出す構文です。

オブジェクトの分割代入
const patient = { id: 'P001', name: '田中 太郎', age: 45, hba1c: 7.2 };

// 従来の書き方
const id    = patient.id;
const name  = patient.name;

// 分割代入（すっきり！）
const { id, name, age } = patient;

// 別名で受け取る
const { name: patientName, hba1c: glycated } = patient;
console.log(patientName);   // → '田中 太郎'

// デフォルト値付き
const { bloodType = '不明' } = patient;

// 関数の引数でも使える（よく見るパターン）
function showPatient({ name, age, hba1c }) {
  console.log(`${name}（${age}歳）HbA1c: ${hba1c}%`);
}
showPatient(patient);
配列の分割代入
const scores = [85, 92, 78, 95];

// 配列分割代入
const [first, second, , fourth] = scores; // 3番目をスキップ

// 残り要素をまとめる（残余要素）
const [top, ...rest] = scores;
console.log(top);   // → 85
console.log(rest);  // → [92, 78, 95]

// 変数の値を交換
let a = 1, b = 2;
[a, b] = [b, a];   // → a=2, b=1

// fetch の結果を受け取るパターン
const [patient, records] = await Promise.all([
  fetchPatient(id),
  fetchRecords(id),
]);
M-2

スプレッド構文 & 残余引数

... は「展開する」演算子です。配列・オブジェクトの複製・結合に使います。

スプレッド構文の活用
// 配列のコピー・結合
const a = [1, 2, 3];
const b = [4, 5];
const merged = [...a, ...b];         // → [1, 2, 3, 4, 5]
const copy   = [...a];               // コピー（元の配列を変更しない）

// オブジェクトのコピー・マージ
const base    = { id: 'P001', name: '田中' };
const updated = { ...base, hba1c: 6.8, age: 46 }; // 上書き可
// → { id: 'P001', name: '田中', hba1c: 6.8, age: 46 }

// 関数の引数に配列を展開
const nums = [3, 1, 4, 1, 5];
console.log(Math.max(...nums));     // → 5

// 残余引数（関数の可変引数）
function average(...scores) {       // 任意の数の引数を配列で受け取る
  return scores.reduce((s, v) => s + v, 0) / scores.length;
}
console.log(average(80, 92, 75, 88)); // → 83.75
M-3

Optional Chaining & Nullish Coalescing

ネストされたオブジェクトへの安全なアクセスや、null/undefined の扱いを簡潔に書けます。

Optional Chaining（?. ）
const patient = {
  id: 'P001',
  contact: {
    email: 'tanaka@example.com'
  }
};

// 従来：null チェックが必要
const phone = patient.contact && patient.contact.phone
                && patient.contact.phone.mobile;

// Optional Chaining（?.）：存在しないなら undefined を返す
const phone = patient.contact?.phone?.mobile;  // → undefined（エラーにならない）
const email = patient.contact?.email;           // → 'tanaka@example.com'

// メソッド呼び出しにも使える
patient.notify?.();   // notify メソッドがあれば呼ぶ、なければ無視

// 配列要素への安全なアクセス
const first = patients?.[0]?.name;
Nullish Coalescing（??）
// ?? — null または undefined のときだけデフォルト値を使う
const hba1c = patient.hba1c ?? '未測定';

// || との違い
0     ||  'デフォルト'  // → 'デフォルト'（0 は falsy なので）
0     ??  'デフォルト'  // → 0（null/undefined でないから）
null  ??  'デフォルト'  // → 'デフォルト'

// ??= — 代入演算子と組み合わせ
patient.bloodType ??= '不明';  // null/undefined のときだけ代入

// 実用例：設定値の取得
function getConfig(userConfig) {
  return {
    timeout:  userConfig?.timeout  ?? 5000,
    retries:  userConfig?.retries  ?? 3,
    endpoint: userConfig?.endpoint ?? 'https://api.example.com',
  };
}
